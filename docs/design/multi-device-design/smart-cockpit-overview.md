# 概述

智能汽车已经从交通工具演变成移动的智能空间，用于日常通勤、家庭出游、商务接待等场景。座舱内的屏幕也从一个演变成多个，覆盖前排驾驶所需的中控屏、仪表、HUD，后排驻车娱乐的幕布和吸顶屏，联动手机、手表、PAD、PC等全场景设备，结合车外投影大灯、舱内语音、手势等多模态交互，打造视觉、听觉、触觉等多感官的沉浸体验。智能座舱作为出行场景下的核心设备，围绕一瞥可见、一目了然、一触即达，努力为用户带来更加安全可控、智能易用、空间沉浸的驾乘体验。

### 交互设计原则

为共同打造安全、流畅且优雅的智能座舱体验，确立以下核心交互原则。这些原则旨在为所有生态服务提供明确的设计导向与协同基准，共同定义卓越用户体验的基石。

### 保证基础体验

在应用/服务设计中，需要关注到设计在多种屏幕中的使用体验，遵守一些基础体验要求，如果不满足这些基础要求，则会极大损害用户的使用体验。例如，如果界面元素的响应热区太小会导致用户很难操作成功，从而无法完成要操作的任务。具体要求请参阅应用 UX 体验标准。

### 保证驾驶安全

智能座舱的使用关乎用户的生命安全。在驾驶场景下，用户对于座舱系统的使用处于 “双任务” 状态，即需要在保证驾驶任务安全性的同时，分出有限的注意资源去完成与座舱系统的交互任务。为保证驾驶安全，在智能座舱的应用和服务设计中，应遵循安全可控，高效操作的原则。具体要求请参阅智能座舱 UX 体验标准。

在此原则下，还需特别关注以下要点：

1. 硬件遮挡：座舱内方向盘、仪表盘边缘等物理结构可能遮挡屏幕部分区域（如中控屏左下角）。设计时应避免将关键控件或信息置于易被遮挡的位置，并考虑驾驶姿态下手指触控的盲区与偏移。

2. 可视可说：所有服务的核心功能应支持全局语音交互（“可见即可说”），确保在驾驶等关键场景中用户无需触控即可安全调用。同时，严格杜绝任何可能长时间干扰驾驶的交互设计（如复杂手势、需长时凝视的浏览），非安全相关的通知必须克制且非侵入。安全与信任是体验的绝对基石，所有设计必须以此为前提。

### 视觉设计原则

### 内容易于阅读

1. 文字清晰易读，保证主要文本内容的字号、颜色、对比度，以及控制文本的长度。

2. 视觉设计需兼顾不同环境光下的内容可读性，保证在白天和夜间信息读取效率及降低对用户的注意力干扰。

3. 关键信息放在驾驶员水平向下30-35度视野范围内，保证用户驾驶过程中仍能注意到，并避免方向盘对信息的遮挡。

4. 避免呈现的信息干扰用户，如滚动文字、视频或复杂的动态图片。

### 目标易于操作

1. 保证合适的操作位置与热区，若用户操作过远或过小的目标时，容易分心和出现误操作。

2. 在操作热区空间较小时，为适应车辆上下颠簸，可略微牺牲宽度以保障热区高度。

3. 保证操作热区之间的安全距离，避免操作区靠太近或重叠。

### 保持界面一致性

1. 用户界面元素必须清晰一致，以提升用户了解屏幕内容的效率。

2. 使用一致的图标、字符描述和交互方式。

3. 明显区分可操作和不可操作区域，提高操作准确性和效率。
