# 服务卡片

### 概述

服务卡片，简称“卡片”，是 HarmonyOS 系统一种呈现信息和交互操作的载体，在应用或元服务中提取出及时、核心、实用的信息和操作，提供给用户简洁、高效的体验。

## 卡片简介

将 HarmonyOS 应用和元服务的重要信息以卡片的形式展示在桌面，用户可通过快捷手势使用卡片，通过轻量交互行为实现服务直达、减少层级跳转的目的。卡片的核心理念在于提供用户容易使用且一目了然的信息内容，将智慧化能力融入到服务卡片的体验中供用户选择，同时满足在不同终端设备上的展示和自适应。

## 卡片的添加入口

用户可以轻松查找和管理卡片，HarmonyOS 中提供多种入口供用户发现和添加

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113526.84427785548015916128421557545258:50001231000000:2800:7358809AB6B784B74A9B9893A1AAA6E71C551659CA5FDF8AE2C63BC245CA1EC4.png "点击放大")

* **从待机屏保中添加**

用户在待机屏保下，通过双指捏合/长按进入编辑态，可选择增加/移除卡片。

* **从桌面或锁屏添加**

在桌面管理界面的入口中通过长按应用图标或双指捏合用户可以查找服务卡片，选择尺寸后添加至桌面。在锁屏界面通过长按时钟或双指捏合可以查找锁屏的卡片并添加。

* **从应用中添加**

用户在应用内长按卡片，如果该卡片支持服务卡片能力，可以直接添加至桌面；通过元服务的系统服务面板可调起对应的卡片面板，将卡片添加至桌面。

* **从元服务中添加**

用户在元服务中点击右上角胶囊中“更多”调起元服务面板，可选择卡片添加至桌面。

## 体验原则

* **突出服务内容，而非全是跳转入口**

谨慎使用按钮这类操作控件，若一个卡片仅有一组信息内容，则点击卡片进入服务或应用即可。避免使用按钮给用户暗示以吸引用户点击按钮的情况，除非展示的内容需要有明确的指令。另外，卡片仍然是以展示服务信息为主，过多的使用按钮和操作块会使卡片看起来更像是一个快捷操作或入口，请在设计中避免上述情况。

* **明确划分有限的操作空间**

卡片的使用空间非常有限，请合理地排布文本、图片和操作入口之间的关系。用户一般不会对一张卡片浏览太久，因此展示过于密集的信息会影响阅读体验，将更多的内容和排布留给较大尺寸的卡片。如果在卡片内设计了可操作的入口按钮，请考虑用户操作热区和误操作的概率。此外，请不要在卡片内使用滑动、拖拽和长按等交互手势，避免与系统交互手势产生冲突。

* **展示必要的信息和图片**

在小尺寸的卡片内，展示 1 到 2 个信息点即可，即使用户切换到更大尺寸的卡片，也不要提供过于分散的入口操作。所有信息都请务必关联到应用的业务能力，不要提供不相关的运营内容、广告或流量入口。设计一个整洁的卡片界面，不要为了吸引眼球而使用跳脱的颜色。

* **轻量交互原则**

简化用户可感知的行为，在卡片内请勿定义复杂的交互和场景，通过点击直达服务或进入新服务界面即可。切勿在卡片内执行流程型工作，例如通过点击按钮出现二级选项后再选择命令，或在卡片内使用长按或多指操作。

## 卡片的构成

卡片的显示主要由内容主体、归属的应用/元服务名称和卡片名称构成。开发者应该借助卡片内容和卡片名称清晰地向用户传递所要提供的服务信息。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113527.26360178749686936504964329134042:50001231000000:2800:2124B62DC5942F63C07193EB287A2BA7E108BEB2D50FBC6CE67A78075D32BB57.png "点击放大")

* 卡片在卡片中心显示的名称为应用/元服务名称，不可更改此名称的展示规则。
* 同一个应用还支持多种不同类型的卡片，不同尺寸与类型可以通过卡片管理界面进行切换和选择。
* 服务卡片在上架时建议要包含小尺寸卡片，即 2\*2 尺寸卡片，该尺寸在不同设备上具备最佳兼容性。

### 卡片交互

## 卡片交互基本原则

* **前后呼应**

卡片内支持多热区，跳转到与之对应的内容界面。

* **简单交互**

不要在卡片内设计整张卡片响应的左右滑动；如非必要，也不要提供上下滑动操作；不要在卡片内设计多指、多次点击等复杂手势交互。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113527.49717043272580279400296128161628:50001231000000:2800:CE6B860CC276ED99F5221C025F9FADD1EAF62113860F7B1F824AB9B841A551BA.png "点击放大")

* **避免冲突**

在桌面长按卡片可拖动或管理卡片，因此卡片内不支持长按和拖动操作。

## 卡片与应用界面跳转规则

* 卡片点击可跳转至对应的元服务或应用内界面；
* 当用户进入到元服务或应用界面时，返回到桌面，再次对卡片进行点击时，仍然会到对应元服务或应用界面，而不是上次停留的界面；
* 从应用服务界面返回到桌面时，服务窗口应该先回到当前卡片的位置，而不是应用图标的位置，遵循从哪来到哪去的基本原则；
* 不要在卡片内通过点击或其他操作后出现弹窗、半模态等其他操作界面，卡片是拉起服务的主要入口，避免不一致的交互行为；
* 元服务或应用提供的卡片，点击后只能进入卡片对应的元服务或应用。

## 卡片热区定义

开发者需要保证可操作热区与界面内容的交互逻辑一致，否则可能影响用户对卡片可操作内容的判断。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113528.69844148012813689863457839440077:50001231000000:2800:8FAD52CACCDBD18B46BE2307120C44640A16C58D98B59FEE7986BBDB86B560BB.png "点击放大")

* 建议整张卡片为一个热区，降低用户操作成本。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113528.75789982134925517477302041616095:50001231000000:2800:78692BB5247214FF0DED94E84CE2FF7345757CA870B774307C6BE8D423B92EF2.png "点击放大")

* 若要在卡片内展示多个热区，需要明确进行划分，防止用户误触。

* 卡片的目的是为用户提供更加直观的服务信息呈现，不要让过多的操作按钮占用卡片空间，使卡片看起来过于零碎。
* 划分多功能区时每个对应的热区都应提供点击效果，给用户明确指示哪些是有效的可点击元素。
* 如果某一块内容与应用内的界面关联度很高，可以作为一整个热区展示。

若应用中包含支持添加至桌面的卡片，在界面中长按对应卡片后可展开卡片菜单。

卡片菜单包含 2 部分：

1. 系统级卡片添加菜单，配置能力后不允许更改；
2. 应用卡片操作菜单，如应用的卡片编辑、排序、更改内容等操作。

系统级卡片添加菜单包含 2 个操作：

1. “添加至桌面”直接将该卡片添加到用户当前空余的桌面位置，完成加桌操作；
2. “更多卡片”可以调起卡片添加面板，用户查看该应用/元服务提供的所有卡片并选择添加至桌面。

每个应用可以配置多个服务卡片，开发者可根据业务需要来设计不同的卡片。同时，每个卡片也支持多种尺寸选择，可根据展示信息的多少来匹配适合的尺寸，小尺寸卡片为必开发选项。

此外卡片支持多实例选择，例如：用户手动添加了当地天气的服务卡片，可以通过对单个卡片的重复添加来实现多个实例。

**卡片编辑流程**

开发者可以根据业务需求为卡片设计简要的编辑能力。请调用系统提供的卡片编辑模板，保证在 HarmonyOS 上用户体验的一致性，不要在服务卡片上直接使用“编辑”字样或图标进行暗示，避免不同的编辑方式混乱用户的使用逻辑。

1. 在桌面找到卡片后，若卡片支持编辑，即可长按卡片找到编辑菜单；
2. 点击编辑菜单后进入一级编辑页；
3. 此页为编辑项的详细介绍及入口；图例中①为系统提供的卡片实时预览，不可编辑，②为开发者自定义编辑项

5. 设置编辑流程时请同时考虑多端效果。

## 卡片的刷新机制

卡片提供自动刷新机制，开发者可以定义卡片自动刷新时间和频率，让用户时刻享受最新的服务内容，有规律的内容变更也更符合用户的心理预期。但频繁的内容刷新并不能带来显著的流量增益，甚至会引起用户对信息真实性的质疑以及使用体验。因此，开发者可以根据业务属性对时间的依赖性进行配置，例如消费类卡片，根据就餐时间刷新卡片内容。让卡片的内容刷新更贴近用户使用的自然规律。

|  |  |
| --- | --- |
| **主动刷新**  由应用主动发起或事件更新后执行的内容刷新行为。 | ![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113530.95940403413917791292836499116573:50001231000000:2800:632669EF4E70BE1908FAECFFC552A05A86ACDB90C29236ACCF2FCEA375162519.png "点击放大") |
| **被动刷新**  在没有在后台运行的服务内容仍然需要保持最新状态，可定义固定时间或间隔对卡片进行更新。卡片刷新时间的频率以 30 分钟为基础，开发者可根据业务性质向上叠加时间长度。例如，影音娱乐类的卡片没有在后台运行时，用户可以每隔 2 小时获取到最新的数据信息，以减少系统后台运行的功耗开销。 | ![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113530.84022357773791424529702582741286:50001231000000:2800:20C398441CA629D9D3255D5D92B7A064D6269AD3A0DD0D7FCE47B44422F86DC7.png "点击放大") |

## 卡片刷新的动效过渡

无论主动还是被动刷新，当用户停留的桌面上需要执行卡片刷新时，为避免用户界面的大幅跳变，应使用恰当的动效衔接完成卡片刷新。常见的动效过渡包含：

|  |  |
| --- | --- |
|  |  |
| **变化**  卡片内按钮等小元素需要响应用户的操作刷新时可使用简单变化的方式或使用 symbol 图标的默认动效，以达到快速响应的效果。例如：点击播放音乐后，播放按钮刷新为暂停按钮。 | **元素滑动**  日期、数据等数字元素可使用元素翻转的选择器滑动动效，以展示距离上次刷新时的变化。例如：天气实时温度发生变化时，翻转数字显示较上次刷新温度上升或下降了。 |
|  |  |
| **渐隐渐现**  整张卡片内容、底图、背板等大面积元素变化时建议使用渐变效果以保障桌面上卡片显示的稳定性。例如：信息资讯切换新的推送内容在卡片上执行变化刷新。 | **推移补位**  当卡片的内容进行周期性变化或轮播时，可使用带有方向性的推移补位动效以表达卡片内容在一定范围内变化。例如：图库的图片内容定时刷新。 |

建议卡片开发中选择 HarmonyOS 的动效形式，开发出一致、高效、流畅的卡片动效体验。

### 卡片内容设计

开发者可以定义每一张卡片的功能，通常来说，卡片尺寸越小对应展示的功能和信息就越少，所能使用的控件也会受限制。请根据核心诉求进行卡片设计，聚焦在关键信息的展示上。

## 尺寸与基础参数

卡片按照桌面宫格布局，以手机 4\*6 宫格为基础分为 4 种尺寸；在锁屏上还支持 1\*1 圆形卡片尺寸。

卡片内容尽可能保证在安全范围内，在圆角剪裁时避免对内容造成影响。

## 卡片布局

卡片的布局应当追求秩序感、整洁度以及特征性。秩序感是对卡片内容布局的规则性约束，要确保内容是有序的，对齐的，阅读流畅的。整洁度以内容质量为主，除核心信息外，不要附加格外的，确保一定留白，保证用户的首眼体验。特征性可以考虑以卡片垂类或是自身品牌特征作为体验，例如品牌色和品牌纹理的使用，特征图形和布局的呈现。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113530.63102657253110429567344370335739:50001231000000:2800:39BA3FB9F234C5B172DB08D447E6832F607B12D02A95E5B9BEFB65E61CC6FBDA.png "点击放大")

当卡片内容布局在不同场景下都是固定时，建议默认以左对齐为主。卡片内容需要保证距离边缘 12vp 的安全边距，以免在圆角剪裁时对内容展示造成影响。

在特殊布局下，也依然要遵守卡片的基础对齐、留白、间距等原则。

## 合理融入品牌特征

适当的将应用品牌色或标示融入到卡片中，从而让卡片更具有辨识度，但请不要打破卡片设计规范原有的规则。

卡片内不建议再出现应用名称，在 HarmonyOS 出现卡片的场景都会伴随着服务名称或所归属的应用名称，因此珍惜卡片内有限的空间用来展示更重要的信息内容。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113533.36171468367930700947610398375293:50001231000000:2800:50FF3391648677D1470A3F1B44E1BF48A3D7748E1C632CC34CE105DB75AA63D5.png "点击放大")

你可以使用文字 LOGO 或有效的服务信息名来作为卡片标题，服务信息名称并不等同于应用名称，请统一放置在左上角。若使用纯图标，请统一放置在右上角。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113533.72322703980060905123558321641134:50001231000000:2800:B38CD73C9C44BB9A856CFBA795D1839E405B1504F804F5058EFB49E0B5E0C439.png "点击放大")

为了方便凸显应用的品牌形象，可以选择在卡片右上角露出应用图标，或可以选择使用其他的异形图标作为点缀。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113533.00144226933311355788331957688414:50001231000000:2800:453BFD8DC9D871C46952FEB8A6F22E0CE7CA3A29CC32A568771D8B91A39749F5.png "点击放大")

文字 LOGO 一般运营属性较强，通常形状上较长且不规则，在布局上建议以左对齐为主，符合用户的阅读和视觉动线。

单独使用品牌 LOGO 时，请使用 20\*20vp 大小的图标，以及 4vp 的圆角。若 LOGO 造型为异形，请确保尺寸不要超过标准尺寸大小，注意色彩及反白等情况，避免图标与背景分离不清晰。

**清晰可见**

待机屏保下出现的卡片建议通过放大和重新排列文本来充分利用展示空间，让用户可以从较远的距离轻松浏览卡片内容。

**沉浸视觉**

待机屏保下出现的卡片应避免大面积使用明度过高的浅色给用户造成视觉上的割裂感，应合理运用色彩的对比度来提升内容的识别度。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113534.55841493930337507920438219494369:50001231000000:2800:763393E9C77956B9B647D6CFAFD4B30D6A6D3C81D6EEA4CD885DA46FDEEEA8EB.png "点击放大")

**轻量交互**

待机屏保下出现的卡片也应遵循简单交互的基本原则。用户在待机屏保下，通过双指捏合/长按进入编辑态，可选择增加/移除卡片。为了避免与待机屏保的系统交互手势相冲突，请不要在卡片内设计左右滑动和上下滑动操作，也不要在卡片内设计长按、拖动、多指、多次点击等复杂手势交互。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113534.07243607539020317812138443794196:50001231000000:2800:15DF93B9BBC29B4F76F5E07FF3D457C152DB81065B8453B197182D4E823A6ADC.png "点击放大")

**刷新动效**

遵循卡片的基础刷新动效规则，在待机屏保下需要执行卡片刷新时，尤其是用户交互时，推荐使用使用恰当的动效衔接完成卡片内容刷新。常见的动效过渡包含：

|  |  |
| --- | --- |
|  |  |
| **变化**  卡片内按钮等小元素需要响应用户的操作刷新时可使用简单变化的方式或使用 symbol 图标的默认动效，以达到快速响应的效果。例如：点击播放音乐后，播放按钮刷新为暂停按钮。 | **渐隐渐现**  整张卡片内容、底图、背板等大面积元素变化时建议使用渐变效果以保障桌面上卡片显示的稳定性。例如：图库的图片内容定时刷新。 |

## 互动卡片

在 HarmonyOS 中支持桌面服务卡片元素超出服务卡片宫格的能力，支持开发者设计和开发效果更丰富、互动感更强的服务卡片，但在设计和开发中对 UI 元素和动效使用的要求更高，请按需选择互动卡片能力。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113535.06252589462883769405528200307934:50001231000000:2800:D7E41459EF4CE231475E508A400387B9BC129824ACD88C7DFEE7A55558A2AE23.png "点击放大")

**互动卡片的显示场景**

在卡片中心、分享等非桌面场景，互动卡片仍然以常规服务卡片状态显示，禁止元素出框，请使用常规布局的卡片填充相关场景。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113535.02345867019904445765148702461669:50001231000000:2800:9F0A0441D6A70B43A771663F187017804EBA9C4A1D53D58CEB45B074F0C4F923.png "点击放大")

**互动卡片类型**

卡片被用户添加至桌面后，允许元素出框，区分以下 2 种能力，请仔细甄别能力与使用场景：

|  |  |
| --- | --- |
|  |  |
| **场景刷新卡片**  在卡片数据与元素执行刷新时 (主动刷新或被动刷新)，可执行 1 次刷新动效并伴随元素出框，通常用户静态元素的微动效展示。 | **交互刷新卡片**  在用户点击卡片时，可执行 1 次刷新动效并伴随元素出框，随后在 1 秒内回到默认状态，通常用于高动态类游戏卡片的交互展示。 |

**元素出框的限制原则**

为保证卡片的整体性，建议元素选择单边或双边出框，尤其在场景刷新卡片中，避免所有边缘出框的情况。全出框仅限短暂的交互动态场景使用。

**元素出框的范围**

为了保证桌面元素不被干扰和遮挡。稳定态展示范围顶部不超过 16vp，底部和左右两侧不超过 8vp。过程态展示范围建议四周不超过 56vp。

## 卡片多设备适配

**根据尺寸的不同，提供对应的功能**

在卡片开发过程中请考虑多设备的适配性，特别是在泛手机端设备上，设备屏幕对卡片尺寸的改变直接影响了卡片内对数据信息的展示。请发挥想象力设计适合不同尺寸的卡片，避免在卡片内容不做任何处理的情况下直接适配成较大尺寸，原则上尺寸越大呈现的信息、内容成分要越丰富，可交互的范围也越大。

![](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260210113536.26816867237027902397962243356155:50001231000000:2800:F64D33C7A73FE5797F25586B1267B2726A1895F578688342CDF8486B85E4A1CF.png "点击放大")

**使用百分比方式配合标注**

由于设备尺寸的不确定性会导致卡片的尺寸发生变化，设计师在交付卡片布局的过程中，可以适当的使用百分比进行标注。

**多端适配尺寸**

卡片需要适配除手机以外的折叠屏及平板等设备，可以在此处下载对应文件，内包含对应的桌面宫格和详细尺寸描述。设计卡片时不可仅依赖卡片设计模板的样式，还需要查看桌面上根据宫格变化尺寸的预览效果，以确保合适的体验。
