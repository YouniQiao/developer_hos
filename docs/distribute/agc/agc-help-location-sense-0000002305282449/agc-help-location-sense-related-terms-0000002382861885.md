| 术语 | 说明 |
| --- | --- |
| Beacon | 也称为信标设备，它能够通过蓝牙通信实现位置感应，感应距离通常在5~30米。与POI相比，服务推荐更为精准。 |
| BLE | 全称为Bluetooth Low Energy，也称之为低功耗蓝牙，属于蓝牙技术中的一种。与之对应的是Classic Bluetooth，经典蓝牙。低功耗蓝牙虽工作原理与经典蓝牙类似，但有着独特的架构体系，所以LE独立出来成为一种蓝牙形态。BLE与经典蓝牙相比，最大的优势是功耗，BLE的功耗比经典蓝牙降低了90%，同时传输距离增大、安全和稳定性提高，支持AES加密和CRC验证，对连接设备通信的安全性提供了保障。 |
| HarmonyOS标签 | 是一种基于App Linking服务的专属标签，主要应用于线下服务引导、活动推广等场景，方便HarmonyOS NEXT设备用户通过系统提供的碰一碰或扫一扫入口快速直达服务。 |
| HarmonyOS意图框架 | 简称“意图框架”，是华为HarmonyOS提供的一套系统级的意图标准体系。 |
| Major | 信标广播中的Major字段值，由4位16进制数组成，允许自定义。  更详细的解释可参考BLE标准协议。 |
| Minor | 信标广播中的Minor字段值，由4位16进制数组成，允许自定义。字段值转换成十进制后表示信标设备的实际广播间隔，单位为毫秒，即XXms广播一次。  更详细的解释可参考BLE标准协议。 |
| NFC | NFC（Near Field Communication，近场通信）是一种可靠的技术，经过高效集成可以极大改善用户体验。它基于13.56MHz无线通信协议，支持在两个相隔不超过10厘米且支持NFC功能的设备之间进行无线通信。 |
| POI | 全称为Point of Interest，即兴趣点，它是地图上具有特定地理坐标的点类数据，通常包括写字楼、商店、景点、学校、医院等有意义的地点。接入近场服务后，当用户进入以POI为圆心，约200米为半径的地理围栏时，系统将会智能推荐用户所需的服务，如景点语音讲解、食堂付款码等，从而显著提升用户体验。 |
| UUID | 全称为Universally Unique Identifier，信标设备标识符。全局唯一，由32位16进制数组成，不足32位时，系统将自动补齐32位。  更详细的解释可参考BLE标准协议。 |
| 设备ID | 在近场服务中使用的设备ID，是由信标设备的UUID+Major+Minor组成的40位16进制数。其中UUID为32位16进制数，须以**6bff00f723fdf74714030**作为前缀且不支持大写字母，Major和Minor须分别转换为4位16进制数。 |
| 信标设备 | 不存在网关模块，无法直接通过互联网与设备云对接，是一种基于BLE的精确微定位技术，帮助手机终端实现位置感知的魔盒设备。 |
| 意图 | 意图框架中开发者与HarmonyOS对接的标准元素，一个意图代表HarmonyOS应用/元服务内一个为用户提供的业务功能。 |
