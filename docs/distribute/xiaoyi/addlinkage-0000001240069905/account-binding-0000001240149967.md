# 账号绑定接口

## **1. 服务器Url**

https://hag.cloud.huawei.com/open-ability/v1

## **2. 账号绑定通知**

**基本信息**

方法：POST

URL：/open-account-events/bind

说明：开发者服务器通知小艺开放平台服务器，绑定开发者使用的OpenID和华为账号UserId

**请求头域**

| 头域名称 | 描述 | 是否必选 | 类型 | 取值范围 |
| --- | --- | --- | --- | --- |
| x-appid | 开发者在华为开发者联盟注册的小程序的编号 | 是 | string | 正则Pattern：[0-9]\{1,20\} |
| Authorization | 应用级AccessToken，前面需要加上Bearer。 | 是 | string | 正则Pattern：[0-9]\{1,20\} |
| Content-Type | 固定值，填application/json。 | 是 | string | 正则Pattern：[0-9]\{1,20\} |
| Accept | 固定值，填application/json。 | 是 | string | 正则Pattern：[0-9]\{1,20\} |

**请求消息**

* 名称: requestBody
* 是否必选：是
* 类型：OpenAccountBindReq
* 描述：请求消息

**响应消息**

**响应码：200**

* 响应体类型:
* 描述：绑定通知成功。

**响应码：400**

* 响应体类型: ResultInfo
* 描述：参数校验失败，code为：invalidParam。

**响应码：401**

* 响应体类型: ResultInfo
* 描述：网关验证应用级AT过期或AT无效，需要重新申请AT。

**响应码：403**

* 响应体类型: ResultInfo
* 描述：网关验证处理开发者权限失败，没有权限调用小艺开放平台的服务。

**响应码：429**

* 响应体类型: ResultInfo
* 描述：限流，因为客户端请求数超过配额而进行限流，code为： usageLimits。客户端可以以指数退避方式延时重试，重试周期最小30s起。

**响应码：500**

* 响应体类型: ResultInfo
* 描述：服务器内部错误， code为：backendError。客户端可以以指数退避方式延时重试，重试周期最小1s起。

**响应码：其它错误码**

* 响应体类型:
* 描述：其他错误

  对于4xx响应，客户端应该根据error信息，定位解决问题。

  对于5xx响应，客户端可以以指数退避方式延时重试，重试周期最小1s起。

## **4. 基本类型结构**

**基本类型结构**

## **4.1 ResultCode**

结果类型，可以作为调用者在代码中识别结果细分类型的依据。

| 类型 | 取值范围 |
| --- | --- |
| string | 枚举值： invalidParam, tokenExpire, authFailed, userNotFound, usageLimits, backendError |

## **4.2 ResultDesc**

错误详情，帮助问题定位，人工阅读

invalidParam： 描述错误的参数，如：Parameter \{\} is invalid/should not be null/should be in [x，y] 等。

tokenExpire： 描述错误本身，如： User AccessToken Expired。

authFailed： 描述受限访问的资源，如： Forbid notify message of other app。

userNotFound： 描述错误本身，如： User not found。usageLimits： 描述受限的具体原因：如：Request per minutes exceeds max limit。

backendError： 不描述具体错误。直接返回 Backend server error。

| 类型 | 取值范围 |
| --- | --- |
| string | / |
