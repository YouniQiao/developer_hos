# 换算规则描述

编辑商品（或应用）价格时，当您在设置完“汇率换算基准价格”并点击刷新后，系统会根据当时汇率（及[税率](`https://developer.huawei.com/consumer/en/doc/start/merchant-service-0000001053025967#section154132916309`)）和如下币种美化/更正规则换算出所选国家/地区商品的用户支付价格（含税）。

## 用户支付价格（含税）换算规则

* 如果您设置的是“含税”类型的“汇率换算基准价格”，则用户支付价格（含税）保持现有计算规则：
  + 用户支付价格（含税）=汇率换算基准价格（含税）\*汇率
  + 税额=[用户支付价格（含税）/（1+税率）]\*税率

  示例：假设汇率换算基准价格（含税）为人民币7.08元，A国使用美元，汇率为“人民币：美元=1：7”，税率10%，则该国用户支付价格（含税）=7.08\*(1/7)=1.0114 USD，进行美化/更正处理后，最终为1.06 USD，税额=（1.06/(1+10%)）\*10%= 0.096363 USD， 采用“四舍五入”方法后即为0.10 USD。

* 如果您设置的是“不含税”类型的“汇率换算基准价格”，则按照如下规则换算出用户支付价格（含税）：
  + 用户支付价格（含税）=汇率换算基准价格（不含税）\*汇率\* (1+税率)
  + 税额=汇率换算基准价格（不含税）\*汇率\*税率

  示例：假设汇率换算基准价格（不含税）为人民币7元，A国使用美元，汇率为“人民币：美元=1：7”，税率10%，则该国用户支付价格（含税）=7\*(1/7)\*(1+10%)=1.10 USD，税额=汇率换算基准价格（不含税）\*汇率\*税率x%=7\*(1/7)\*10%=0.10 USD。

* 税额计算过程中，不会进行美化/更正处理，仅采用“四舍五入”方法保留两位小数。
* 计算出的用户支付价格（含税）如含有小数，则还会进行币种的美化/更正处理，具体请参考[币种美化/更正规则](#section1752011185611)。

## 币种美化/更正规则

非美元币种国家/地区按照当时的汇率先将设置的“汇率换算基准价格”兑换为以美元为单位的价格，再由以美元为单位的价格兑换为该国家/地区币种为单位的价格，如涉及税率，还需加上对应税额，并按照如下币种美化/更正规则调整为最终商品价格，即“用户支付价格（含税）”。

* 通用币种要求国家/地区：按照汇率换算后，视价格不同情况进行如下调整。
  + 如汇率换算后价格正好为整数或仅有两位小数，则将此价格（保留两位小数）作为商品（或应用）的最终价格。如换算后价格为1.23，则默认选取1.23作为该商品（或应用）的最终价格。
  + 如汇率换算后价格含有三位及以上的小数，则自动截断第二位（不含）后面的小数部分，并根据第二位小数X的情况进行调整。
    - 如0≤X≤5，则第二位小数取值为6，如换算后价格为1.213，则最终价格为1.26。
    - 如6≤X≤8，则第二位小数取值为9，如换算后价格为1.264，则最终价格为1.29。
    - 如X=9，则将截断后的价格加上0.01作为最终价格，如换算后价格为1.2967，则最终价格为1.30。

* 特殊币种要求国家/地区：部分国家币种存在特殊要求，包括仅支持整数和以五分之一为最小单位（均保留两位小数）。
  + 仅支持整数的国家/地区，按照汇率换算后，若为非整数，则向上取整（保留两位小数）作为最终价格。如换算后价格为5.02，则默认选取6.00作为最终价格。
  + 以五分之一为最小单位的国家/地区，按照汇率换算后，若为非整数，则向上取符合五分之一要求的数值（保留两位小数）作为输入价格。如换算后价格为1.23，则默认选取1.40作为该商品的最终价格。
