| 问题/需求描述 | 意见 |
| --- | --- |
| 商品创建完成后，不支持修改的字段有哪些？ | 商品创建完成后，不支持修改的字段：  produceNo 商品ID  purchaseType 商品类型  subGroupID 订阅分组ID |
| 首次接入，沙盒测试商品也需要随版本上架才能测试吗？ | 沙盒测试商品无需随版本上架审核，创建成功保存即可用于测试。 |
| 商品上架的周期大约多久？ | 第一次上架应用添加商品，应用审核通常需要2-3天，应用版本审核通过后，关联的商品也会生效，再次单独提交商品，审核周期目标是24H内。 |
| 上下架商品是否需要提交审核？  （下架或者重新上架） | 不需要（上架后，下架，再重新上架就不需要审核流程）。 |
| 对于新用户促销，该如何使用？新用户促销PMS建好之后并没有offerId。 | 优惠ID只针对的是自定义促销，推介促销由平台直接判断使用，开发者可通过hasEligibilityForIntroOffer字段(详细参数参见iap.queryProducts返回值)判断用户是否享受过推介促销。 |
| 两个主体为开发者的母公司和子公司的关系，母公司签约和开通了数字商品服务，子公司还需要单独签约和申请开通服务吗？ | 数字商品服务开关在后台是按照APP维度的，但数字商品服务协议的签署是按照开发者主体维度的，因此即使母公司主体和对应应用已经签约和开通服务，子公司也需要单独签约、完成服务开通。  [配置数字商品服务](https://developer.huawei.com/consumer/cn/doc/app/digital-products-serve-0000001931836308) |
| 创建数字商品可选择的方式有哪些？ | 新增商品共有三种方式：  1. 通过在AGC控制台新增单个商品的方式；  2. 通过在AGC控制台批量导入商品的方式；  3. 通过创建商品API接口创建单个商品，或者通过批量创建商品API接口批量创建商品。 |
| 数字商品价格是否默认含税以及其他国、地区税率、汇率？ | 1. 默认为含税类型，设置价格时开发者选择基准国家/地区的价格是否含税，并以此计算出全球各国家/地区的最终含税价格；  2. 此价格为展示给用户查看和支付的数字商品价格（含税），华为将根据当地税率，扣税后与开发者分成，分成以对账单为准。 |
| 数字商品外币币种的汇率更新频率？ | 现阶段华为汇率每日刷新，但是不会更改开发者已经保存的商品价格，如果开发者需要刷新商品价格，可以手动根据当前最新汇率刷新。 |
| 同一订阅组中是否有订阅商品的创建数量限制？用户订购自动续期订阅商品时，对于一个订阅组，可以订阅多少个商品？ | 1. 每个自动续期订阅商品必须属于某个订阅组，一个订阅组中可以包含多个订阅型商品；  2. 同一时刻用户只能购买订阅组中的一个商品，不同订阅组相互之间不影响，可以同时订阅；  3. 订阅组内当前没有商品的创建限制数量，但是单个应用创建商品的上限是50000个。 |
| 数字商品审核一定要随应用版本一起提交吗？还是仅限首次提交场景？ | 为应用提交首个数字商品或新增数字商品类型时，必须同时提交新的应用版本。 |
| 商品要提交审核后才能在APP内使用和支持购买吗？为什么审核不通过的商品在沙盒环境下可以支持购买，在审核APP版本时发现无法购买导致版本无法通过？ | 1.数字商品需先提交审核，审核通过后商品才会正式生效、才能上架到AppGallery，支持在生产环境下的APP内展示和购买；  2.在商品提交审核前，推荐使用沙盒测试功能进行商品调测：通过设置沙盒测试账户，在沙盒环境下模拟数字商品的交易过程，在测试期间发现问题可以及时进行修复，这样确保了在商品上架生效以后可以提供给用户稳定流畅的商品购买体验。 |
| 商品在添加时，选择错了类型无法修改，由于相关商品已在支付宝申请通过同项，当前商品id被占用无法修改，希望可以协助修改商品类型。 | 官网操作指南已明确提示，商品创建成功后，商品类型和ID无法修改。 |
| 开发者计划上架一个公开测试版本，可以关联商品提交审核吗？ | 应用市场支持发布公开测试版本，公开测试版本可以关联商品提交审核，版本审核通过后商品也随之生效。 |
| 订阅取消的具体路径？ | 用户可以通过系统设置及华为应用市场提供的订阅管理功能取消订阅。开发者可以在App内通过接口拉起订阅管理界面。 |
| IAP Kit 应用内购为沙盒环境，在[开发者中心]-\&gt;[我的报表]-\&gt;[支付报表]中可以查询到对应的订单吗？ | 不可以，支付报表中仅统计和展示生产环境的真实订单。 |
| 用户在开通会员选择支付方式时，鸿蒙系统除了使用华为支付模式，可以使用微信或支付宝等第三方支付方式吗？ | 数字商品服务收银台包括华为支付、支付宝、微信支付等支付方式。 |
| 用户扣费失败后是否存在轮询扣费？周期多久？ | 若扣费失败，会在一定期限内重复尝试扣费，若失败次数过多，最终会停止重试。目前是24小时内重试5次，之后如果失败则进入失效期，失效期每天尝试一次，之后结束尝试。失效的订阅最长有180天的保留期，在保留期内，用户无法享受订阅型商品的服务，但可以恢复订阅以再次享受应用提供的服务。 |
| 用户可申诉的时间？ | 用户申诉仅限已申请过退款的订单，在订单交易后的180天内可以发起申诉。 |
| 数字商品服务的结算款收款账号是不是在华为后台填写的商户服务银行账号？ | 是的，用于后续结算。 |
| 用户开票，发票抬头是伙伴侧还是华为侧？ | 用户使用华为数字商品收银台付款，收款方为华为，发票由华为向用户开具。 |
| 不同国家税率不同，这部分的结算如何操作？ | 结算明细报告中会有税率字段信息。  税率可参考文档：https://developer.huawei.com/consumer/en/doc/start/merchant-service-0000001053025967#h1-1591000387924-0 |
