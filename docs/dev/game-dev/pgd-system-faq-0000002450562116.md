## 系统更新顺序

系统的更新顺序是按照注册顺序确定的，因此在具有明确依赖关系的系统间，需要准确按照依赖关系注册系统，避免出现时序问题。

下例中，渲染系统是根据Transform渲染游戏对象的，因此用于更新实体Transform数据的系统需要在渲染系统之前执行，需要在注册处理Transform系统后注册渲染系统。

```
// 问题：依赖的数据在系统执行时还未准备好
public class RenderSystem : PgdSystem<PgdPosition, Sprite>
{
    protected override void OnUpdate()
    {
        // 如果MovementSystem还没执行，位置数据可能是过期的
        GetQuery().ForEachEntity((ref PgdPosition pos, ref Sprite sprite, IEntity entity) => {
            RenderSprite(sprite, pos.vec3);
        });
    }
}

// 解决：确保系统添加顺序符合依赖关系
world.RegisterSystem(new MovementSystem());  // 先更新位置
world.RegisterSystem(new RenderSystem());    // 再渲染
```
