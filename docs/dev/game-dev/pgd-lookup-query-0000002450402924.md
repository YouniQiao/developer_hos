Lookup索引系统需要感知组件值的变化，才能将实体从旧值的索引移动到新值的索引。

| 方式 | 索引更新 | 特点 |
| --- | --- | --- |
| entity.Set\&lt;T\&gt;(component) | 自动更新。 | 框架能捕获变更并同步更新索引。 |
| entity.AddComponent\&lt;T\&gt;(component) | 自动更新。 |
| entity.Add\&lt;T1, T2\&gt;(...) | 自动更新。 |
| entity.Remove\&lt;T\&gt;() | 自动更新。 |
| 直接修改Component引用，或在Query中修改Span | 不会更新，且会影响Set功能。  禁止单独或者混合使用非上述接口修改被Lookup索引的组件值。 | 只改变了数据，框架无法感知，索引不会更新。 |

在使用Lookup功能之前，请务必理解关键点，否则会导致索引不同步问题。示例如下：

```
public struct PlayerIdComponent : ILookup<int>
{
 public int PlayerId;
 public int GetLookup() => PlayerId;
}
var player = world.CreateEntity(new PlayerIdComponent { PlayerId = 1001 });

// 正确：使用entity.Set()方法
player.Set(new PlayerIdComponent { PlayerId = 2001 }); // 索引自动更新
// 正确：在ForEachEntity中使用Set
query.ForEachEntity((ref PlayerIdComponent c, IEntity e) =>
{
 e.Set(new PlayerIdComponent { PlayerId = 2001 });
});

// 错误：直接修改引用（索引不会更新！）
ref var comp = ref player.GetComponent<PlayerIdComponent>();
comp.PlayerId = 2001; // 索引仍然指向 1001
// 错误：通过Query的Span修改（索引不会更新！）
var query = world.Query<PlayerIdComponent>();
query.ForEachEntity((ref PlayerIdComponent value, IEntity entity) =>
{
    value.PlayerId = 2001;
});
```

## 基本值查询

通过HasValue查询哪些实体有指定值。

```
// 创建测试实体
var player1 = world.CreateEntity(
    new PlayerName { name = "Alice" },
    new PlayerLevel { level = 25 },
    new PlayerClassComponent { playerClass = PlayerClass.Warrior }
);

var player2 = world.CreateEntity(
    new PlayerName { name = "Bob" },
    new PlayerLevel { level = 30 },
    new PlayerClassComponent { playerClass = PlayerClass.Mage }
);

var player3 = world.CreateEntity(
    new PlayerName { name = "Charlie" },
    new PlayerLevel { level = 25 },
    new PlayerClassComponent { playerClass = PlayerClass.Archer }
);

// === HasValue查询：查找具有特定值的实体 ===

// 查找名为"Alice"的玩家
var aliceQuery = world.Query<PlayerName>().HasValue<PlayerName, string>("Alice");
Console.WriteLine($"找到Alice: {aliceQuery.EntityCount}"); // 1

// 查找25级的玩家
var level25Query = world.Query<PlayerLevel>().HasValue<PlayerLevel, int>(25);
Console.WriteLine($"25级玩家数量: {level25Query.EntityCount}"); // 2

// 查找战士职业的玩家
var warriorQuery = world.Query<PlayerClassComponent>()
    .HasValue<PlayerClassComponent, PlayerClass>(PlayerClass.Warrior);
Console.WriteLine($"战士数量: {warriorQuery.EntityCount}"); // 1

// 组合查询：25级的战士
var level25WarriorQuery = world.Query<PlayerLevel, PlayerClassComponent>()
    .HasValue<PlayerLevel, int>(25)
    .HasValue<PlayerClassComponent, PlayerClass>(PlayerClass.Warrior);

level25WarriorQuery.ForEachEntity((ref PlayerLevel level, ref PlayerClassComponent cls, IEntity entity) => {
    Console.WriteLine($"25级战士: {entity.Id}, 等级: {level.level}, 职业: {cls.playerClass}");
});
```

## 范围查询

```
// === ValueInRange查询：查找值在指定范围内的实体 ===

// 查找等级在20-30之间的玩家
var levelRangeQuery = world.Query<PlayerLevel>()
    .ValueInRange<PlayerLevel, int>(20, 30);

levelRangeQuery.ForEachEntity((ref PlayerLevel level, IEntity entity) => {
    Console.WriteLine($"玩家 {entity.Id} 等级: {level.level}");
});

// 字符串范围查询（按字典序）
var nameRangeQuery = world.Query<PlayerName>()
    .ValueInRange<PlayerName, string>("A", "C");

nameRangeQuery.ForEachEntity((ref PlayerName name, IEntity entity) => {
    Console.WriteLine($"名称在A-C范围的玩家: {name.name}");
});

// 组合范围查询
var combinedRangeQuery = world.Query<PlayerLevel, PlayerName>()
    .ValueInRange<PlayerLevel, int>(20, 35)
    .ValueInRange<PlayerName, string>("A", "Z");

Console.WriteLine($"符合条件的玩家数量: {combinedRangeQuery.EntityCount}");
```
