## 组件生命周期

修改组件值，需要获取Component引用。

```
var entity = world.CreateEntity();
entity.AddComponent(new Health(100));

// 错误：组件是值类型，这样获取的是副本
var health = entity.GetComponent<Health>(); // 副本
health.current = 50; // 修改的是副本，原组件不变

// 正确：使用ref获取引用
ref var healthRef = ref entity.GetComponent<Health>();
healthRef.current = 50; // 直接修改原组件
```

## 组件不存在异常

```
// 危险：可能抛出异常
try
{
    var component = entity.GetComponent<SomeComponent>();
}
catch (ComponentNotFoundException)
{
    // 处理组件不存在的情况
}

// 安全：使用TryGet方法
if (entity.TryGetComponent<SomeComponent>(out var component))
{
    // 组件存在，安全使用
}
```

## 包含引用类型组件的实体克隆问题

当组件包含引用类型时，克隆或拷贝实体需要特殊处理。

```
// 问题：包含引用类型的组件
public struct Inventory : IComponent
{
    public List<int> items;
    public int capacity;
}
var original = world.CreateEntity(new Inventory
{
    items = new List<int> { 1, 2, 3 },
    capacity = 10
});

// 危险：当前框架支持通过反射自动进行深拷贝，性能急剧恶化。
var clone = original.CloneEntity();

// 解决方案：实现CopyMethod
public struct Inventory : IComponent
{
    public List<int> items;
    public int capacity;

    // 必须提供的深拷贝方法
    public static void CopyMethod(in Inventory source, ref Inventory target, in CopyContext context)
    {
        if (source.items == null)
        {
            target.items = null;
            return;
        }
        target.items ??= new List<int>();
        target.items.Clear();
        target.items.AddRange(source.items);
        target.capacity = source.capacity;
    }
}
```
