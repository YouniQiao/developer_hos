Event模块提供轻量的事件发布-订阅机制，进行事件注册与发布，同时支持基于委托的过滤。事件管理的基础单元为EventHub，每个IECSWorld中持有一个EventHub实例，可以管理当前World中的所有事件。

为了保障性能，事件系统默认处于关闭状态。若开发者需要，可开启EventHub。

```
var world = new IECSWorld();
// 开启EventHub
world.EnableEventHub();
// 关闭EventHub
world.DisableEventHub();
```

PGD提供内置事件，支持组件&标签变更、实体创建等关键操作，当前已支持的内置事件类型和监听接口如下：

| 事件属性 | 简要描述 |
| --- | --- |
| OnComponentChanged | World属性监听组件变化事件。 |
| OnEntityCreated | World属性监听实体创建事件。 |
| OnEntityDestroyed | World属性监听实体销毁事件。 |
| OnTagChanged | World属性监听标签变化事件。 |
| OnChildEntityChanged | World属性监听子实体变化事件。 |
| OnSystemChanged | World属性监听系统变化事件。 |

## 使用建议

事件系统具备一定的性能开销，事件结构体和事件处理回调函数中不宜保存过于复杂的数据和执行逻辑。
