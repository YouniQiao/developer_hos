## 使用EntityFastAccess

当需要通过IEntity对同一个实体进行多次随机组件访问时，使用EntityFastAccess可以获得更好的性能。它通过缓存实体在Archetype中的位置信息，避免了多次重复的节点查找开销。

![](./img/dae516c9.png)

若在Query中，您可以通过ForEachEntity或ArchetypeChunk修改Component更快，因为直接访问原生数组，没有额外的查找开销。

```
var world = new IECSWorld();
var entity = world.CreateEntity();
var fastAccess = entity.EntityFastAccess;
// 比entity.GetComponent<T>()更快
ref var health = ref fastAccess.Get<Health>();
ref var velocity = ref fastAccess.Get<Velocity>();
// 处理逻辑...
```

## 合理使用标签

```
// 好的做法：使用标签进行快速分类
public struct ActiveTag : ITag { }
public struct PlayerTag : ITag { }

// 避免：用组件存储简单的布尔状态
public struct Status : IComponent  // 不推荐
{
    public bool isActive;
    public bool isPlayer;
}
```

## 优化组件大小

```
// 好的做法：紧凑的组件设计
public struct Transform : IComponent
{
    public Vector3 position;    // 12 bytes
    public Quaternion rotation; // 16 bytes
    public Vector3 scale;       // 12 bytes
    // 总计: 40 bytes
}

// 避免：包含不必要的大型数据
public struct BadTransform : IComponent  // 不推荐
{
    public Matrix4x4 worldMatrix;     // 64 bytes
    public Matrix4x4 localMatrix;     // 64 bytes
    public Transform[] children;      // 引用类型
    // 总计: 128+ bytes + 堆分配
}
```
