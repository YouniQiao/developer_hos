HarmonyOS 5.0及以上平台因为安全问题屏蔽部分游戏项目中原来的函数、方法，若您有用到，需对游戏代码中不匹配的函数、方法等进行手动修改适配。

## Cocos Creator 2.X

### 添加cc.sys.OS\_OPENHARMONY平台

游戏代码中可能有Android、iOS等多种平台的判断和处理逻辑，需在平台判断分支处加上对HarmonyOS 5.0及以上平台的处理。您可以全局搜索关键字cc.sys.os，具体修改内容可根据实际情况进行分析。

例如：

```
if(cc.sys.os == cc.sys.OS_ANDROID){
// 游戏逻辑
}
```

修改为：

```
if(cc.sys.os == cc.sys.OS_ANDROID || cc.sys.os == cc.sys.OS_OPENHARMONY){
// 游戏逻辑
}
```

### 屏蔽WebAssembly函数

HarmonyOS平台不支持WebAssembly，相关代码屏蔽后不影响游戏逻辑。例如，下方示例直接绕过HarmonyOS平台处理游戏逻辑：

```
// 若不是HarmonyOS平台，执行下面的游戏逻辑，其中globalThis.oh为HarmonyOS渠道特定全局变量。
if (!globalThis.oh) {
    try {
        i = new WebAssembly.Instance(new WebAssembly.Module(new Uint8Array([173, 66, 32, 134, 132, 32, 2, 173,])), {
        }).exports;
    }
    catch (e) {
    }
}
```

## Cocos Creator 3.X

### 添加sys.OS\_OPENHARMONY平台

游戏代码中可能有Android、iOS等多种平台的判断和处理逻辑，需在平台判断分支处加上对HarmonyOS 5.0及以上平台的处理。您可以全局搜索关键字sys.os，具体修改内容可根据实际情况进行分析。

例如：

```
if(sys.os == sys.OS_ANDROID){
// 游戏逻辑
}
```

修改为：

```
if(sys.os == sys.OS_ANDROID || sys.os == sys.OS_OPENHARMONY){
// 游戏逻辑
}
```

### 屏蔽WebAssembly函数

HarmonyOS 5.0及以上平台不支持WebAssembly，通过使用sys.hasFeature(sys.Feature.WASM)判断当前平台是否支持WebAssembly。例如，如下示例绕过平台处理游戏逻辑：

```
if (sys.hasFeature(sys.Feature.WASM)) {
    try {
        i = new WebAssembly.Instance(new WebAssembly.Module(new Uint8Array([173, 66, 32, 134, 132, 32, 2, 173,])), {
        }).exports;
    }
    catch (e) {
    }
}
```
