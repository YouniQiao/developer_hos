## 性能监控和调试

```
// 监控World状态
Console.WriteLine($"实体总数: {world.EntityNum}");
Console.WriteLine($"原型数量: {world.ArchetypesCount}");
Console.WriteLine($"总容量: {world.TotalArchsCapacity}");
Console.WriteLine($"缩容比例: {world.ShrinkRatio}");

// 设置缩容策略（可选）
world.ShrinkRatio = 5.0; // 当Entity的使用率低于20%时进行缩容，默认低于10%进行缩容。

// 获取World概览
Console.WriteLine(world.ToString()); // 输出: The total number of entity is: 1000
```

## ID回收策略

* ID回收策略默认开启。
* 启用RecycleIdsActive可以重用删除实体的ID，节省内存。

  ```
  var world = new IECSWorld();
  world.RecycleIdsActive = false;   // 禁用ID回收
  ```
* 禁用时ID分配更快，但可能导致稀疏的ID分布。**禁用期间删除的EntityId不会被回收，重启也无法恢复****。**

## 批量创建优势

* 使用archetype.CreateEntities()批量创建大量实体比逐一创建快数倍。
* 适合初始化大量NPC、子弹、粒子等场景。

## ShrinkRatio内存管理机制

ShrinkRatio是PGD的内存管理核心参数，用于防止内存浪费。

### 工作原理

触发条件：TotalArchsCapacity &gt; ShrinkRatio × EntityNum

| 条件 | 说明 |
| --- | --- |
| TotalArchsCapacity | 所有原型的总内存容量。 |
| EntityNum | 当前存活的实体总数。 |
| ShrinkRatio | 缩容触发比例。  默认值10。 |

### 缩容机制

|  |  |
| --- | --- |
| 触发判断 | 每当原型容量发生变化时，计算是否需要缩容。  如果需要缩容，框架内部会设置NeedShrinkRatio标记为true。 |
| 实际缩容 | 在实体删除时，判断当实体数量是否低于capacity/4，且NeedShrinkRatio为true。 |
| 缩容幅度 | 将容量缩减至原来的50%。 |

* 实际应用示例：

  ```
  var world = new IECSWorld();

  // 场景1：创建大量实体后删除大部分
  for (int i = 0; i < 10000; i++)
  {
      world.CreateEntity(new PgdPosition(), new Velocity());
  }
  // 此时 TotalArchsCapacity 约为 10000+

  // 删除90%的实体
  foreach (var entity in world.Entities.Take(9000))
  {
      entity.DeleteEntity();
  }
  // 现在 EntityNum = 1000，但 TotalArchsCapacity 仍然很大

  // 检查是否需要缩容
  bool needShrink = world.TotalArchsCapacity > world.ShrinkRatio * world.EntityNum;
  // 如果 10000+ > 10 * 1000，则 needShrink = true

  Console.WriteLine($"总容量: {world.TotalArchsCapacity}");
  Console.WriteLine($"实体数: {world.EntityNum}");
  Console.WriteLine($"缩容比例: {world.ShrinkRatio}");
  Console.WriteLine($"需要缩容: {needShrink}");
  ```

* 调优建议

  ```
  // 内存敏感场景：更激进的缩容策略
  world.ShrinkRatio = 3.0;  // 当容量超过实体数3倍时就缩容

  // 性能优先场景：保守的缩容策略
  world.ShrinkRatio = 20.0; // 允许更大的内存冗余，减少缩容频率
  ```

### 适用场景分析

| 场景类型 | 推荐ShrinkRatio | 说明 |
| --- | --- | --- |
| 实体数量稳定的游戏 | 10-15 | 默认值，平衡性能和内存。 |
| 大量动态创建/删除 | 3-8 | 更频繁缩容，节省内存。 |
| 高性能要求场景 | 15-25 | 减少缩容操作，保持性能。 |
| 内存受限设备 | 2-5 | 激进缩容，最大化内存利用。 |

### 监控和调试

```
// 定期监控内存使用情况
Console.WriteLine($"内存利用率: {(double)world.EntityNum / world.TotalArchsCapacity:P2}");

// 当利用率持续低于合理范围时，考虑调整ShrinkRatio
if (world.TotalArchsCapacity > 0)
{
    double utilizationRate = (double)world.EntityNum / world.TotalArchsCapacity;
    if (utilizationRate < 0.1) // 利用率低于10%
    {
        Console.WriteLine("考虑降低ShrinkRatio以更频繁地回收内存");
    }
}
```
