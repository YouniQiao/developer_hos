Batch（批处理）是PGD提供的高性能批量处理实体的操作机制，用于优化大量实体的创建和修改操作。传统的逐个实体操作可能导致多次结构性变化，而批处理将这些操作合并为一次或少数几次结构变化，优化结构性变更从而获得性能收益。

## 核心特性

* 高性能创建：一次性创建大量具有相同组件配置的实体。
* 批量修改：对多个实体进行统一的组件添加、移除、标签操作。
* 内存优化：减少内存分配和垃圾回收压力。
* 结构优化：最小化原型（Archetype）变化次数。
* 对象池化：复用批处理对象，避免重复分配。

## 批处理类型

PGD提供两种主要的批处理机制：

| 类型 | BatchEntityBuilder | BatchProcessor |
| --- | --- | --- |
| 主要用途 | 批量创建实体。 | 批量修改现有实体。 |
| 目标对象 | 尚未存在的实体。 | 已存在的实体。 |
| 主要方法 | * CreateEntity() * AddComponent\&lt;T\&gt;(component) | * Process() * ProcessOn() |
| 生命周期 | 实体创建阶段。 | 实体修改阶段。 |
| 性能优化 | 避免多次结构变更。 | 合并多次结构变更。 |
