在元服务启动的过程中，代码注入是一个主要的耗时环节。注入代码量的大小与内存占用与注入耗时正相关。

利用“按需注入”和“用时注入”的特性，可以优化代码注入环节的耗时和内存占用。

app.json中配置如下内容：

```
{
  "lazyCodeLoading": "requiredComponents"
}
```

自ASCF运行时1.0.13起，配置后，元服务会有选择地注入必要的代码，以降低元服务的启动时间和运行时内存。

## 按需注入

**版本限制：**

* ASCF运行时版本1.0.13及以上版本支持，1.0.13以下兼容但无优化效果。
* 工具调试请使用ASCF Toolkit 1.0.6及以上版本。

通常情况下，在元服务启动时，启动页面依赖的所有页面和组件会全部合并注入，包括其他未访问的页面以及未用到自定义组件，同时所有页面和自定义组件的 JS 代码会被立刻执行。这造成很多没有使用的代码在元服务运行环境中注入执行，影响注入耗时和内存占用。

**注意事项**

* 启用按需注入后，元服务仅注入当前访问页面所需的自定义组件和页面代码。未访问的页面、当前页面未声明的自定义组件不会被加载和初始化，对应代码文件将不被执行。**请开发者修改配置后务必确认元服务的表现正常**。
* 启用按需注入后，页面JSON配置中定义的所有组件和app.json中usingComponents配置的全局自定义组件，都会被视为页面的依赖并进行注入和加载。**建议开发者及时移除 JSON 中未使用自定义组件的声明**，并尽量**避免在全局声明使用率低的自定义组件**，否则可能会影响按需注入的效果。对于加载较为耗时的自定义组件，建议配置占位组件以生效“用时注入”特性。

## 用时注入

**版本限制：**

* ASCF运行时版本1.0.13及以上版本支持，1.0.13以下兼容但无优化效果。
* 工具调试请使用ASCF Toolkit 1.0.6及以上版本。

在开启“按需注入”特性的前提下，“用时注入”可以指定一部分自定义组件不在元服务启动时注入，而是在真正渲染的时候才进行注入。

在已经指定lazyCodeLoading为requiredComponents的情况下，不论在主包还是分包，只要为自定义组件配置占位组件（同[分包异步化](/docs/dev/atomic-dev/ascf/develop-subpackage-loading/asynchronous-subcontracting)），组件就会自动被视为用时注入组件：

* 每个页面内，第一次渲染该组件前，该组件都不会被注入。
* 每个页面内，第一次渲染该组件时，该组件会被渲染为其对应的占位组件，渲染流程结束后开始注入。
* 注入结束后，占位组件被替换回对应组件。
