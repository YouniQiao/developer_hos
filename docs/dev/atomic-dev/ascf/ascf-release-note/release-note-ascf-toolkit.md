## 1.0.17

**发布日期**：2026/04/30

该版本新增和增强特性如下所示：

1. 新增行业模板功能，支持开发者通过行业模板快速创建元服务项目。
2. 新增调试面板，支持实时查看Console日志、Api（接口调用与网络请求）及Storage（本地存储）相关信息。
3. 新增配置字段launchOptions，支持配置元服务的启动页面及启动参数。
4. 新增支持ascf.private.config.json个人配置文件，便于开发者进行本地差异化配置。
5. 新增命令行方式配置签名密钥，提升签名流程的自动化能力。

修复的问题如下所示：

1. 优化睫毛图生成工具的取色逻辑，提升颜色提取的准确性。
2. 修复Unicode字符在编译后双引号丢失的问题。
3. 修复开启按需加载时，分包组件作为全局组件无法正确加载的问题。
4. 修复标签data属性值为链式调用时编译报错的问题。

## 1.0.16

**发布日期**：2026/03/31

该版本新增和增强特性如下所示：

1. 新增配置参数compileMode，支持直接编译未转换项目运行启动。
2. 新增配置参数emptyJsonpName, 当存在多个Taro包并开启热更新时需要配置为对应Taro包中Webpack配置的output.jsonpFunction。
3. 当开启按需注入时，进一步优化主包体积。

修复的问题如下所示：

1. 修复require动态变量值时编译报错问题。
2. class样式问题修复。

## 1.0.15

**发布日期**：2026/02/05

该版本新增和增强特性如下所示：

1. 新增generate命令，支持使用命令行创建ASCF元服务项目。
2. 新增支持在HarmonyOS 4及以下设备上调试运行ASCF元服务。
3. 新增支持网络调试功能，支持在HarmonyOS 4及以下设备上调试，能看到网络请求响应信息。

修复的问题如下所示：

1. 修复标签class属性解析问题。
2. 修复标签属性key是data+大写字母场景差异问题，该场景修改为普通属性处理。

## 1.0.12

**发布日期**：2025/10/28

该版本新增和增强特性如下所示：

1. 优化转换命令工具，增强参数校验和提示。
2. 优化构建命令，自动识别JAVA\_HOME和NODE\_HOME配置，不依赖于系统环境变量配置。

5. 修复"a&lt;b"表达式模板编译的问题。
6. 修复Vue3模板编译下已知的问题。

## 1.0.10

**发布日期**：2025/09/12

该版本新增和增强特性如下所示：

* 优化编译构建执行性能。
* 优化调试功能错误提示，请参考[启用调试异常](https://developer.huawei.com/consumer/cn/doc/atomic-ascf/debug-ascf-code#启用调试异常)。

* 解决@swc/helpers依赖路径找不到问题。

## 1.0.9

**发布日期**：2025/09/01

该版本新增和增强特性如下所示：

1. 优化应用间HSP共享库依赖判断逻辑，支持依赖并安装新版本，简化模拟器调试运行
2. 优化分包编译逻辑，解决部分场景下分包编译报错问题。
3. 优化swc编译逻辑，解决部分场景下找不到@swc/helpers的问题。
4. 调试命令对于v8调试功能使用集成devtools，支持自动拉起浏览器调试，提升调试体验。
5. 优化编译构建性能。

## 1.0.8

**发布日期**：2025/07/30

该版本新增和增强特性如下所示：

1. ASCF Toolkit Hvigor Plugin 新增支持元服务使用AtomicServiceEnhancedWeb组件的插件，开发请参考：[开发AtomicServiceEnhancedWeb组件](https://developer.huawei.com/consumer/cn/doc/atomic-guides/develop-atomicserviceenhancedweb)。
2. 优化 Mac OSX系统中依赖全局配置node和hdc的PATH路径要求体验，ASCF Toolkit自动根据默认路径或者环境变量识别。
3. 优化 disableSubpackages 配置逻辑，对uniapp项目暂时默认启用分包，开发者可手动关闭。
4. 优化 templateHoist 配置逻辑，检测到项目中存在 ascf/ascf\_src/base.hxml 文件时默认启用，减少开发者手动配置步骤。

## 1.0.6

**发布日期**：2025/07/22

该版本新增和增强特性如下所示：

1. 编译器支持lazyCodeLoading按需加载功能，在app.json中配置 "lazyCodeLoading": "requiredComponents" 生效。
2. 对于在开启了swc编译的情况下，优化了编译后包的大小。
3. 支持自定义TabBar配置，可参考文档：[开发自定义Tabbar](https://developer.huawei.com/consumer/cn/doc/atomic-ascf/develop-custom-tabbar)。
4. 修复了ASCF Toolkit命令在Mac OSX中路径兼容性问题。
5. 修复了ASCF Toolkit修改JSON文件和重新编译存在的缓存问题。
