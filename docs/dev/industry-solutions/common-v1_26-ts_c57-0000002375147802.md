## 问题现象

在使用鼠标在执行特定操作或将鼠标拖动至特定区域时，用于指示鼠标位置的光标会出现丢失或样式变化的情况。

## 背景知识

* [鼠标光标开发指导](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/pointerstyle-guidelines)：鼠标光标控制功能提供了显示和隐藏鼠标光标的能力、光标样式查询和设置的能力。使用场景例如：用户在全屏观看视频时，开发者可以控制鼠标光标的显示隐藏；当用户执行取色时，开发者可以将鼠标光标样式切换为取色器样式。

  | 接口名称 | 描述 |
  | --- | --- |
  | [setPointerVisible](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-pointer#pointersetpointervisible) | 设置鼠标指针显示或隐藏状态，该接口用于设置全局鼠标光标的显示状态。 |
  | [setPointerStyle](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-pointer#pointersetpointerstyle) | 设置鼠标光标样式，该接口会影响指定窗口鼠标光标样式。 |
* 鼠标光标控制：[cursorControl](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-attributes-cursor#cursorcontrol)控制鼠标光标的显示样式，分为setCursor和restoreDefault。
* [处理鼠标移动](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-interaction-development-guide-mouse#处理鼠标移动)：鼠标事件通过onMouse接口注册一个回调来接收，当鼠标事件发生时，会按照鼠标光标所在位置下的组件进行派发，派发过程遵循事件冒泡机制。

  | 名称 | 描述 |
  | --- | --- |
  | [onHover](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-events-hover#onhover) | 鼠标或手写笔进入或退出组件时，触发hover事件。isHover表示鼠标或手写笔是否悬浮在组件上，进入时为true，离开时为false。 |
  | [onMouse](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-mouse-key#onmouse) | 当前组件被鼠标按键点击时或者鼠标在组件上悬浮移动时，触发该回调。event返回值包含触发事件时的时间戳、鼠标按键、动作、鼠标位置在整个屏幕上的坐标和相对于当前组件的坐标。 |

## 问题定位

根据问题现象分析，光标丢失可能由指针样式变化或光标不显示导致。同时使用UI布局工具获取光标丢失和样式变化的组件区域。

1. 指针样式变化：
   * 搜索setPointerStyle和[setCursor](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-attributes-cursor#setcursor)，查看是否设置改变鼠标样式类型，结合搜索onHover（悬浮事件回调）和onMouse（鼠标事件回调），确认交互逻辑是否合理，特别是检查是否在正确的组件区域控制样式变化。
   * 对于交互的场景的不同需要设置合适的光标显示，参考背景知识光标交互分析设计原则是否符合用户体验。
2. 光标消失：搜索setPointerVisible，查看是否有设置鼠标指针显示或隐藏状态，同时分析触发显示和隐藏的逻辑是否正确。

## 分析结论

* **场景一：**未正确设置组件区域的鼠标悬停事件（onHover）与鼠标点击事件（onMouse）的回调函数，并且未能合理使用setPointerStyle和cursorControl.setCursor方法来动态调整光标样式，导致用户的交互体验不佳。
* **场景二：**通过setPointerVisible设置鼠标光标的显示与隐藏的触发逻辑不正确，如进入视频全屏时隐藏了光标，在退出视频全屏状态时未及时显示光标。

## 修改建议

* **场景一：**在确保光标样式符合[光标交互](https://developer.huawei.com/consumer/cn/doc/design-guides/hmi-cursor-0000001795531205)的前提下，可使用setCursor函数来更改鼠标光标的显示样式，参考[示例](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-attributes-cursor#示例)。
* **场景二：**以用户在全屏观看视频时为例，进入视频全屏时可以调用鼠标光标的隐藏接口设置鼠标光标不可见，在退出视频全屏状态时显示光标，提升用户体验。实现示例参考[设置鼠标光标隐藏](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/pointerstyle-guidelines#设置鼠标光标隐藏)。
