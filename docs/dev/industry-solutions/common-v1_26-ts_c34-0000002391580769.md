## 问题现象

应用在使用过程中，频繁上下滑动列表导致闪退。

## 背景知识

* 用户在使用应用时，如果出现点击无反应或应用无响应等情况，并且持续时间超过一定限制，就会被定义为应用无响应，详情参考[AppFreeze（应用冻屏）检测](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/appfreeze-guidelines)。
* AppFreeze日志规格说明可以参考[日志规格](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/appfreeze-guidelines#日志规格)。
* [渲染范围控制](https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-control-rendering-range)：在应用开发中，控制UI渲染范围，防止界面卡顿或掉帧。
* [布局节点减少](https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-reduce-layout-nodes)：在进行页面布局开发时，应该尽量减少布局节点，避免系统绘制更多的布局组件，达到优化渲染性能、减少内存占用的目的。
* [HiSmartPerf](https://developer.huawei.com/consumer/cn/doc/AppGallery-connect-Guides/smartperf-tool-0000001873208929)：无需ROOT设备，即可准确、高效地采集到应用运行时的CPU、GPU等性能数据，了解应用的性能状况。

3. 使用HiSmartPerf工具抓取该过程的Trace信息，从Duration可以看出preload任务执行时间达到9s。![](./img/e4204dd7.png "点击放大")
4. 框选主线程29310中preload任务所在的泳道，其中绝大部分时间都在执行MeasureListItem节点布局任务。过滤MeasureListItem任务，节点布局任务执行总时长达到6.8s，任务数量达到823个。![](./img/435b435f.png "点击放大")
5. 部分节点布局复杂，布局时间达到525ms，且节点数量过多，结合ListItem的使用场景可知应用在加载列表项时没有采用懒加载的方式。

## 分析结论

页面布局复杂并且长列表未采用懒加载导致应用卡死。

## 修改建议

使用懒加载优化长列表加载速度，参考文档[长列表加载](https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-lazyforeach-optimization#section1847219719551)。

```
interface DataChangeListener {
  onDataReloaded(): void;

  onDataAdd(index: number): void;

  onDataDelete(index: number): void;

  onDataChange(index: number): void;
}

export class MyDataSource {
  private _data: string[] = [];
  private _listeners: DataChangeListener[] = [];

  pushData(item: string): void {
    this._data.push(item);
    this._listeners.forEach(listener => {
      listener.onDataAdd(this._data.length - 1);
    });
  }

  getAllData(): string[] {
    return this._data;
  }

  totalCount(): number {
    return this._data.length;
  }

  getData(index: number): string {
    return this._data[index];
  }

  registerDataChangeListener(listener: DataChangeListener): void {
    this._listeners.push(listener);
  }

  unregisterDataChangeListener(listener: DataChangeListener): void {
    const index = this._listeners.indexOf(listener);
    if (index !== -1) {
      this._listeners.splice(index, 1);
    }
  }
}

@Entry
@Component
struct MyComponent {
  private data: MyDataSource = new MyDataSource();

  aboutToAppear() {
    for (let i = 0; i <= 100; i++) {
      this.data.pushData(`Hello ${i}`);
    }
  }

  build() {
    List({ space: 3 }) {
      LazyForEach(this.data, (item: string) => {
        ListItem() {
          Row() {
            Text(item).fontSize(50)
              .onAppear(() => {
                console.info(`appear: ${item}`);
              });
          }.margin({ left: 10, right: 10 });
        };
      }, (item: string) => item);
    }.cachedCount(5);
  }
}
```
