## 场景介绍

运动记录是运动健康类应用中高频使用场景之一，如用户在跑步运动时，为了避免运动过程中因误触等操作意外终止，需要在结束运动交互过程中添加长按处理逻辑。

本示例使用[Path](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-drawing-components-path)组件绘制结束按钮的长按过程动画，使用[animateTo](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-uicontext#animateto)动画接口实现暂停运动、结束运动时的按钮过渡动画。

## 实现思路

1. 定义函数getPathCommands，根据进度变量progress计算闭合曲线路径绘制过程中的端点坐标。

   ```
   getPathCommands(progress: number): string {
     // Get the abscissa of the endpoint
     let abscissa: number = this.getAbscissa(progress / Constants.PERCENT_RATE);
     // Get the square of half the distance between the two ends
     let distanceSquare: number = this.getDistanceSquare(abscissa);
     if (distanceSquare >= Constants.ZERO) {
       // Root extraction of half the distance between the two ends
       let distance: number = Constants.ZERO;
       if (distanceSquare === Constants.ZERO) {
         distance = Constants.ZERO;
       } else {
         distance = Math.sqrt(distanceSquare);
       }
       // Calculate the ordinate of the first endpoint
       let firstOrdinate: number = Constants.RADIUS - distance;
       // Calculate the ordinate of the second endpoint
       let secondOrdinate: number = Constants.RADIUS + distance;
       return this.formatPathCommands(firstOrdinate, secondOrdinate, abscissa, Constants.RADIUS);
     }
     return '';
   }
   ```
2. 生成曲线绘制命令并传入[Path](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-drawing-components-path)组件的commands属性中，实现UI层面的进度动画效果。

   ```
   // 生成最终曲线绘制命令
   this.pathCommands = this.getPathCommands(this.progress);
   Path()
     // 传入最终绘制命令
     .commands(this.pathCommands)
     // 取消Path组件交互效果
     .hitTestBehavior(HitTestMode.None);
   ```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
