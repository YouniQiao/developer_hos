## 问题现象

如何解决使用安全审计权限时会报9568289，提示error: install failed due to grant request permissions failed错误的问题？

## 背景知识

* [9568289 权限请求失败导致安装失败](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/bm-tool#section9568289-权限请求失败导致安装失败)：权限请求失败导致安装失败。
* [安全审计](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/devicesecurity-audit)：提供统一的安全审计数据订阅与取消订阅接口，应用可以获取设备上的安全审计数据，以支撑审计相关业务。
* [ohos.permission.QUERY\_AUDIT\_EVENT](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/permissions-for-enterprise-apps#ohospermissionquery_audit_event)：允许应用查询安全审计事件。

## 解决方案

* 确认当前应用类型为[MDM应用](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/mdm-kit)还是企业普通应用，非企业应用不可以使用该权限。'ohos.permission.QUERY\_AUDIT\_EVENT'权限在API 12-13时仅面向MDM应用开放；从API 14开始，开放范围变更为企业普通应用。
* 请确认当前设备类型为2in1设备，当前安全审计[支持的设备](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/devicesecurity-introduction#section10258144202913)仅为PC/2in1。

## 分析结论

未申请ohos.permission.QUERY\_AUDIT\_EVENT权限的情况下使用了安全审计能力。

## 修改建议

企业类应用应当先申请ohos.permission.QUERY\_AUDIT\_EVENT权限，然后在PC/2in1设备上使用安全审计能力。

## 常见FAQ

Q：如何判断当前应用是否为系统应用？

A：可以根据以下步骤进行判断：

1. 连接设备。
2. 进入终端Shell：hdc shell。
3. 在终端Shell中执行以下命令打印日志：bm dump -n com.example.myapplication | grep isSystemApp。
4. isSystemApp字段返回值为true，则说明当前应用已经设置为系统应用："isSystemApp": true。

Q：安全审计能力能监听到剪切板复制粘贴，账号登录登出操作吗？

A：目前单客户端/多客户端都支持监听剪切板复制粘贴，账号登录登出操作，更多支持审计相关业务可以参见文档：[单客户端订阅场景](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/devicesecurity-audit-subscribe-arkts-suevent)，[多客户端订阅场景](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/devicesecurity-audit-subscribe-arkts)。

Q：通过securityAudit.on('auditEventOccur', auditEventInfo, callback)；订阅了应用审计事件，更新xx应用callback没有被触发。

A：使用MDM应用安装事件回调函数[EnterpriseAdminExtensionAbility.onBundleAdded](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-enterpriseadminextensionability#enterpriseadminextensionabilityonbundleadded)订阅应用更新事件。
