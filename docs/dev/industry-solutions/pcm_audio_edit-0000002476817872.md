## 场景介绍

PCM音频编辑是影音娱乐类应用的高频使用场景之一，如用户在进行音频录制时，发现某个时间段音频录制效果不好，需对音频进行覆盖或裁剪操作。

本示例使用[AudioCapturer](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-audio-audiocapturer)实现PCM音频录制，使用[AudioRenderer](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-audio-audiorenderer)实现PCM音频播放，对PCM音频编辑时，可通过覆盖或裁剪操作对PCM音频文件进行修改，以及通过[Canvas](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-components-canvas-canvas)实现音频波形的实时绘制。

## 实现思路

1. 先通过[AudioCapturer](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-audio-audiocapturer)录制某段音频，并将音频保存在file目录下，音频保存目录可自定义。
2. 以删除某段音频为例，根据传入的开始时间和结束时间，同时根据音频采样率、采样格式、采集通道数计算出对应音频文件需要裁剪的开始位置和结束位置，再根据开始位置和结束位置对音频文件进行裁剪，裁剪完成之后，通知录制时间发生了变化。

   ```
   // AudioRecorder.ets
   public deleteAudio(startTime: number, endTime: number) {
     this.unableRedoAudio();

     const startPos = TimeUtil.millisecondsToFileSize(startTime, this.audioSteamInfo);
     const endPos = TimeUtil.millisecondsToFileSize(endTime, this.audioSteamInfo);
     const newFilePath = fileMgr.cutFile(this.audioFileInfo.filePath, startPos, endPos, this.defaultTempFileDir);
     if (newFilePath) {
       this.historyFilePaths.push(newFilePath);
       this.historyFileIndex += 1;
       this.audioFileInfo.filePath = newFilePath;
       this.notify('recordTime', this.recordTime());
     }
   }
   ```

   ```
   // FileManager.ets
   public cutFile(srcFilePath: string, startPos: number, endPos: number, cutFileSaveDir?: string): string | undefined {
     try {
       const index = srcFilePath.lastIndexOf('/');
       const fileDir = cutFileSaveDir ? cutFileSaveDir : srcFilePath.slice(0, index);
       if (!fileIo.accessSync(fileDir)) {
         fileIo.mkdirSync(fileDir);
       }

       const dstFilePath = fileDir + '/' + util.generateRandomUUID();
       const dstFile = fileIo.openSync(dstFilePath, fileIo.OpenMode.WRITE_ONLY | fileIo.OpenMode.CREATE);

       const srcFile = fileIo.openSync(srcFilePath, fileIo.OpenMode.READ_ONLY);
       const srcFileSize = fileIo.statSync(srcFilePath).size;

       const lastPos = endPos > srcFileSize ? srcFileSize : endPos;
       if (startPos > 0) {
         const buffer = new ArrayBuffer(startPos);
         fileIo.readSync(srcFile.fd, buffer, { offset: 0, length: startPos });
         fileIo.writeSync(dstFile.fd, buffer, { offset: 0, length: startPos });
       }
       if (lastPos !== srcFileSize) {
         const buffer = new ArrayBuffer(srcFileSize - lastPos + 1);
         fileIo.readSync(srcFile.fd, buffer, { offset: lastPos, length: srcFileSize - lastPos + 1 });
         fileIo.writeSync(dstFile.fd, buffer, { offset: startPos, length: srcFileSize - lastPos + 1 });
       }

       fileIo.closeSync(srcFile.fd);
       fileIo.closeSync(dstFile.fd);
       return dstFilePath;
     } catch (error) {
       logger.error(`cut file fail, code: ${error.code}, message: ${error.message}`);
     }

     return undefined;
   }
   ```
3. 计算波形绘制信息，首先需根据当前指示线的编号，计算出开始绘制波形的编号和需要绘制波形的总数量，再根据分割区域将其拆分成某几段区域，通过[Canvas](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-components-canvas-canvas)进行逐一绘制。

   ```
   // WaveSampler.ets
   public waveDrawInfos(preciseOffset: number, maxHalfWaveCount: number): WaveDrawInfo[] {
     const offsetCount = preciseOffset / Wave.WAVE_SPACE; // 发生偏移的数量，即指示器右侧部分数量，其数量包含指示线

     // 计算指示线位置绘制的编号，小于0，说明滑动已超出范围，需做规整
     this.centerIndex = this.waveDataList.length - offsetCount;
     if (this.centerIndex < 0) {
       return this.splitWaveDrawInfo({ start: 0, center: 0, count: maxHalfWaveCount });
     }

     // 计算开始位置绘制的编号，小于0，说明左侧绘制位置达不到最大绘制个数，需做规整
     let startIndex = this.centerIndex - maxHalfWaveCount;
     if (startIndex < 0) {
       startIndex = 0;
     }

     // 计算绘制波形的总个数，超出数据长度，需做规整
     let rightWaveCount = this.waveDataList.length - this.centerIndex;
     rightWaveCount = rightWaveCount > maxHalfWaveCount ? maxHalfWaveCount : rightWaveCount;
     let count = this.centerIndex - startIndex + rightWaveCount;
     if (count > this.waveDataList.length) {
       count = this.waveDataList.length;
     }

     return this.splitWaveDrawInfo({ start: startIndex, center: this.centerIndex, count: count + 1 });
   }
   ```
4. 对波形进行缩放操作时，根据波形时间间隔对分割区域和波形数据源进行重新映射，需要进行重新映射的操作还包括撤销、还原和保存。

   ```
   // WaveSampler.ets
   public updateWaveDataByInterval(waveInterval: number) {
     this.waveInterval = waveInterval;
     this.remapDivideArea();
     this.remapWaveDataSource();
   }
   ```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
