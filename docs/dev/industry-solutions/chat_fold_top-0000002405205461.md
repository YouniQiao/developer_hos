## 场景介绍

聊天列表置顶及折叠是社交通讯类应用的典型场景之一，如用户需要置顶或者取消当前聊天对象，以及置顶聊天过多时进行折叠。

本示例基于[List](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-container-list)实现聊天列表，使用[@State装饰器](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-state)控制置顶聊天的折叠和展开状态，使用[swipeAction](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-container-listitem#swipeaction9)设置左划按钮，点击该按钮可置顶或取消置顶消息。

## 实现思路

1. 使用Set类型的pinnedSessions存储置顶会话的接收者ID并控制置顶会话的显示顺序，sessionList存储所有会话数据。
2. 置顶会话显示在会话列表顶部，非置顶会话显示在置顶会话下方，置顶会话支持折叠功能。
3. 滑动会话项时显示置顶/取消置顶按钮，点击置顶/取消置顶按钮时更新pinnedSessions并重新排序会话列表。

```
// 加载置顶会话
List() {
    // 先显示置顶的会话
    ForEach(Array.from(this.pinnedSessions).reverse().map(receiver => {
      return this.sessionList.find(session => session.receiver === receiver);
    }).filter(Boolean), (item: SessionItem, index: number) => {
    // 列表渲染
  });
}
.visibility(this.isCollapsed?Visibility.Visible:Visibility.None) // 控制显示隐藏

// 加载非置顶会话
List() {
  // 显示非置顶的会话
  ForEach(this.sessionList.filter(session => !this.pinnedSessions.has(session.receiver)),
    (item: SessionItem, index: number) => {
    // 列表渲染
    });
}

// 置顶/取消置顶逻辑
@Builder
itemEnd(item: SessionItem) {
  Row() {
    Column() {
      Image(item.isPinned ? $r('app.media.unpin') : $r('app.media.pin'));
    }
    .onClick(() => {
      // 点击切换置顶与不置顶的状态
      if (item.isPinned) {
        this.pinnedSessions.delete(item.receiver);
        item.isPinned = false;
      } else {
        this.pinnedSessions.add(item.receiver);
        item.isPinned = true;
      }
      this.sortSessionList();
    });
  }
}

// 会话列表排序
private sortSessionList(): void {
  this.sessionList.sort((a, b) => {
    if (this.pinnedSessions.has(a.receiver) && !this.pinnedSessions.has(b.receiver)) {
      return -1;
    } else if (!this.pinnedSessions.has(a.receiver) && this.pinnedSessions.has(b.receiver)) {
      return 1;
    } else {
      return 0;
    }
  });
}
```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
