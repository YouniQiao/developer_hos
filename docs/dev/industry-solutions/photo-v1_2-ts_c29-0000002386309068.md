## 问题现象

应用在使用时页面卡死闪退。

## 背景知识

* JsCrash异常根据不同的异常场景，在Reason字段进行了分类，分为Error、TypeError、SyntaxError、ReferenceError、RangeError等错误类型。参考文档[JS Crash（进程崩溃）检测](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/jscrash-guidelines)。
* JsCrash日志规格说明可以参考[日志规格](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/jscrash-guidelines#日志规格)。
* 用户在使用应用时，如果出现点击无反应或应用无响应等情况，并且持续时间超过一定限制，就会被定义为应用无响应，详情参考[AppFreeze（应用冻屏）检测](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/appfreeze-guidelines)。
* AppFreeze日志规格说明可以参考[日志规格](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/appfreeze-guidelines#日志规格)。

## 问题定位

### 场景一

1. 确认闪退类型，闪退问题首先排查是否出现JsCrash或者CppCrash，查看faultlogger目录，是否生成了形如jscrash-应用包名或者cppcrash-应用包名的故障日志。
2. 确认闪退原因。

   ```
   Reason:Error
   Error name:Error
   Error message:Init error. The WebviewController must be associated with a Web component
   Error code 17100001
   ```

   从故障日志中可以获取到故障原因和异常信息。

   * 根据日志中的Error Code：17100001可以在[WebView错误码](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/errorcode-webview)中查询到相关解释。
   * 异常信息是Error message:Init error. The WebviewController must be associated with a Web component，WebviewController还没有和具体的Web组件关联，无法进行相应的操作。

## 分析结论

### 场景一

WebviewController还没有和具体的Web组件关联，无法进行相应的操作。

### 场景二

当前运行任务执行时间过长，导致主线程阻塞，系统主动杀掉无响应应用。

### 场景三

应用创建了大量数据库连接，系统出现文件描述符资源耗尽错误errno：24，怀疑存在数据库连接关闭异常。最终导致打开新的数据库（chat\_customer.db）链接失败阻塞。

## 修改建议

### 场景一

建议将相应操作放在[onControllerAttached](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-basic-components-web-events#oncontrollerattached10)(内有示例代码)中，此回调当Controller成功绑定到Web组件时触发。

### 场景二

优化网络请求耗时，参考[网络诊断：Network分析](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-profiler-network)，使用Profiler提供Network模板，分析请求各阶段耗时，进行对应优化。

如果优化困难，可以考虑提前将网络请求数据缓存，下次直接加载缓存数据，再通过发送网络请求二次刷新数据。

### 场景三

怀疑存在数据库连接池泄露，建议结合代码和场景对如下原因进行排查。

* 数据库连接未正确关闭。
* 资源管理不当。
* 连接池配置过大。
* 数据库连接超时设置不合理。
