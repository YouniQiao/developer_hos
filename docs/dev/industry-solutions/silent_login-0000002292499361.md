## 场景介绍

静默登录是各类应用中的高频使用场景之一，如初次登录后希望后续不再显示登录页面。

本示例通过[关系型数据库](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/data-persistence-by-rdb-store)实现用户信息存储，[PersistentStorage](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-persiststorage)实现静默登录状态的持久化存储。

## 实现思路

* 登录时检查数据库中用户信息是否已存在，以决定是更新还是插入账户信息。
* [PersistentStorage](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-persiststorage)持久化存储静默登录项，点击[Checkbox](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-checkbox)更新状态。

```
Button($r('app.string.login'))
  .onClick(async () => {
    let index: number = this.accountList.findIndex(v => v.userName === this.userName);
    if (index > -1) {
      // 数据库存在用户信息，检查密码是否正确
    } else {
      // 数据库中不存在用户信息，进行注册
    }
  })
// 在EntryAbility.ets中进行静默登录选项初始化
PersistentStorage.persistProp('isRemembered', false);
// 登录页进行状态更新
@StorageLink('isRemembered') isRemembered: boolean = false;
Checkbox({ name: 'checkbox', group: 'checkboxGroup' })
  .onChange(() => {
    this.isRemembered = !this.isRemembered;
  })
```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
