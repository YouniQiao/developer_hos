## 问题现象

在多设备工程中，如何开发服务卡片。

## 背景知识

* 卡片开发服务[Form Kit](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/form-kit)提供了一种在桌面、锁屏等系统应用上嵌入显示应用信息的开发框架和API，可以将应用内用户关注的重要信息或常用操作抽取到服务卡片（简称“卡片”）上，通过将卡片添加到桌面、锁屏等系统应用上，以达到信息展示、服务直达的便捷体验效果。
* ArkTS卡片有两种创建卡片包的方式：[共包方式创建卡片](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-ui-widget-creation#方式一共包方式创建卡片)、[独立包方式创建卡片](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-ui-widget-creation#方式二独立包方式创建卡片)，其中独立包方式创建卡片从API 20开始支持。
* 静态共享包[HAR](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/har-package)可以包含代码、C++库、资源和配置文件。通过[开发静态共享包](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-har)HAR可以实现多个模块或多个工程共享ArkUI组件、资源等相关代码。
* 为了实现“一次开发，多端部署”的开发理念，可以根据[架构设计](https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-multi-device-overview#section73681810173312)中介绍的三层架构工程，以便于理清业务逻辑，减少代码耦合，实现灵活部署和模块级复用。

## 解决方案

参考[多设备工程部署](https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-multi-device-ide)，在项目product目录中创建多个设备entry类型的入口包，随后根据不同的开发场景需求完成卡片的开发。

* 场景一：不同设备开发不同的卡片，各卡片间没有复用布局或组件。

  参考[ArkTS卡片开发](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-ui)，在各个设备entry类型的入口包中创建卡片、配置卡片文件、管理卡片生命周期并开发卡片UI布局和互动刷新等。
* 场景二：不同设备开发相同的卡片，或不同的卡片间有复用组件或布局。
  1. 参考[开发静态共享包](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-har)开发并构建HAR包，ArkTS卡片中仅支持导入静态共享包。该HAR包的位置根据实际业务存放在公共能力层或基础特性层。
  2. 参考[创建ArkTS卡片](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-ui-widget-creation)，在项目product目录中每个设备entry类型的入口包中分别创建卡片。
  3. 根据创建的HAR包名称和路径，在创建卡片的entry产品包的oh-package.json5的dependencies字段中，配置HAR静态共享包依赖。示例代码如下：

     ```
     "dependencies": {
       "@ohos/adaptivelayout": "file:../../features/adaptiveLayout",
       "@ohos/responsivelayout": "file:../../features/responsiveLayout",
       "@ohos/common": "file:../../common",
       "library": "file:../../library"
     },
     ```
  4. 参考[引用HAR的ArkUI组件](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/har-package#引用har的arkui组件)，在卡片页面中复用HAR包公共组件。示例代码如下：

     ```
     // 导入静态共享包公共组件
     import { MainPage } from 'library';

     @Entry
     @Component
     struct WidgetCard {
       readonly title: string = 'Default Hello World';
       readonly actionType: string = 'router';
       readonly abilityName: string = 'DefaultAbility';
       readonly message: string = 'add detail';
       readonly fullWidthPercent: string = '100%';
       readonly fullHeightPercent: string = '100%';

       build() {
         Row() {
           Column() {
             Text(this.title)
               .fontSize($r('app.float.font_size'))
               .fontWeight(FontWeight.Medium)
               .fontColor($r('sys.color.font'));

             // 复用静态共享包公共组件
             MainPage();
           }
           .width(this.fullWidthPercent);
         }
         .height(this.fullHeightPercent)
         .backgroundColor($r('sys.color.comp_background_primary'))
         .onClick(() => {
           postCardAction(this, {
             action: this.actionType,
             abilityName: this.abilityName,
             params: {
               message: this.message
             }
           });
         });
       }
     }
     ```

## 常见FAQ

Q：独立卡片如何设置可运行设备，以实现独立卡片包在不同设备上的复用？

A：以[独立包方式创建卡片](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-ui-widget-creation#方式二独立包方式创建卡片)后，可以根据需要修改独立卡片包的module.json5中的deviceTypes字段，以标识可使用卡片的设备类型。但由于独立卡片包与应用模块为一对一关联（独立卡片包需要配置formExtensionModule字段以关联应用模块，对应应用模块需要配置formWidgetModule字段以关联卡片包），所以独立卡片包不支持模块级复用。
