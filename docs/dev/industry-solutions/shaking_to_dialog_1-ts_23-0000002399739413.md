## 问题现象

1. 开发者使用应用市场的版本测试功能，但是无法确保同一批用户参与连续测试。比如说用户提了一些问题，在新的版本里面解决了，再次测试的时候，这个用户大概率抢不到新版本的测试名额，这样开发者就找不到提出这个问题的用户来验证闭环问题。
2. 所以希望应用市场能提供一个方法，确保在两次测试版本期间能指定部分核心用户能参与体验和测试验证。

## 背景知识

华为应用市场提供了两种常见的测试类型，帮助开发者在应用正式版本发布之前，挑选特定的用户群组来测试HarmonyOS应用或元服务，或向AppGallery用户公开发布您的测试版本。分为邀请测试和公开测试，对比如下：

| 测试阶段 | 邀请测试 | 公开测试 |
| --- | --- | --- |
| 面向对象 | 可邀请友好、信任的小范围用户 | 面向全网所有用户公开招募测试 |
| 邀请人数 | 上限10000人 | 下载安装次数上限1000万次 |
| 支持同时测试的版本数 | 100个 | 1个 |
| 发布AppGallery测试专区 | 必须 | 可选 |
| 是否支持分享链接 | 支持 | 支持 |

## 解决方案

如果希望核心用户每次能参与到后续版本的测试中，提供如下方案供参考：

1. 邀请测试：
   * 适用于：早期版本，问题较多，只供信任的用户体验和反馈问题，测试用户数较少（小于10000）。
   * 前提条件：邀请测试需要提前收集需要参与测试用户的账号，**用户只需参与了某个版本的邀请测试，后续的邀请测试版本都可以自动参与。**
   * 发布方式：
     + 创建测试用户/用户组。
     + 添加测试用户，可以选择测试账号添加或者邀请码添加。
     + 创建邀请测试版本，勾选测试用户，默认发布到测试专区，建议勾选生成分享链接。

       ![](./img/cbe48fd6.png "点击放大")
     + 提交邀请测试版本，等待审核通过。
   * 参与方式：
     + 邀请短信参与：新版本邀请测试上架后，存量用户都会收到应用市场发送的邀请测试短信，用户点击短信中的测试链接即可自动跳转到应用测试版本页面下载安装应用。
     + 测试专区参与：用户也可以直接进入应用市场尝鲜专区找到应用下载安装。
     + 邀请链接参与：创建版本时勾选了生成分享链接。可以通过分享“邀请链接”或“邀请链接+邀请码”方式发送给需要参与测试的用户。

2. 公开测试-发布到测试专区：
   * 适用于：后期版本，问题较少，功能较完善，希望获取更多用户群体参与测试反馈问题完善版本，测试用户数较大（大于10000）。
   * 前提条件：公开测试不需要提前收集测试用户账号，用户主动参与测试。
   * 发布方式：
     + 创建公开测试版本，设置下载安装次数足够大（当前最多支持1000万）。**务必勾选发布到测试专区**。建议勾选生成分享链接。![](./img/ed9d71d7.png "点击放大")
     + 提交公开测试版本，等待审核通过。
   * 参与方式：
     + 测试专区参与：用户可以直接进入应用市场尝鲜专区找到应用下载安装，因为数量足够大，用户一定能参与测试。
     + 邀请链接参与：创建版本时如果勾选了生成分享链接，通过“邀请链接”方式发送给需要参与测试的用户。

3. 公开测试-不发布到测试专区：
   * 发布方式：
     + 创建公开测试版本，按需设置下载安装次数。**不勾选发布到测试专区。务必勾选生成分享链接。**

       ![](./img/cbe056ac.png "点击放大")
     + 提交公开测试版本，等待审核通过。
   * 参与方式：
     + 邀请链接参与：通过分享“邀请链接”方式发送给需要参与测试的核心用户提前体验。
     + 更新版本：继续更新公开测试版本，勾选发布到测试专区，确保普通用户可以在尝鲜专区下载使用。

## 常见FAQ

Q：邀请测试是否需要审核，审核时效是多久？

A：邀请测试版本提交后，系统会进行基础的安全，隐私，兼容性审核，最多2小时内完成。如无基本问题，应用一般可在半小时内上架，如超过2小时未上架，应用将进入人工审核环节，需要等待人工审核结果。

Q：公开测试是否需要审核，审核时效是多久？

A：公开测试版本等同于全网发布版本，需要遵循全网正式上架版本的审核流程。但公开测试版本上架后可以直接转全网版本，不需要再次人工审核。

Q：应用的基数用户群体较大，想按比例控制部分用户参与测试，如何实现？

A：开发者可以选择公开测试的方式发布应用，在发布时限制合理的下载安装次数，每次发布有效期为30天，到期需要再次发布，发布上架后用户可在应用尝鲜区参与公测。
