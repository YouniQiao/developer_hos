## 场景介绍

协同办公日程管理是企业办公类应用的高频使用场景之一，当前的日程管理系统，除支持个人日程查看外，还可以查看同事及团队日程、向同事分享个人日程，有效解决了团队协同场景中信息共享效率低、成员时间协调成本高的问题。

本方案深度结合系统能力，为用户带来高效、便捷、安全的协同办公体验，核心优势如下：

1. **可靠双端同步**：基于[Calendar Kit](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/calendar-kit)实现应用日程与系统日历无损同步，确保日程提醒零遗漏，无需跨应用反复切换；
2. **极简协作配置**：通过弹窗容器+开关组件实现团队/同事日程的显示控制，操作路径短，配置灵活可控，页面加载与渲染性能优异；
3. **极简分享体验**：基于[Share Kit](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/knock-share-between-phones-overview)与[Deep Linking](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/deep-linking-startup)实现碰一碰近场分享与一键导入，无需扫码、复制链接，大幅降低协同成本。

典型场景如下：

| 编号 | 场景名称 | 场景描述 | 实现方案 |
| --- | --- | --- | --- |

## 实现思路

1. 创建日程及同步到系统日历：用户在协同办公日程管理模块填写日程全量信息（主题、时间、提醒等）并提交，系统校验参数后保存数据，通过[Calendar Kit](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/calendar-kit)同步至系统日历，最终反馈创建及同步成功。

   ![](./img/a17ec4b5.png "点击放大")

   **关键点说明：**

   * **系统日历双向同步**：使用[Calendar Kit](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/calendar-kit)实现系统日历增删改查能力，实现日程重复规则、时区、提醒规则的无损转换，适配跨时区、重复周期等场景。
   * **权限规范配置**：在entry模块的module.json5文件中申请系统日历读写权限。
   * **参数合法性校验**：在写入系统日历前，对日程主题、时间、提醒等必填参数进行合法性校验，确保数据格式符合系统日历要求，避免写入失败。
   * **简洁同步流程**：直接通过[Calendar Kit](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/calendar-kit)将日程写入系统日历，流程简洁高效，用户提交后即可完成同步。

   **关键代码片段：**

   * 获取应用内的创建的日程信息ScheduleInfo数据对象并转为手机日历的[Event](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-calendarmanager#event)类数据。

     ```
     // 日程信息
      export interface ScheduleInfo {
        // 基础信息
        id?: string, // 日程唯一ID
        eventId?: number, // 可能关联的系统事件ID
        title: string, // 日程标题（必填）
        desc?: string, // 日程描述
        // 时间、地点相关
        isAllDay?: boolean, // 是否全天事件
        startTime: string, // 开始时间
        endTime: string, // 结束时间
        location?: string, // 地点描述
        latitude?: number, // 纬度
        longitude?: number, // 经度
        // 重复规则
        recurrenceRule?: number, // 重复规则ID
        reminderTime?: number[], // 提前提醒时间（分钟）
        conflicting?: boolean, // 是否与其他日程冲突
        // 界面相关
        color?: string, // 日程显示颜色
        noShow?: boolean, // 是否隐藏
        // 分类标识
        type?: string, // 日程类型（个人/团队/同事）
      }
     ```
   * 将日程添加至日历：首先获取日历权限，然后使用[addEvent](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-calendarmanager#addevent)函数将Event类数据添加至日历中，添加成功后再添加到本地存储中。

     ```
     save() {
       // 验证标题
       if (!this.schedule.title?.trim()) {
         this.getUIContext().getPromptAction().showToast({ message: $r('app.string.schedule_title_empty_tip') });
         this.getUIContext().getFocusController().requestFocus('titleID');
         return;
       }

       // 此处模拟区分，团队日程不进入系统日历中，实际业务中可以根据日程信息中的类型进行判断
       if (this.schedule.type === Constants.GROUP_SCHEDULE) {
         RequestProxy.saveOne(this.schedule).then(() => {
           ScheduleUtils.updateEvent(this.schedule);
         });
         this.getUIContext().getPromptAction().showToast({ message: $r('app.string.new_succ_tip') });
         RouterModule.closeDialog({});
       } else {
         // 个人日程：写入系统日历+本地保存
         ScheduleUtils.addEvent(this.schedule).then(eventId => {
           if (!eventId) {
             this.getUIContext().getPromptAction().showToast({ message: '创建系统日程失败' });
           }
           this.schedule.eventId = eventId; // 保存系统日历返回的ID
           RequestProxy.saveOne(this.schedule).then(() => {
             ScheduleUtils.updateEvent(this.schedule);
           });
           this.getUIContext().getPromptAction().showToast({ message: $r('app.string.new_succ_tip') });
           RouterModule.closeDialog({});
         });
       }
     }
     ```
2. 查看同事/团队日程：通过日程设置组件SettingSchedule来选择需要查看的同事/日程，通过日历视图组件ScheduleView来根据日程属性来渲染不同颜色进行区别展示。

   ![](./img/fc2bc007.png "点击放大")

   **关键点说明：**

   * **弹窗交互模式**：采用NavDestinationMode.DIALOG实现SettingSchedule底部弹窗容器，用户无需跳转页面，在首页即可完成配置，操作路径便捷，符合交互规范。
   * **开关组件精细化控制**：使用CheckBox开关组件，按“团队日程”、“同事日程”两大分组展示列表，用户可独立开启/关闭单个团队或同事的日程显示，支持精细化配置，灵活适配不同使用需求。
   * **视图实时刷新**：保存配置后，首页日历视图实时监听并刷新，按最新配置显示/隐藏对应日程，无需重启应用，交互体验流畅连贯。

   **关键代码片段：**

   * 查询可查看同事和团队。

     ```
     private initSelectedItems() {
       // 从DataManager读取当前可见的团队
       DataManager.TEAM_DATA_MAP.forEach((value, key) => {
         // 检查该团队是否有可见日程

       DataManager.OTHER_DATA_MAP.forEach((value, key) => {
         const hasVisible = value.some(schedule => !schedule.noShow);
         if (hasVisible) {
           this.others.push(key.userName);
         }
       });
     }

     ```
     save() {
       try {

             value.forEach((schedule) => {

3. 个人日程分享：用户分享个人日程，构建[DeepLinking](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/deep-linking-startup)链接，再调用[ShareKit](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/share-introduction)组件通过[碰一碰](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/knock-share-between-phones-overview)分享日程数据。

   ![](./img/eb0db379.png "点击放大")

   **关键点说明：**

   * **碰一碰分享**：使用[ShareKit](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/share-introduction)实现标准化近场分享能力，无需扫码、复制链接，提高面对面协作场景效率。
   * **DeepLinking路由配置**：在module.json5中配置链接规则，确保接收方点击链接可自动跳转至应用导入页面。
   * **Base64标准化编码**：对日程数据进行标准化编码，确保日程数据在碰一碰传输过程中格式统一，避免特殊字符导致的解析异常，简化传输流程，提升跨设备兼容性。
   * **事件驱动自动解析**：基于EventHub实现分享数据的跨页面通知，接收方自动解析[DeepLinking](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/deep-linking-startup)链接与Base64编码数据，弹出导入弹窗，实现一键导入同步。

   **关键代码片段：**

   * 构建DeepLink链接，需要提前配置[module.json5](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/deep-linking-startup#配置modulejson5文件)文件，详细配置参考[配置module.json5文件](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/deep-linking-startup#配置modulejson5文件)。

     ```
     static shareSchedule(schedule: ScheduleInfo, context: UIContext): string {
       try {
         // 1.验证是否可以分享
         if (!ShareUtil.canShare(schedule)) {
           context.getPromptAction().showToast({
             message: '只有个人日程可以分享',
             duration: 2000
           });
           return '';
         }

         // 2.获取当前用户信息，这里仅作示例，实际根据业务逻辑设置分享者信息
         const userInfo = AccountUtil.getUserInfo();
         const userName = userInfo?.nickname || '赵六';

         // 3.构建分享数据
         const shareScheduleData: ShareScheduleData = ShareUtil.creatShareScheduleData(schedule, userName);

         // 4.构建分享内容
         const shareLink = buildShareText(shareScheduleData);
         return shareLink;
       } catch (error) {
         Logger.error(TAG, 'Failed to share schedule', error);
         context.getPromptAction().showToast({
           message: '分享失败，请重试',
           duration: 2000
         });
       }
       return '';
     }
     ```
   * 调用ShareKit的share方法实现碰一碰分享。

     ```
     // 碰一碰分享回调事件，此处使用DeepLinking作为示例，如使用AppLinking需要应用在AGC平台配置请参考官网指南
     // 在示例中需要在打开分享弹框后，再去碰一碰才能生效
     private shareCallback = (sharableTarget: harmonyShare.SharableTarget) => {
       // 1.构建链接，链接的scheme和host必须与接收方配置的完全一致
       // 此处以DeepLinking作为示例，例如链接：link://shareSchedule?data=分享数据，在module.json5中需要配置scheme和host
       const linking: string = ShareUtil.shareSchedule(this.schedule, this.getUIContext());

       // 2.构造要分享的数据，此处仅作示例，实际可替换为文件、图片或自定义的Want参数
       const shareData: systemShare.SharedData = new systemShare.SharedData({
         utd: utd.UniformDataType.HYPERLINK,
         content: linking,
         title: `日程分享`, // 卡片标题
         description: `日程: ${this.schedule.title}`, // 卡片描述
         // 如果有预览图，可设置thumbnailUri或thumbnail字段
       });

       // 3.调用接口，执行分享
       sharableTarget.share(shareData);
     };
     ```
   * 获取碰一碰分享数据，通过EventHub发送事件通知页面。

     ```
     private handleKnockShareData(want: Want): void {
       // 获取分享的内容
       // 根据发送方构造的utd类型，数据可能在不同字段中
       const link = want.uri;
       if (link && link.startsWith('link://')) {
         // 根据链接解析参数，跳转到应用内对应页面
         this.isFromKnockShare = true;
         this.knockShareData = link;

         // 将数据存储到AppStorage，全局UI都可以访问
         AppStorage.setOrCreate('isFromKnockShare', this.isFromKnockShare);
         AppStorage.setOrCreate('knockShareData', this.knockShareData);

         // 通过EventHub发送事件通知页面
         this.context.eventHub.emit('knockShareDataUpdated', {
           isFrom: true,
           data: link
         });
         console.info('碰一碰数据已存储到AppStorage:', this.knockShareData);
       }
     }
     ```
   * 解析碰一碰分享数据。

     ```
     private parseShareDataToObject(link: string): ShareScheduleData | null {
       try {
         const queryStartIndex = link.indexOf('?');
         if (queryStartIndex === -1) {
           console.info(`解析失败：链接格式不正确 ${link}`);
           return null;
         }

         const shareScheduleData = extractScheduleInfoFromLink(link) as ShareScheduleData;

         // 验证必要字段是否存在
         if (!shareScheduleData.title || !shareScheduleData.startTime) {
           console.info('解析失败：缺少必要字段');
           return null;
         }

         console.info(`解析成功：${shareScheduleData.title}`);
         return shareScheduleData;
       } catch (e) {
         console.error('解析链接失败:', e);
         return null;
       }
     }

     export function extractScheduleInfoFromLink(shareLink: string): ShareScheduleData | null {
       try {
         // 假设链接格式为：link://shareSchedule?data=分享数据
         let encodedData = '';

         // 尝试从URL参数中提取
         if (shareLink.includes('?data=')) {
           encodedData = shareLink.split('?data=')[1].split('&')[0];
         }
         // 直接作为编码字符串
         else {
           encodedData = shareLink;
         }

         return decodeScheduleInfo(encodedData);
       } catch (error) {
         console.error('从链接提取日程信息失败:', error);
         return null;
       }
     }
     ```
