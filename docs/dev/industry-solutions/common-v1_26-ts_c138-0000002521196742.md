## 问题现象

* **问题一：**

  开发者在使用云调试安装APP失败，报“BusinessError 17700101: Bundle manager service is excepted.”，如下图：

  ![](./img/81182aea.png "点击放大")
* **问题二：**

  应用安装报“应用不匹配当前系统版本，请选择其他系统版本设备”。

  ![](./img/4aeb4307.png)
* **问题三：**

  应用包上传报“请您上传hap/app格式的文件！”。

  ![](./img/c0f64023.png)
* **问题四：**

  云调试安装release包提示安装失败，报“error: failed to install bundle. code:9568297 error: install failed due to older sdk version in the device”。
* **问题五：**

  云调试上传软件包耗时过长，浪费免费体验时长。
* **问题六：**

  云调试安装应用提示：该应用类型不适用于所选设备。应用是2in1，选择的调试真机为matebook pro，但还是报应用类型错误。

## 解决方案

* **问题一解决方案：**

  可以从以下方面进行排查：

  1. 检查文件完整性，如果是在线文件请确认是否下载完成。
  2. 检查文件名是否带有特殊符号（如“~”等字符）导致解压失败，需修改相关文件名。如果以上方法还不能解决您的问题，可以继续参考官网[17700101 包管理服务异常](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/errorcode-bundle#section17700101-包管理服务异常)排查解决。
* **问题二解决方案：**

  此类问题是因为机型的系统和应用不兼容，以该问题为例：

  1. 查看工程目录build-profile.json5文件的compatibleSdkVersion字段，标识应用/元服务运行所需兼容的最低SDK版本为4.0.0（10）。

     ![](./img/eb76776b.png "点击放大")
  2. 开发者在云调试选择的机型API为9，因此该机型的系统和应用不兼容。选择API为10或者10以上的机型即可。

     ![](./img/e7bb034a.png)
* **问题三解决方案：**

  HarmonyOS NEXT的设备只能上传HAP和APP包。

  1. 如果要上传apk，机型选择页面，筛选条件-系统版本选择以下三个类型。

     ![](./img/c26974b6.png "点击放大")
  2. 选择设备进入，应用上传入口显示支持apk。

     ![](./img/5f3cbc66.png "点击放大")
* **问题四解决方案：**

  该问题原因是云调试设备SDK版本较低，可以通过筛选云调试设备的系统版本来获取相匹配的设备。在云调试平台选择筛选条件，选择与安装包匹配的设备调试：

  ![](./img/4e454c23.png "点击放大")
* **问题五解决方案：**
  1. 在云调试-我的信息-应用中，提前上传需要调试的应用。

     ![](./img/d7a72b14.png "点击放大")
  2. 选择调试设备，应用中显示已上传的应用，点击安装即可。

     ![](./img/85a49313.png "点击放大")
* **问题六解决方案：**

  “该应用类型不适用于所选设备”这个报错除了是应用类型错误，也可能是因为应用的版本过高，导致云调试的真机设备不能安装。查看工程目录build-profile.json5文件的compatibleSdkVersion字段，修改该字段版本让其小于或等于云调试真机版本。

  ![](./img/9d01637e.png)
