## 场景介绍

Native实现PCM封装为m4a、amr格式的音频是影音娱乐类应用的高频使用场景之一，如用户在进行音频创作时，需要先采集PCM音频，然后对PCM音频做裁剪，裁剪后的PCM文件则需要将其封装成其他格式的音频，如m4a、amr格式的音频。

本示例使用[AudioCapturer](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-audio-audiocapturer)实现PCM音频录制，[AVCodec Kit](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/avcodec-kit-intro)实现PCM编码并封装为m4a、amr格式的音频。

## 实现思路

1. 创建音频录制器。

   ```
   // AudioRecorder.ets
   public async init(context: Context, audioSteamInfo?: audio.AudioStreamInfo): Promise<void> {
       // do something
       // ...

       try {
         // 清空文件夹中的所有文件
         if (fileIo.accessSync(this.defaultFileDir)) {
           fileIo.rmdirSync(this.defaultFileDir);
         }
         fileIo.mkdirSync(this.defaultFileDir);

         this.recorder = await audio.createAudioCapturer(options);
       } catch (error) {
         Logger.error(`init AudioRecorder fail, code: ${error.code}, message: ${error.message}`);
       }
     }
   ```
2. 启动音频录制，并将音频保存在file目录下，音频保存目录可自定义，后续根据该录制文件进行编码为其他格式的音频，如m4a、amr格式。

   ```
   // AudioRecorder.ets
   public start() {
       // do something
       // ...

       this.audioFileInfo.createAudioFile(this.defaultFileDir);
       this.recorder.start();
     }
   ```
3. 初始化音频编码配置。

   ```
   // AudioEncode.cpp
   int32_t AudioEncode::InitCodecConfig(const FileInfo &fileInfo, SampleConfig *sampleConfig)
   {
       if (mEncoding) {
           return AVPROC_ERR_ALREADY_CODEC;
       }

       mEncoding = true;
       mFileInfo = fileInfo;
       mSampleConfig = sampleConfig;

       mAudioContext = new(std::nothrow) AVProcContext();
       if (!mAudioContext) {
           Release();
           return AVPROC_ERR_INIT_FAIL;
       }

       mAudioCodec = new(std::nothrow) AudioCodec(mAudioContext, mSampleConfig);
       if (!mAudioCodec) {
           Release();
           return AVPROC_ERR_INIT_FAIL;
       }

       mAudioMuxer = new(std::nothrow) AVMuxer(mSampleConfig);
       if (!mAudioMuxer) {
           Release();
           return AVPROC_ERR_INIT_FAIL;
       }

       int32_t ret = mAudioMuxer->InitMuxerConfig(fileInfo);
       if (ret != AVPROC_ERR_OK) {
           Release();
           return ret;
       }

       return AVPROC_ERR_OK;
   }
   ```
4. 启动音频编码和封装。

   ```
   // AudioEncode.cpp
   int32_t AudioEncode::StartEncode()
   {
       if (!mAudioCodec || !mAudioMuxer) {
           OH_LOG_ERROR(LOG_APP, "Codec(%{public}d) or Muxer(%{public}d) is nullptr", !mAudioCodec, !mAudioCodec);
           return AVPROC_ERR_CODEC_FAIL;
       }

       int32_t ret = mAudioMuxer->StartMuxer();
       if (ret != AVPROC_ERR_OK) {
           Release();
           return ret;
       }

       ret = mAudioCodec->StartCodec();
       if (ret != AVPROC_ERR_OK) {
           Release();
           return ret;
       }

       mEncodeInputThread = std::make_unique<std::thread>(&AudioEncode::AudioEncodeInputThread, this);
       mEncodeOutputThread = std::make_unique<std::thread>(&AudioEncode::AudioEncodeOutputThread, this);
       if (mEncodeInputThread == nullptr || mEncodeOutputThread == nullptr) {
           Release();
           return AVPROC_ERR_CODEC_FAIL;
       }

       mEncodeInputThread->detach();
       mEncodeOutputThread->detach();

       return AVPROC_ERR_OK;
   }
   ```
5. 停止音频编码和封装，停止时会先等待编码和封装完成的通知，收到完成的通知后，释放资源。

   ```
   // AudioEncode.cpp
   int32_t AudioEncode::StopEncode()
   {
       std::unique_lock<std::mutex> lock(mEncodeMutex);
       mEncodeCond.wait(lock);

       if (mAudioCodec != nullptr) {
           mAudioCodec->StopCodec();
       }
       if (mAudioMuxer != nullptr) {
           mAudioMuxer->StopMuxer();
       }

       Release();
       return mErrorCode;
   }
   ```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
