## 问题现象

用户尝试使用输入法输入文本时，无法输入字母、数字或特殊符号。

## 背景知识

* [InputType枚举说明](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textinput#inputtype枚举说明):通过type(value: InputType)，可以给TextInput设置输入框类型，默认值为type(InputType.Number)。不同的类型对应不同的输入模式，每种输入模式支持输入的内容也有所不同。
* [inputFilter](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textinput#inputfilter8)：通过正则表达式设置输入过滤器。匹配表达式的输入允许显示，不匹配的输入将被过滤。单字符输入场景仅支持单字符匹配，多字符输入场景支持字符串匹配，例如粘贴。典型场景参考[TextInput限制输入](https://developer.huawei.com/consumer/cn/doc/architecture-guides/common-v1_26-ts_82-0000002273289716)。从API version 11开始，设置inputFilter且输入的字符不为空字符，会导致[type](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textinput#type)接口附带的文本过滤效果失效。

## 问题定位

1. 通过UI布局工具查看输入框组件的type属性，或者查看代码，找到对应输入框组件，检查输入框类型type设置是否正确。
2. 如果正确设置了输入框类型，查看是否通过正则表达式设置了输入过滤器。

## 分析结论

### 场景一

输入框组件未正确设置输入框类型，如需要输入手机号/身份证号的输入框，设置为TextInput.type(InputType.PhoneNumber)后，身份证最后一位若为X将无法输入。

### 场景二

应用通过正则表达式设置了输入过滤器，过滤掉了需要输入的内容。

## 修改建议

### 场景一

选择正确的枚举值，设置输入框组件为合理的类型。

### 场景二

修改正则表达式，正确过滤不需要的内容。以只能输入字母和数字的场景为例，可以参考如下代码：

```
@Entry
@Component
struct InputFilterPage{
  @State account: string = '';

  build() {
    Column(){
      Text('账号登录')
        .fontSize(25)
        .margin(10)
      TextInput({ placeholder: '请输入账号（仅可输入字母和数字）', text: this.account })
        .width('90%')
        .inputFilter('^[0-9A-Za-z]+$', (e) => {
          console.log(JSON.stringify(e));
        })
    }
    .width('100%')
    .height('100%')
    .justifyContent(FlexAlign.Center)
  }
}
```
