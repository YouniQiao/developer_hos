## 问题现象

问题一：华为市场上公司账号调试证书申请数量已达上限，申请增加调试证书。

问题二：同一开发者账号下，若已上架应用并计划沿用现有发布证书上架新应用：

1. 开发新应用时是否还需生成p12文件？
2. 应用备案所需的指纹与公钥均从发布证书获取，那么使用同一发布证书的不同应用，备案时的指纹与公钥是否相同？

## 背景知识

* 在AGC平台进行证书申请操作时，存在数量限制，仅支持申请3个发布证书与3个调试证书，如已达到上限，需要在证书管理页面中“废除”多余的证书文件。当有多个应用的时候，cer文件（证书）跟应用不绑定，在新建[Profile（p7b文件）](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-publish-app#section793484619307)时需要选择应用名，此时才跟应用绑定。多个应用使用相同的签名文件，p7b文件不同即可。
* 基于同一发布证书的多个应用，可共用同一套MD5指纹与签名公钥完成备案。

## 解决方案

在调试阶段，多个应用可共用同一调试证书；但每个应用需单独申请对应的Profile文件。此外，各应用可使用同一套MD5指纹与签名公钥完成备案流程，以提升开发效率与管理便利性。

* 针对问题一：调试证书是可以共用的，调试证书的共用流程与发布证书相同，可参考[发布证书共用流程](https://developer.huawei.com/consumer/cn/doc/app/agc-help-releasefaq-0000001110342644#section843592204116)。
* 针对问题二：
  + 在同一开发者账号下发布多个应用时，可复用同一发布证书。除了需要为每个应用单独申请Profile文件，以明确各应用的权限配置及设备安装许可等专属信息外，其余文件均可复用。
  + 在应用备案流程中，基于同一发布证书的多个应用，可共用同一套MD5指纹与签名公钥完成备案，有助于提升发布效率并保障备案信息的一致性与合规性。

## 常见FAQ

Q：删除证书对应用是否有影响？

A：发布证书删除后对在架版本应用没有影响，但是建议不要删除或者删除后保留本地，后续应用更新时需要使用。

Q：不同产品的csr文件不一样如何共用证书？

A：不同应用可以使用同一个发布证书，Profile文件不一样。

使用同一个csr文件申请发布证书。

升级应用时保证使用相同的证书和Profile文件就没有问题。AGC页面删除后不影响在架的版本运行。新版本需要使用相同的csr文件生成证书cer和Profile，然后下载至工程中编译应用即可。

Q：开发者账号能否支持多个发布证书？

A：每个账号最多可申请3个发布证书，发布证书是可以共用，更多问题可以参考[官网](https://developer.huawei.com/consumer/cn/doc/app/agc-help-release-cert-0000002283336729)。

Q：签名流程中涉及到的文件有p12、csr、cer、p7b这些，他们是否可以多应用共用？以及他们分别应该由开发者创建还是在AGC上生成？

A：

|  | p12（密钥库文件） | csr（证书请求文件） | cer（证书） | p7b（Profile） |
| --- | --- | --- | --- | --- |
| 生成方式 | 可通过IDE自主创建 | 可通过IDE自主创建 | AGC上生成 | AGC上生成 |
| 多应用共用 | 是 | 是 | 是 | 否 |
| 调试/发布模式共用 | 是 | 是 | 否 | 否 |

Q：废除的证书是否占用账号申请证书的数量？

A：废除的证书不计入账号的证书申请数量限制。
