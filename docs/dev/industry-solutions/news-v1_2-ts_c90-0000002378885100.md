## 问题现象

* 现象一：加载页面时，整张图片先显示白色或黑色，再显示实际图片。
* 现象二：加载页面时，图片先显示占位图，再显示图片。

## 背景知识

* [资源提前加载](https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-preloading-resources)：在附带网络请求的页面跳转场景中，完成时延耗时长的绝大多数原因都是因为网络数据HTTP请求时间长。由于网络是从操作系统侧发起和控制的，且网络环境存在不可控性，所以我们很难在业务逻辑的代码中优化请求速度。因此应尽可能的提前发起网络请求。
* [Image白块解决方案](https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-image-white-lump-solution)：加载网络图片时，整个过程可分为四个关键阶段：组件创建、图片资源下载、图片解码和最终刷新显示。当加载的图片资源过大时，组件需等待下载与解码完成后才进行刷新。由于下载阶段耗时较长，图片在完全渲染前会显示为空白或浅色占位图，这种现象被称为“Image白块”。它不仅影响视觉体验，还可能降低用户对应用性能的感知。

## 问题定位

1. 使用[DevEco Testing](https://developer.huawei.com/consumer/cn/doc/harmonyos-faqs/faqs-deveco-testing)查看页面布局，确认图片是Image组件。
2. 排查在父组件里[aboutToAppear](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-custom-component-lifecycle#abouttoappear)中是否提前发起用于缓存子组件图片的网络请求。

   ```
   aboutToAppear(): void {
     httpRequest();  // 请求子组件图片的数据
   }
   ```
3. 排查当父页面点击按钮跳转子页面时，是否触发pixMap请求读取应用沙箱中已缓存解码的网络图片并存储在[localStorage](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-state-management#localstorage9)中。排查是否使用LocalStorageLink保存图片数据，以及是否使用[createPixelMap](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-image-imagesource#createpixelmap7)方法创建PixelMap对象。

   ```
   Button('push Path to pageOne')
     .onClick(() => {
       if (!localStorage.get('imageData')) { // 减少重复读取和存储
         this.getPixMap(); // 读取图片数据并存储
       }
       this.childNavStack.pushPath({ name: 'pageOne' }); // 跳转子页面
     })
   ```
4. 在网络请求的方法中，排查是否使用[fs.access](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-file-fs#fsaccess)检查文件是否存在，当文件存在时不再重复请求，并写入沙箱中。

   ```
   fs.access(fileUrl, fs.AccessModeType.READ).then((res) => {
     // 如果文件不存在，发起请求
     if (!res) {
       http.createHttp()
         // 图片地址
         .request('https://example.com/xxx.png',
           (error: BusinessError, data: http.HttpResponse) => {
             if (error) {
               return;
             }
             if (http.ResponseCode.OK === data.responseCode) {
               const imageData: ArrayBuffer = data.result as ArrayBuffer;
               // 将文件放入沙箱中
               readWriteFileWithStream(imageData)
             }
           }
         )
     }
   }
   ```
5. 排查在子组件中是否通过使用[@StorageLink](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-appstorage#storagelink)修饰的变量ImageData进行数据刷新，图片送显。

   ```
   @LocalStorageLink('imageData') imageData: PixelMap | undefined = undefined;
   Image(this.imageData)
     .objectFit(ImageFit.Auto)
     .width('100%')
     .height('100%')
   ```

## 分析结论

未提前加载图片资源，导致页面跳转时图片先显示白块、黑块或占位图，再显示图片。

## 修改建议

建议提前加载图片资源，具体实现方式参考[Image白块解决方案](https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-image-white-lump-solution)。
