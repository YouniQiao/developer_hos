## 场景介绍

多张图片转Pixelmap合成GIF动图是拍摄美化类应用中的典型场景之一。

本示例基于[packToFileFromPixelmapSequence](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-image-imagepacker#packtofilefrompixelmapsequence18)实现将多个PixelMap编码成GIF文件，且本示例可以基于不同版本API展示不同功能。

## 实现思路

1. 通过[@ohos.deviceInfo](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-device-info)获取设备API版本号，判断是否展示动图合成功能。

   ```
   // 获取设备API版本号
   @State sdkApiVersionInfo: number = deviceInfo.sdkApiVersion;
   // 判断是否展示动图合成功能
   if (this.sdkApiVersionInfo >= Constants.API_18){}
   ```

2. 通过[packToFileFromPixelmapSequence](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-image-imagepacker#packtofilefrompixelmapsequence18)将图库中选择的图片合成为GIF动图。

   ```
   let context = this.getUIContext().getHostContext() as common.UIAbilityContext;
   let path: string = context.cacheDir + '/result.gif';
   if (fs.accessSync(path)) {
       fs.unlinkSync(path)
       Logger.info('Delete duplicated result file' + path);
   }
   let file = fs.openSync(path, fs.OpenMode.CREATE | fs.OpenMode.READ_WRITE);
   // 动图编码参数
   let ops: image.PackingOptionsForSequence = {
       frameCount: result.photoUris.length, // 指定GIF编码中的帧数为选择的图片数
       delayTimeList: [Constants.DELAY_TIME_LIST_50], // GIF编码中帧的延迟时间，单位为10ms
       disposalTypes: [Constants.DISPOSAL_TYPES_3], // GIF编码中的帧过渡模式，3（恢复到之前的状态）
       loopCount: Constants.LOOP_COUNT_0 // 指定GIF编码中循环次数为无限循环
   };
   let packer = image.createImagePacker();
   // 合成动图
   await packer.packToFileFromPixelmapSequence(pixelMapList, file.fd, ops);
   ```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
