## 问题现象

应用打开，在前台时点击播放背景音乐，连续侧滑2次，中间应用提示“再按一次退出”，退至手机桌面，此时背景音乐还在继续播放。

## 背景知识

* [长时任务](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/continuous-task)：应用退至后台后，在后台需要长时间运行用户可感知的任务，如播放音乐、导航等。
* [onBackPress](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-custom-component-lifecycle#onbackpress)：属于自定义组件的生命周期回调函数之一，当用户点击返回按钮或侧滑返回时触发，仅@Entry装饰的自定义组件生效。

## 分析结论

应用通过侧滑回到手机桌面处理逻辑并不是结束进程，而是进入后台状态。此时在长时任务的情况下，仍然可以播放背景音乐。

## 修改建议

应用在自定义侧滑退出应用时应结束应用进程，而不是进入后台。ArkTS提供了UIAbilityContext.[terminateSelf](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-inner-application-uiabilitycontext#terminateself)和UIAbilityContext.[terminateSelfWithResult](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-inner-application-uiabilitycontext#terminateselfwithresult)两个接口用于主动退出停止。参考代码：

```
import { common } from '@kit.AbilityKit';

@Entry
@Component
struct Index {
  private message: string = '连续侧滑2次退出应用';
  private firstBackTimestamp: number = 0;
  private context: common.UIAbilityContext = this.getUIContext().getHostContext() as common.UIAbilityContext;
  promptAction = this.getUIContext().getPromptAction();

  build() {
    Column() {
      Text(this.message)
        .fontWeight(FontWeight.Bold);
    }
    .height('100%')
    .width('100%')
    .justifyContent(FlexAlign.Center);
  }

  onBackPress(): boolean | void {
    let now = Date.now();
    if (now - this.firstBackTimestamp < 1000) {
      this.context.terminateSelf();
    } else {
      this.firstBackTimestamp = now;
      this.promptAction.showToast({
        message: '再侧滑一次退出应用',
        duration: 1000
      });
    }
    return true;
  }
}
```
