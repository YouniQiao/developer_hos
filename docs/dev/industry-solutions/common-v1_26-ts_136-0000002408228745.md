## 问题现象

开发一个卡片，FormLink如何使用if/else进行渲染控制？

## 背景知识

[FormLink](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-container-formlink)提供静态卡片交互组件，用于静态卡片内部和提供方应用间的交互，当前支持router、message和call三种类型的事件。

## 解决方案

FormLink组件本身不直接支持if/else渲染控制，可尝试将FormLink组件放在一个容器组件中，然后对容器组件应用if/else条件来间接实现这一功能。

```
@Entry
@Component
struct FormLinkDemo {
  @State flag: boolean = false;

  build() {
    Column() {
      Text('这是一个静态卡片').fontSize(20).margin(10)
      Column() {
        if (this.flag) {
          // router事件用于静态卡片跳转到对应的UIAbility
          FormLink({
            action: 'router',
            abilityName: 'EntryAbility',
            uri: 'example://uri.ohos.com/link_page',
            params: {
              'message': 'testForRouter' // 自定义要发送的message
            }
          }) {
            Button('router event').width(120)
          }.margin(10)
        } else {
          // message事件触发FormExtensionAbility的onFormEvent生命周期
          FormLink({
            action: 'message',
            abilityName: 'EntryAbility',
            uri: 'example://uri.ohos.com/link_page',
            params: {
              'message': 'messageEvent' // 自定义要发送的message
            }
          }) {
            Button('message event').width(120)
          }.margin(10)
        }
      }

      Button('切换').onClick(() => {
        this.flag = !this.flag;
      })
    }
    .justifyContent(FlexAlign.Center)
    .width('100%').height('100%')
  }
}
```
