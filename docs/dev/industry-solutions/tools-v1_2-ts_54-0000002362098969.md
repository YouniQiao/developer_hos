## 问题现象

如何实现Navigation和router的混用？

## 背景知识

组件导航[Navigation](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-navigation-navigation)与页面路由[router](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-router)都能实现应用内页面之间的切换，Navigation由于可以在不同组件间跳转，使用起来更为便捷和灵活。它拥有更强的跨平台开发能力，能够更自如地管理页面堆栈，并且支持更多样化的过渡动画效果以及更细致的生命周期管理。

## 解决方案

* Navigation和router可以混合使用，但有一些限制和推荐使用的策略：
  + 为了确保应用的一致性和整体交互体验，推荐在不涉及复杂场景的情况下使用router，而在需要复杂功能的情况下使用Navigation。
  + 在使用Navigation时，需要注意一些细节，例如，Navigation自带导航栏，标题栏，工具栏等，但可以设置隐藏属性（hideNavBar，hideTitleBar，hideToolBar）仅实现路由功能，对于原来使用router路由并已经设置了相关标题栏等UI的页面不用做过多修改。
  + 在混合使用时，需要确保路由跳转的一致性和正确性，避免不必要的错误。例如，跳转时增加一个字段表示来源，参考[Navigation示例4（Navigation带参返回）](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-navigation#示例4navigation带参返回)。通过传参表明是通过Navigation实现的页面跳转；当使用router跳转页面时，即使跳转的是Navigation页面，也仍然使用[getParams()](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-router#getparams)传递参数。

* 对于router跳转到的页面需要使用Navigation作为根容器，才能在这个页面跳转到下个NavDestination页面。这是因为router跳转的是page页面，而NavDestination是Navigation的子组件，二者层次不同。以下为Navigation和router混用的三种常见场景。
  + **场景一**：Navigation页面与router页面间的跳转，如：Navigation->NavPageOne->RouterPageOne->NavPageTwo。该混用场景常见于“全局导航+局部流程”的页面设计，例如首页到隐私模块页面等。

    使用router方式跳转到RouterPageOne，RouterPageOne使用Navigation作为根容器跳转到NavPageTwo，这里RouterPageOne的Navigation和第一个Navigation相互独立，都有各自的栈，互不影响。开发者根据需求参考[系统路由表](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-navigation-navigation#系统路由表)自行配置。

    NavIndex页：

    ```
    @Entry
    @Component
    struct NavIndex {
      pageInfo: NavPathStack = new NavPathStack();

      build() {
        Navigation(this.pageInfo) {
          Column() {
            Button('ToNavDestinationOneByNavPathStack', { stateEffect: true, type: ButtonType.Capsule })
              .width('80%')
              .height(40)
              .margin(24)
              .onClick(() => {
                this.pageInfo.pushPath({ name: 'NavPageOne' });
              });
          };
        }.title('NavIndex');
      }
    }
    ```

    NavPageOne页：

    ```
    @Builder
    export function PageOneBuilder() {
      NavPageOne();
    }

    @Component
    struct NavPageOne {
      pageInfo: NavPathStack = new NavPathStack();

      build() {
        NavDestination() {
          Text('NavDestinationOne')
            .width('80%')
            .height(50)
            .margin(24);
          Button('ToRouterPage1ByRouter', { stateEffect: true, type: ButtonType.Capsule })
            .width('80%')
            .height(40)
            .margin(10)
            .onClick(() => {
              // 使用router跳转到下一个页面
              this.getUIContext().getRouter().pushUrl({
                url: 'pages/RouterPageOne'
              });
            });
        }
        .height('100%')
        .width('100%');
      }
    }
    ```

    需注意router管理@Entry页面之间的跳转（页面级路由栈）。Navigation管理NavDestination组件之间的跳转（组件级路由栈），NavDestination作为Navigation的子组件存在，无法独立于Navigation使用。若通过router跳转到一个需要承载NavDestination的页面，该页面（@Entry）必须用Navigation作为容器。

    RouterPageOne页：

    ```
    @Entry
    @Component
    struct RouterPageOne {
      pageInfo: NavPathStack = new NavPathStack();

      build() {
        // 注意：使用NavDestination，会导致无法跳转到指定NavDestination
        // 使用Navigation，可以正常跳转
        NavDestination() {
          Column() {
            Text('RouterPage1')
              .margin(24);
            Button('ToNavDestination2ByNavPathStack').onClick(() => {
              this.pageInfo.pushPath({ name: 'NavPageTwo' });
            });
          };
        }
        .height('100%')
        .width('100%');
      }
    }
    ```

  + **场景二**：一个页面仅在应用打开后出现一次，在使用应用中不断返回也不会再次出现。

    使用router跳转到一个Navigation作为根容器的页面，然后清空router路由栈中的页面。

  + **场景三**：Navigation页面与router页面间的跳转为：Navigation->PageOne->pageTwo->RouterPage->PageThree。

    该混用场景常见于使用的三方SDK布局为router导航，且根容器不是Navigation。可以使用router的单例模式（[页面跳转](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-routing#页面跳转)的场景三）拉起Navigation首页，通过路由监听或者传参等方式控制Navigation首页跳转PageThree：

    ```
    @Entry
    @Component
    struct SceneThree {
      pageInfos: NavPathStack = new NavPathStack();
      isUseInterception: boolean = false;

      onPageShow(): void {
        let rouNam: number | undefined = AppStorage.get('router');
        if (rouNam === 1) {
          this.getUIContext().getRouter().clear(); // 清除掉历史页面router
          this.pageInfos.pushDestination({ name: 'pageThree' }, false);
        }
      }

      build() {
        Stack() {
          Navigation(this.pageInfos) {
            Button('pageOne', { stateEffect: true, type: ButtonType.Capsule })
              .width('80%')
              .height(40)
              .margin(20)
              .onClick(() => {
                this.pageInfos.pushPath({ name: 'pageOne' });
              });
          }.title('NavIndex').titleMode(NavigationTitleMode.Mini);
        };
      }
    }
    ```

    PageOne页：

    ```
    @Builder
    export function PageOneBuilder() {
      PageOne();
    }

    @Component
    export struct PageOne {
      pageInfos: NavPathStack = new NavPathStack();

      build() {
        NavDestination() {
          Column() {
            Button('pageTwo', { stateEffect: true, type: ButtonType.Capsule })
              .width('80%')
              .height(40)

                this.pageInfos.pushPath({ name: 'pageTwo' });

        .onBackPressed(() => {
          this.pageInfos.pop();
          return true;
        }).onReady((context: NavDestinationContext) => {
          this.pageInfos = context.pathStack;
        })
        .customTransition(null);
      }
    }
    ```

    PageThree页：

    ```
    import { router } from '@kit.ArkUI';

    @Builder
    export function PageThreeBuilder() {
      PageThree();
    }

    @Component
    export struct PageThree {
      pathStack: NavPathStack = new NavPathStack();

      build() {
        NavDestination() {
          Column() {
            Button('pop', { stateEffect: true, type: ButtonType.Capsule })
              .width('80%')
              .height(40)
              .margin(20)
              .onClick(() => {
                this.pathStack.pop();
              });
          }.width('100%').height('100%');

        }.title('pageThree')
        .onBackPressed(() => {
          let rouNam: number | undefined = AppStorage.get('router');
          // 消除返回重影
          setTimeout(() => {
            this.pathStack?.pop();
          }, rouNam === 1 ? 500 : 0);
          if (rouNam === 1) {
            this.getUIContext().getRouter().clear(); // 清除掉历史页面router
            AppStorage.delete('router');
            this.getUIContext().getRouter().pushUrl({ url: 'pages/RouterPage' }, router.RouterMode.Single, (err) => {
              if (err) {
                console.error(`Invoke pushUrl failed, code is ${err.code}, message is ${err.message}`);
                return;
              }
              console.info('Invoke pushUrl succeeded.');
            });
          }
          return true;
        }).onReady((context: NavDestinationContext) => {
          this.pathStack = context.pathStack;
        });
      }
    }
    ```

## 总结

混合使用Navigation和router时应注意以下关键点：

1. router页面需以Navigation为根容器。
2. 避免路由栈冲突：Navigation通过NavPathStack管理自身路由栈，router有独立的路由栈机制。混合跳转时（如：NavDestination -> RouterPage），需显式清空router栈中不需要的页面（如闪屏页）。
3. 跳转方向需符合层级：router仅支持跳转到用@Entry的页面，无法通过router直接从NavDestination跳转到另一个NavDestination，需要经过@Entry页面中转。推荐单独使用Navigation组件导航，不建议混用。
