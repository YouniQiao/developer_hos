## 场景介绍

商品对比查询是购物比价类应用的高频使用场景之一，如选择其他产品与当前产品进行各方面对比。

本示例基于[半模态转场](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-attributes-sheet-transition)和[Radio](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-radio)实现商品对比查询的功能。

## 实现思路

1. 点击“PK去对比”按钮后，将isSheetShow置true，拉起包含对比商品列表的[半模态转场](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-attributes-sheet-transition)。弹窗内容在sheetContent()中实现，通过[Radio](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-radio)唯一选择对比的商品。

   ```
   Navigation(this.pageInfos) {
   }
   .bindSheet($$this.isSheetShow, this.sheetContent(), {
     height: Constants.FULL_HEIGHT,
     blurStyle: BlurStyle.Thick,
     showClose: false,
     backgroundColor: Constants.BACKGROUND_COLOR
   })
   Radio({ value: 'Radio1', group: 'radioGroup' })
     .onChange(() => {
       this.isChosen = goodsItem.id;
       this.isClickable = true;
     })
   ```
2. 点击“进行比价”按钮后，关闭弹窗并跳转至商品对比页，并传递包含原商品与对比商品的ID的字段。

   ```
   Button($r('app.string.PK'))
     .onClick(() => {
       if (this.isChosen !== 0) {
         this.isSheetShow = false;
         const goodsChooseParam: PageParam = { goodsOriginal: this.goodsOriginal, goodsPK: this.isChosen };
         this.pageInfos.pushPathByName('PKDetails', goodsChooseParam);
       }
     })
   ```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
