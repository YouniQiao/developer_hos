## 场景介绍

个人名片页是综合办公类应用的高频场景之一，如用户需要保存个人二维码或扫一扫他人二维码。

本示例通过弹窗引导用户授权实现个人名片页二维码保存至图库，使用[scanBarcode](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/scan-scanbarcode-api)实现二维码扫一扫功能。

## 实现思路

* 个人名片页设置：
  1. 生命周期函数aboutToAppear中设置二维码初始属性并生成二维码；
  2. 创建[ImagePacker](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-image-imagepacker)，在应用沙箱内新建并写入二维码内容，通过弹窗引导用户授权获取临时权限保存二维码至图库。
* 扫描他人二维码：使用[scanBarcode](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/scan-scanbarcode-api)实现二维码扫一扫功能，判断扫码结果为正确后返回结果。

```
// 创建二维码
generateBarcode.createBarcode(content, options);
// 判断是否收到正确的二维码，正确后返回扫码结果
scanBarcode.startScanForResult(this.context, options);
// 创建ImagePacker
imagePackerApi: image.ImagePacker = image.createImagePacker();
// 应用沙箱路径
path = context.filesDir + '/test.png';
// 在沙箱新建并打开文件
file = fs.openSync(path, fs.OpenMode.READ_WRITE | fs.OpenMode.CREATE);
// 写入pixelMap图片内容
fs.writeSync(file.fd, buffer);
// 关闭文件
fs.closeSync(file.fd);
// 使用showAssetsCreationDialog保存沙箱中的图片
srcFileUris: Array<string> = [fileUri.getUriFromPath(context.filesDir + '/test.png')];
photoAccessHelper.getPhotoAccessHelper(context);
imageFile1 = fileIo.openSync(desFileUris[0], fileIo.OpenMode.CREATE | fileIo.OpenMode.READ_WRITE);
let srcPath = context.filesDir + '/test.png';
let dstPath = imageFile1.fd;
fileIo.copyFileSync(srcPath, dstPath);
fileIo.closeSync(imageFile1.fd);
```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。

## 权限说明

获取相机使用权限：[ohos.permission.CAMERA](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/permissions-for-all-user#ohospermissioncamera)。
