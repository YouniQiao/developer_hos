## 问题现象

应用有自己的Crash监控平台，当前Crash问题只收集上报了堆栈信息，部分Crash问题仅根据堆栈信息很难定位，如何获取并上报Crash问题日志。

## 背景知识

* [HiAppEvent](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/hiappevent-intro)是在系统层面为应用开发者提供的一种事件打点机制，帮助应用记录在运行过程中发生的故障信息、统计信息、安全信息、用户行为信息，支撑开发者分析应用的运行情况。以便进一步统计分析访问数、日常用户活跃数量、用户操作习惯以及其他影响用户使用产品的关键因素。
* [Worker](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/worker-introduction)的主要作用是为应用程序提供一个多线程的运行环境，实现应用程序执行过程与宿主线程分离。通过在后台线程运行脚本处理耗时操作，避免计算密集型或高延迟任务阻塞宿主线程。
* [rcp](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/remote-communication-rcp)提供HTTP数据请求功能。应用程序可通过HTTP发起数据请求。常见的HTTP方法包括GET、POST、HEAD、PUT、DELETE、PATCH、OPTIONS等。
* [应用沙箱](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/app-sandbox-directory)是一种以安全防护为目的的隔离机制，避免数据受到恶意路径穿越访问。在这种沙箱的保护机制下，应用可见的目录范围即为“应用沙箱目录”。

## 解决方案

应用可以调用HiAppEvent的addWatcher接口订阅Crash事件，当应用进程发生故障时，DFX系统捕获相关信息，生成事件和日志，并写入到共享目录。HiAppEvent监听到事件后，将事件回调给应用，应用可以通过external\_log参数来获取日志文件的沙箱路径，并在Worker子线程中读取日志文件上传到自己的Crash监控平台。

关键步骤如下：

1. 订阅系统事件，事件名指定为hiAppEvent.event.APP\_CRASH和hiAppEvent.event.APP\_FREEZE。为确保能够订阅到崩溃事件，可以在EntryAbility.ets的onCreate方法中调用该方法：

   ```
   addCrashWatcher(): void {
     try {
       hiAppEvent.addWatcher({
         name: 'watcher',
         appEventFilters: [
           {
             domain: hiAppEvent.domain.OS,
             names: [hiAppEvent.event.APP_CRASH, hiAppEvent.event.APP_FREEZE]
           }
         ],
         onReceive: (domain: string, appEventGroups: Array<hiAppEvent.AppEventGroup>) => {
           hilog.info(0x0000, 'testTag', `HiAppEvent onReceive: domain=${domain}`);
           for (const eventGroup of appEventGroups) {
             hilog.info(0x0000, 'testTag', `HiAppEvent eventName=${eventGroup.name}`);
             for (const eventInfo of eventGroup.appEventInfos) {
               let logPath: string = eventInfo.params['external_log'][0];
               hilog.info(DOMAIN, 'testTag', `logPath:${logPath}`);
               this.uploadInWorker(logPath);
             }
           }
         }
       });
     } catch (error) {
       hilog.error(DOMAIN, 'testTag', 'Failed to addWatcher hiAppEvent. err: %{public}s', JSON.stringify(error));
     }
   }
   ```

   HiAppEvent监听到事件发生后，会回调onReceive方法，eventInfo中的external\_log参数表示Crash日志文件的沙箱路径。
2. 创建Worker子线程，为日志文件上传准备子线程运行环境。上传日志涉及联网，为避免阻塞主线程，保障应用流畅性，上传日志需要运行在子线程中。将上一步获取的日志文件沙箱路径作为参数，通过postMessage方法传递给Worker子线程：

   ```
   uploadInWorker(srcPath: string) {
     let workerInstance = new worker.ThreadWorker('entry/ets/workers/UploadWorker.ets');
     try {
       workerInstance.postMessage({
         srcPath: srcPath,
       });
     } catch (error) {
       hilog.info(DOMAIN, 'testTag', `workerInstance::postMessage has errors: ${JSON.stringify(error)}`);
     }

     workerInstance.onexit = (code): void => {
       hilog.info(DOMAIN, 'testTag', `workerInstance::onexit has been exit ${code}`);
     };
     workerInstance.onerror = (err): void => {
       hilog.info(DOMAIN, 'testTag', `workerInstance::onerror has errors: ${JSON.stringify(err)}`);
     };
     workerInstance.onmessageerror = (event): void => {
       hilog.info(DOMAIN, 'testTag', `workerInstance::onmessageerror has errors: ${JSON.stringify(event)}`);
     };
     workerInstance.onmessage = (message): void => {
       hilog.info(DOMAIN, 'testTag', `workerInstance::onmessage uploadSuccess:${message.data.upLoadSuccess}`);
     };
   }
   ```
3. 在entry/ets/workers/UploadWorker.ets文件的workerPort.onmessage方法中使用rcp实现日志上传服务器的功能。将日志文件沙箱路径srcPath组装成MultipartForm数据通过post方法向应用服务器提交请求，文件上传成功后使用fs.unlink方法删除沙箱中的日志文件，避免日志文件数量溢出超过上限导致日志上传功能失效。

   ```
   workerPort.onmessage = (event: MessageEvents) => {
     let srcPath: string = event.data.srcPath;
     try {
       let accessible = fs.accessSync(srcPath);
       if (!accessible) {
         hilog.error(DOMAIN, 'testTag', 'srcPath not accessible');
         return;
       }
       const session = rcp.createSession();
       const formData = new rcp.MultipartForm({
         file: {
           contentOrPath: srcPath
         }
       });
       // serverUrl需要替换为服务器实际地址
       session.post('serverUrl', formData)
         .then((res: rcp.Response) => {
           if ([200, 206].includes(res.statusCode)) {
             fs.unlink(srcPath).catch(() => {
               hilog.error(DOMAIN, 'testTag', 'unlink srcPath error');
             });
             workerPort.postMessage({
               upLoadSuccess: true
             });
           }
         })
         .catch((err: BusinessError) => {
           hilog.error(DOMAIN, 'testTag', `post failed: code: ${err.code}, message: ${err.message}`);
         }).finally(() => {
         session.close();
       });
     } catch (exception) {
       hilog.error(DOMAIN, 'testTag', `upload failed. Code:${exception.code}, message:${exception.message}`);
     }
   };
   ```

* UploadWorker.ets文件:

  ```
  import { MessageEvents, ThreadWorkerGlobalScope, worker } from '@kit.ArkTS';
  import { rcp } from '@kit.RemoteCommunicationKit';
  import { hilog } from '@kit.PerformanceAnalysisKit';
  import { fileIo as fs } from '@kit.CoreFileKit';

  const DOMAIN = 0x0000;
  const workerPort: ThreadWorkerGlobalScope = worker.workerPort;

  /**
   * Defines the event handler to be called when the worker thread receives a message sent by the host thread.
   * The event handler is executed in the worker thread.
   *
   * @param event message data
   */
  workerPort.onmessage = (event: MessageEvents) => {
    let srcPath: string = event.data.srcPath;
    try {
      let accessible = fs.accessSync(srcPath);
      if (!accessible) {
        hilog.error(DOMAIN, 'testTag', 'srcPath not accessible');
        return;
      }
      const session = rcp.createSession();
      const formData = new rcp.MultipartForm({
        file: {
          contentOrPath: srcPath
        }
      });
      // serverUrl需要替换为服务器实际地址
      session.post('serverUrl', formData)
        .then((res: rcp.Response) => {
          if ([200, 206].includes(res.statusCode)) {
            fs.unlink(srcPath).catch(() => {
              hilog.error(DOMAIN, 'testTag', 'unlink srcPath error');
            });
            workerPort.postMessage({
              upLoadSuccess: true
            });
          }
        })
        .catch((err: BusinessError) => {
          hilog.error(DOMAIN, 'testTag', `post failed: code: ${err.code}, message: ${err.message}`);
        }).finally(() => {
        session.close();
      });
    } catch (exception) {
      hilog.error(DOMAIN, 'testTag', `upload failed. Code:${exception.code}, message:${exception.message}`);
    }
  };
  ```
