## 场景介绍

应用卡片添加至桌面是实用工具类应用中的典型场景之一，如用户可长按桌面应用，将卡片中的常用功能、设备存储等信息添加至桌面常驻展示。用户通过“长按桌面”或使用“小部件”功能，可以自由选择和调整这些卡片的大小、位置和更新频率。这种轻量级展示方式，既能保证信息的即时可见性，又不会干扰到主屏幕的使用体验。

本示例通过[@ohos.batteryInfo](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-battery-info)获取电量信息，通过[@ohos.deviceInfo](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-device-info)获取设备信息，通过[@ohos.file.storageStatistics](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-file-storage-statistics)获取存储相关信息。

## 实现思路

1. 在项目中创建服务卡片后，在EntryFormAbility中完善[FormExtensionAbility.onAddForm](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-app-form-formextensionability#formextensionabilityonaddform)函数，以达到在创建服务卡片实例时，初始化卡片信息的目的。

   ```
   // 卡片创建时触发的生命周期函数
   onAddForm(want: Want): formBindingData.FormBindingData {}
   ```
2. 本示例选择的卡片数据更新方式有两种：
   * 从应用更新数据到卡片：本示例采用订阅电量变化公共事件的方式，从应用主动更新数据到卡片。

     ```
     // 订阅电量信息变化事件
     subscribeBatteryInfo() {}
     // 向卡片发送更新数据的函数
     sendFormMsg() {}
     ```
   * 拉起应用至后台时更新卡片：在卡片页面注册onClick()点击事件，在回调中调用[postCardAction](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-postcardaction#postcardaction-1)接口，action参数选择call并配置需要调用的方法和传递数据，触发call事件后台拉起指定UIAbility。其中method参数为必选参数，用于触发UIAbility中对应的同名方法。

     ```
     // EntryAbility中实现postCardAction()接口method参数对应同名方法
     updateBatteryInfoCall = (data: rpc.MessageSequence) => {}

     // 在EntryAbility的onCreate()中进行监听
     this.callee.on('updateBatteryInfoCard', this.updateBatteryInfoCall);

     // 在EntryAbility的onDestroy()中解除监听
     this.callee.off('updateBatteryInfoCall');
     ```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。

## 权限说明

允许应用在后台持续运行权限：[ohos.permission.KEEP\_BACKGROUND\_RUNNING](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/permissions-for-all#ohospermissionkeep_background_running)。
