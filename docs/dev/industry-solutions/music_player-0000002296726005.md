## 场景介绍

导入本地音乐文件是影音娱乐类应用中高频使用场景之一。

本示例主要使用[@ohos.file.picker](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-file-picker)加载公共目录的音乐文件，并读取歌手名、歌曲封面等信息进行展示。

## 实现思路

1. 使用[DocumentViewPicker](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-file-picker#documentviewpicker)加载本地文件，限制选择文件的后缀类型，仅添加音频文件。

   ```
   let documentPicker = new picker.DocumentViewPicker(context);
   let selectResult: string[] = await documentPicker.select({ fileSuffixFilters: ['.mp3', '.flac', '.ogg', '.wav'], maxSelectNumber: 500 });
   ```
2. 通过[AVMetadataExtractor](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-media-avmetadataextractor)类，读取音频文件的元数据，如歌曲名、歌手名、歌曲封面等。

   ```
   // 创建AVMetadataExtractor对象
   let avMetadataExtractor = await media.createAVMetadataExtractor();
   // 设置fdSrc
   avMetadataExtractor.fdSrc = fs.openSync(uri);
   // 获取元数据（promise模式）
   let metadata = await avMetadataExtractor.fetchMetadata();
   ```
3. 使用[AVPlayer](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-media-avplayer)的avPlayer.play()方法播放音频文件。
4. 对已经导入的文件进行[持久化授权](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/file-persistpermission)，并持久化URI。

   ```
   let policies: Array<fileShare.PolicyInfo> = this.getFilePolicyInfo(pathArray)
   if (!this.checkSystemCapability()) {
     Logger.error(TAG, 'persistPermission can not use')
     return;
   }
   try {
     await fileShare.persistPermission(policies)
     Logger.info(TAG, 'persistPermission successfully')
   } catch (err) {
     Logger.error(TAG, 'persistPermission failed with error message: ' + err.message + ', error code: ' + err.code)
     this.dataPreferences?.clearSync()
   }
   ```
5. 在应用启动完成前，[激活已经持久化的权限访问文件](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/file-persistpermission#激活已经持久化的权限访问文件或目录)。

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。

## 权限说明

获取支持持久化访问文件URI权限：[ohos.permission.FILE\_ACCESS\_PERSIST](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/restricted-permissions#ohospermissionfile_access_persist)。

![](./img/b9b01324.png)

本示例使用了受限权限ohos.permission.FILE\_ACCESS\_PERSIST。在调试阶段，可通过DevEco Studio自动签名完成申请。在自动签名的过程中，将由DevEco Studio完成向AGC申请受限权限的步骤，开发者可直接使用，无需额外申请，具体请参考[自动签名-操作步骤](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-signing#section151231211105010)。在发布阶段，需根据申请发布Profile完成受限权限的手动申请，具体请参考[申请受限权限](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/declare-permissions-in-acl)。
