## 场景介绍

本示例基于[@ohos.resourceManager](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-resource-manager)加载本地预置数据库（或从网络下载的数据库），实现文章列表刷新的功能。

## 实现思路

1. 在EntryAbility组件初始化OnCreate时，使用[getRawFd](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-resource-manager#getrawfd9)和[fs.write](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-file-fs#fswrite)实现copyDBToSandbox方法将rawfile目录下的db文件拷贝至应用沙箱。

   ```
   // EntryAbility.ets
   onCreate(want: Want, launchParam: AbilityConstant.LaunchParam): void {
     // ...
     AppStorage.setOrCreate<Context>('context', this.context);
     getDBOperatorManager().copyDBToSandbox(this.context, 'book.db');
   }
   ```
2. 在页面加载aboutToAppear时，使用[relationalStore.getRdbStore](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-data-relationalstore-f#relationalstoregetrdbstore)实现readBookList接口通过RdbStore读取沙箱下db数据库文件，实例化Book对象，创建BookArray，存入读取到的数据。

   ```
   // PresetBookPage.ets
   aboutToAppear(): void {
     // ...
     getDBOperatorManager()
       .readBookList(this.context, 3)
       .then(bookArray => {
         this.data.reloadData(bookArray);
         this.filterVipData(bookArray);
       });
   }
   ```
3. 进行页面渲染，通过布局组件，渲染刷新杂志文章页面。

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
