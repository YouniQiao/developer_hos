## 场景介绍

聊天消息表情评论是社交通讯类应用的高频使用场景之一，如用户可长按消息触发表情发送弹窗，点击某一表情后，该表情作为评论出现在消息下方。

本示例基于文本组件的[bindSelectionMenu](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-text#bindselectionmenu11)和[selection](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-text#selection11)属性实现文本长按全选功能，基于[openCustomDialog](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-promptaction#opencustomdialog12)和[openBindSheet](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-uicontext#openbindsheet12)实现包括文本选择弹窗、表情弹窗与半模态窗口在内的自定义弹窗，结合使用[Emitter](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-emitter)传送数据实现表情评论功能。

## 实现思路

1. 定义全局自定义构建函数，实现表情弹窗、半模态弹窗、文本选择弹窗样式。

   ```
   // 构建表情弹窗样式
   @Builder
   export function buildEmojiDialog(params: EmojiParams) {
     // ...
   }
   // 构建半模态弹窗样式
   @Builder
   export function buildMessageBindSheet(params: MessageParams) {
     // ...
   }
   // 构建文本选择弹窗样式
   @Builder
   export function buildCustomTextMenu(positionX: number) {
     // ...
   }
   ```
2. 通过文本组件的[bindSelectionMenu](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-text#bindselectionmenu11)属性启用文本长按全选功能，并触发文本选择弹窗。

   ```
   Text(this.messageModel?.content)
     .bindSelectionMenu(TextSpanType.DEFAULT, this.CustomMenuBuilder(), TextResponseType.LONG_PRESS, {
       onAppear: () => {
         this.textState.start = 0;
         this.textState.end = this.messageModel.content.length;
         // ...
         PromptActionClass.openTextMenu();
       }
     })
   ```
3. 使用[Emitter](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-emitter)分发弹窗点击事件数据。

   ```
   static sendDialogOnClickEvent(value: ItemOnClickEventParam) {
     // ...
     emitter.emit(event, eventData);
   }
   static sendBindSheetOnClickEvent(value: number) {
     // ...
     emitter.emit(event, eventData);
   }
   ```
4. 页面获取数据后调用自定义方法更新节点参数。

   ```
   emitter.on(eventDialogCloseBtnOnclick, (eventData: emitter.EventData) => {
     // 判断接收数据非空
     // ...
     this.dialogOnClick(eventData.data.value)
   })
   emitter.on(eventEmojiOnclick, (eventData: emitter.EventData) => {
     // 判断接收数据非空
     // ...
     this.bindSheetItemOnClick(eventData.data.value)
   })
   ```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
