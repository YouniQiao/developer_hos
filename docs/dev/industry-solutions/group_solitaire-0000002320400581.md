## 场景介绍

群接龙是社交通讯类应用中的典型场景之一，如用户在群内统计群成员信息时，可发起群接龙进行统计，提高统计效率。

本示例通过数组和自定义对象的多层嵌套实现接龙信息的收集和展示功能。

## 实现思路

1. 接龙发起人创建接龙数据，并将此数据的唯一标识BubbleID传入数据处理中心。
2. 数据处理中心创建第一次接龙，完成接龙后将数据保存，并根据BubbleID传回数据进行展示。
3. 接龙参与人参与接龙，获取当前接龙数据并接龙，完成接龙后将最新数据保存，并传回数据进行展示。

```
// 创建新接龙对象，并跳转至接龙页面
let newSolitaire: Bubble = new Bubble();
this.navPathStack.pushPathByName('solitairePage', newSolitaire.BubbleID);
// 根据ID找到传入对象
this.selectId = JSON.stringify(this.navPathStack.getParamByName('solitairePage')[0]);
if (this.selectId === '"' + this.rightData[index].BubbleID + '"') {
  // ...
}
// 更新聊天框展示数据
this.rightData[this.indexBubbleNumber].solitaire.solitaireArray = deepCopy(this.showSolitaire);
```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
