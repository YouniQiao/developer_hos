## 场景介绍

基于rcp实现文件下载暂停与恢复是实用工具类应用的高频使用场景之一。

本示例基于[rcp](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/remote-communication-rcp)实现文件下载时的暂停与恢复功能，通过[@ohos.file.fs](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-file-fs)将下载文件数据保存至本地文件。

## 实现思路

1. 在组件加载时对rcp请求设置、下载文件大小、本地保存的文件位置进行初始化。

   ```
   // 初始化rcp请求相关设置
   this.initRcpConfig();
   // 请求获取当前待下载文件的大小
   this.getFileTotalSize();
   // 获取保存的本地文件路径
   let lastSplitIndex = this.downloadFile.url.lastIndexOf(CommonConstants.FILE_SEPARATOR);
   let fileName = this.downloadFile.url.substring(lastSplitIndex + 1);
   this.fileDir =
     this.getUIContext().getHostContext()?.filesDir + CommonConstants.FILE_SEPARATOR + this.downloadFile.fileName +
     CommonConstants.FILE_NAME_SEPARATOR + fileName;
   ```
2. 初始化rcp请求设置时，为请求设置相应的暂停策略以及数据处理回调函数，以处理下载数据以及更新下载进度，下载数据通过[fs.writeSync](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-file-fs#fswritesync)写入本地文件中保存。

   ```
   this.session = rcp.createSession();
   // 设置接收暂停策略，这里设置的大小是rcp框架所能缓存的接收数据的最大值
   let receivePolicy: rcp.ReceivingPauseByCache = {
     kind: 'cacheSize',
     size: CommonConstants.SYS_CACHE_LIMIT
   }
   let pausePolicy: rcp.PausePolicy = {
     receiving: receivePolicy
   }
   this.request = new rcp.Request(this.downloadFile.url, 'GET');
   this.request.configuration = {
     transfer: {
       pausePolicy: pausePolicy,
       timeout: { connectMs: CommonConstants.CONNECT_TIMEOUT, transferMs: CommonConstants.TRANSFER_TIMEOUT }
     }
     // 其他设置
   }
   // 设置处理数据的回调
   this.request.destination = (incomingData: ArrayBuffer) => {
     this.lastTransferPosition += incomingData.byteLength;
     this.downloadFile.progress =
       Math.round(this.lastTransferPosition / this.totalSize * CommonConstants.HUNDRED_PERCENT);
     this.cacheSize += incomingData.byteLength;
     this.bufferQueue.add(incomingData);
     return this.cacheSize >= CommonConstants.APP_CACHE_LIMIT ? new Promise((resolve, reject) => {
       // 等到数据累积一定量后再写入，通过定时来延长任务处理时间
       setTimeout(() => {
         // 将累积的数据写入本地文件保存
         this.writeFile();
         resolve();
       }, CommonConstants.DELAY_TIME)
     }) : new Promise((resolve) => {
       resolve();
     });
   };
   ```
3. 点击开始下载，更新任务状态和图标内容，并通过[session.fetch](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/remote-communication-rcp#section20139131372817)发出下载请求。

   ```
   // 更新任务状态和显示图标
   this.downloadFile.state = DownloadState.START;
   this.itemIcon = this.getItemIcon();
   // 发出请求并获取到响应
   let response = await this.session?.fetch(this.request);
   this.complete = true;
   // 处理响应结果，数据已经在回调中处理，此处无需处理数据
   ```
4. 下载过程中，点击暂停图标，更新任务状态和显示图标，并通过[session.cancel](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/remote-communication-rcp#section14571174220320)取消当前下载任务。

   ```
   this.downloadFile.state = DownloadState.PAUSED;
   this.hasPaused = true;
   this.itemIcon = this.getItemIcon();
   // 取消当前下载请求
   this.session?.cancel(this.request);
   this.complete = true;
   ```
5. 暂停后点击开始图标，根据上一次暂停时记录的下载位置重新发起下载请求，进行断点续传。

   ```
   this.downloadFile.state = DownloadState.START;
   this.itemIcon = this.getItemIcon();
   // 设置续传起始位置为上次暂停时下载的位置
   this.request.transferRange = { from: this.lastTransferPosition };
   this.complete = false;
   let response = await this.session?.fetch(this.request);
   this.complete = true;
   ```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
