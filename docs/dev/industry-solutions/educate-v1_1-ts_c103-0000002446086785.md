## 问题现象

下载应用后首次打开，会弹出通知授权弹窗，但是还没来得及点击“允许”或“不允许”，弹窗就自动消失了。

## 背景知识

* [notificationManager.requestEnableNotification](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-notificationmanager#notificationmanagerrequestenablenotification10-1)：应用需要获取用户授权才能发送通知。在通知发布前调用该接口，可以拉起通知授权弹窗，让用户选择是否允许发送通知。
* [层级页面](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念)：层级页面是焦点框架中特定容器组件的统称，涵盖Page、Dialog、SheetPage、ModalPage、Menu、Popup、NavBar、NavDestination等。在一个应用程序中，任何时候都至少存在一个层级页面组件，并且该组件会持有当前焦点。当该层级页面关闭或不再可见时，焦点会自动转移到下一个可用的层级页面组件上，确保用户交互的连贯性和一致性。

## 问题定位

1. 首先排查调用通知授权弹窗接口时机是否正确。

   requestEnableNotification应当在应用界面加载完成后（即调用loadContent成功），方可调用该接口。通过搜索"onCreate"、"onWindowStageCreate"、"onForeground"、"loadContent"、"requestEnableNotification"关键词查看日志，发现调用通知授权弹窗是在loadContent之后调用的。

3. 尝试基于这个思路进行问题复现，发现页面跳转确实会让授权弹窗失去焦点并自动消失。

   ```
   // EntryAbility.ets
   onWindowStageCreate(windowStage: window.WindowStage): void {
     windowStage.loadContent('pages/PageOne', (err) => {
       if (err.code) {
         console.error(`Failed to load the content, code is ${err.code}.`);
         return;
       }
       // 拉起通知授权弹窗
       notificationManager.requestEnableNotification(this.context).then(() => {
         console.info('requestEnableNotification success');
       }).catch((err: BusinessError) => {
         console.error(`requestEnableNotification failed, code is ${err.code}, message is ${err.message}`);
       });
       console.info('Succeeded in loading the content.');
     });
   }

   // PageOne.ets
   aboutToAppear() {
     setTimeout(() => {
       router.pushUrl({ url: 'pages/PageTwo' });
     }, 500)
   }
   ```

## 分析结论

由于在应用启动到跳转到首页的过程中，申请通知授权弹窗时机和页面跳转产生冲突，页面多次跳转使出现的弹窗因失去焦点而消失关闭，导致了弹窗出现后很快自动消失的问题。

## 修改建议

* 如果应用在启动后会涉及多个页面的自动跳转，建议不要在onWindowStageCreate中拉起通知授权弹窗，而是在最后一个不会再继续往下跳转的页面中拉起弹窗。
* 或者在loadContent加载的页面中，调用requestEnableNotification接口，然后在回调返回结果中控制是否授权的页面跳转逻辑。

```
import { common } from '@kit.AbilityKit';
import { BusinessError } from '@kit.BasicServicesKit';
import { notificationManager } from '@kit.NotificationKit';

@Entry
@Component
struct PageOne {
  @Provide('pageInfos') pageInfos: NavPathStack = new NavPathStack();
  context: common.UIAbilityContext = this.getUIContext().getHostContext() as common.UIAbilityContext;

  aboutToAppear() {
    // 拉起通知授权弹窗
    notificationManager.requestEnableNotification(this.context).then(() => {
      console.info('requestEnableNotification success');
      this.pageInfos.pushPathByName('PageTwo', null);
    }).catch((err: BusinessError) => {
      console.error(`requestEnableNotification failed, code is ${err.code}, message is ${err.message}`);
    });
  }

  @Builder
  pageMap(name: string) {
    if (name === 'PageTwo') {
      PageTwo();
    }
  }

  build() {
    Column() {
      Navigation(this.pageInfos) {
        Text('PageOne')
          .fontSize(30)
          .margin(30);
      }
      .navDestination(this.pageMap);
    }.width('100%');
  }
}

@Component
struct PageTwo {
  @Consume('pageInfos') pageInfos: NavPathStack;

  build() {
    Column() {
      NavDestination() {
        Text('PageTwo')
          .fontSize(30)
          .margin(30);
      };
    }.width('100%');
  }
}
```
