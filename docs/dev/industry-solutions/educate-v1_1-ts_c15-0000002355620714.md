## 问题现象

应用在使用华为一键登录时无法跳转至登录成功的页面，页面无反应。

## 背景知识

1. [华为账号登录](https://developer.huawei.com/consumer/cn/doc/design-guides/id-0000001880001344#section41792374210)提供两种组件：
   * [华为账号一键登录](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-phone-unionid-login)：华为账号一键登录为[BUTTON](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-component-manager#section19823216112610)类型，可以获取到用户的手机号和UnionID/OpenID。
   * [华为账号登录](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-unionid-login)：华为账号登录为[ICON](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-component-manager#section19823216112610)类型。可以获取到UnionID/OpenID。开发者可以根据需求自行选择使用哪种登录方式。
2. 在使用华为账号一键登录时kit会打印日志关键字：quickLoginWithHuaweiID，如果有日志quickLoginWithHuaweiID success打印则说明调用成功，排除账号问题。华为账号登录的关键字为LoginWith，本文主要介绍华为账号一键登录时的一些场景。

   ```
   I C05320/huawei_id_svc/HWID_SA_055: DealOplogEvent enter
   I C04707/huawei_id_svc/HaInnerSA: OnEvent(hwid_100, 0, HMS_API_CALLED)
   I A04705/com.example.hm/LoginWithHuaweiIDButton: quickLoginWithHuaweiID success
   ```
3. 若没有上述日志打印，调用失败则根据文档查找错误码，对应进行处理，具体错误码可参考：[AuthenticationErrorCode](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-authentication#section138902217187)。

## 问题定位

1. 搜索日志关键字quickLoginWithHuaweiID success，查看日志中是否有调用成功。
2. 没有调用成功的话需要根据背景知识3进行排查。
3. 调用成功后依然无法登录则需要检查代码逻辑方面的问题。

## 分析结论

### 场景一

```
OnResult:514# NapiExecuteRequestCallback OnResult, errCode: 1001500001
```

1. 如果遇到1001500001报错，原因是因为指纹证书校验失败，可能原因如下。client\_id未配置或配置的值不正确。
2. 应用的指纹证书未配置或配置错误。
3. Debug安装包和Release安装包的签名存在差异。

### 场景二

当日志中有1001502014报错时表示应用未申请scopes或permissions权限。

```
Failed to request authorization.Code:1001502014,message:The app does not have the required scopes or permissions.no scope permission
```

1. 需要开发者查看是否申请了scopes或permissions权限。
2. 检查代码中权限范围申请authRequest.scopes是否准确。若仅仅接入华为账号一键登录而其中携带了其他权限标识如authRequest.scopes = ['quickLoginAnonymousPhone', 'phone']而未申请对应权限，也会造成此错误。

### 场景三

如果提示出现如下报错为参数错误，可以看下params. style设置为BUTTON类样式还是ICON类样式，ICON类是华为账号登录按钮的样式，不是华为一键登录按钮的样式，用户在使用华为账号一键登录时错误的使用了ICON类样式。参考背景知识1。

```
[LOGIN_WITH_HUAWEI_ID_WIDGET] 华为一键登录失败: {"code":401,"message":"Parameter error.Only supports plain
text buttons."}
```

### 场景四

1. 首次使用华为账号一键登录可以正常登录，注销之后登录异常。搜索关键字发现一键登录调用成功。

   ```
   quickLoginWithHuaweiID success
   ```
2. 在排查日志的过程中发现服务器返回了500错误代码，表示服务器端发生了未预料到的问题。说明开发者对一些特殊的情况没有做处理，比如说账号注销之后是否可以登录，后续需要开发者进行排查自己的服务器。

### 场景六

如在调用loginClick接口时使用的codeStr是通过[AuthorizationWithHuaweiIDResponse](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-authentication#section10418450161620)获取的，并不是[LoginWithHuaweiIDButton](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-huawei-id-button#section1624716107193)组件获取的，会导致服务端获取不到手机号，无法与应用系统数据库关联，最后登录失败。

登录失败的原因是Authorization Code值需要通过LoginWithHuaweiIDButton组件获取。

### 场景七

若出现如下报错：请求失败，请检查网络ErrCode:4002。则需要查看是否是自身设计的错误码，含义为JSON解析异常。

## 修改建议

### 场景一

1. 检查module type为entry的模块下module.json5中的client\_id是否配置或配置的值是否正确，请参考[配置Client ID](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-client-id)。
2. 检查AGC上应用的指纹证书，详情请见[配置公钥指纹](https://developer.huawei.com/consumer/cn/doc/app/agc-help-cert-fingerprint-0000002278002933)。
3. 更新app.json5中的versionCode使本地缓存失效后再进行重试。

### 场景二

1. 参考[申请scope权限](https://developer.huawei.com/consumer/cn/doc/atomic-guides/account-guide-atomic-permissions)申请权限。
2. 若只是接入华为账号一键登录，注意authRequest.scopes = ['quickLoginAnonymousPhone']即可。

### 场景三

参考背景知识1正确使用style类型。

### 场景四

手机号调用成功依然无法登录为需求实现异常，开发时应考虑各种场景，包括是否需要支持账号注销，账号注销后能否正常登录等，无法登录时应弹出对应的提示窗。

### 场景五

根据应用业务逻辑，开放华为账号信息作为登录来源，正常登录，并跳转相应信息页面（注册信息补充页面或者主要业务页面）。

### 场景六

使用LoginWithHuaweiIDButton组件中获取Authorization Code，具体实现可参考：[LoginWithHuaweiIDButton (华为账号Button登录组件)](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-huawei-id-button)

### 场景七

根据应用业务逻辑修改错误码指出的错误，检查JSON格式以确保JSON字符串符合标准格式。

## 常见FAQ

Q：一键登录服务端有开发示例吗？

A：可以参考官网提供的[服务端开发整体流程](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-phone-unionid-login#section286103315177)，核心为调用接口获取AccessToken，然后通过AccessToken获取一键登录手机号。可参考[示例代码](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-obtain-user-token#section14184218174113)。

Q：loginComponentManager中获取的Authorization Code和通过LoginWithHuaweiIDButton组件拿到authorizationCode有什么区别？

A：没有区别，LoginWithHuaweiIDButton组件和loginComponentManager中都需要通过Authorization Code获取AccessToken，这部分业务逻辑相同。

Q：华为一键登录接口成功：quickLoginWithHuaweiID success，但服务器返回了RespCode:600是什么原因？

A：600错误码指的是服务器端没有返回响应头部，只返回实体内容，需要检查服务器返回相关的响应策略。
