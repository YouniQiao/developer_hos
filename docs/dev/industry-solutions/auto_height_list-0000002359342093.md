## 场景介绍

首页菜单横向滚动高度自适应是购物比价类应用的高频使用场景之一。

本示例基于[List](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-container-list)实现不同高度菜单列表滑动时，自适应高度，且下方列表跟随菜单高度改变而移动。

## 实现思路

1. 使用[onAreaChange](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-component-area-change-event#onareachange)监听容器区域变化，获取单行高度，通过单行高度、行数、行间距计算容器最大高度与最小高度。

   ```
   // 监听容器区域变化事件
   .onAreaChange((_, n) => {
     if (!this.containerWidth) {
       this.containerWidth = Number(n.width)
       this.containerHeight = Number(n.height)
       // 计算容器最大高度：总高度 = (单行高度 * 行数) + (行间距 * (行数-1))
       this.containerMaxHeight =
         (this.containerHeight) * this.lastPageLines + 14 * (this.lastPageLines - 1)
       // 设置容器最小高度：单行高度
       this.containerMinHeight = this.containerHeight
   })
   ```
2. 滑动时更新容器高度，使用[ListScroller](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-container-list#listscroller11)计算偏移量，实现动态高度变化效果。

   ```
   // 计算横向滚动偏移量总和：当前滚动位置 + 新增偏移量 - 容器宽度*(分页数-2)
   let offsetX = this.listScroller.currentOffset().xOffset + x - this.containerWidth * (this.pageSize - 2)
   // 当横向偏移量为正时（超出基线位置），触发高度动态变化
   if (offsetX > 0) {
     let h = offsetX / this.containerWidth * (this.containerMaxHeight - this.containerMinHeight)
     // 根据计算结果更新容器高度（当前高度 = 最小高度 + 动态变化高度）
     this.containerHeight = this.containerMinHeight + h
   } else {
     this.containerHeight = this.containerMinHeight
   }
   ```

## 约束与限制

* 本示例支持API Version 17 Release及以上版本。
* 本示例支持HarmonyOS 5.0.5 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 5.0.5 Release及以上版本进行编译运行。
