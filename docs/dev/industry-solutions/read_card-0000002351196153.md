## 场景介绍

本示例通过[updateForm](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-app-form-formprovider#formproviderupdateform)接口实现应用向卡片同步信息，通过[router事件](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-ui-widget-event-router)实现卡片跳转到应用对应界面。用户点击阅读记录卡片，可查看上次阅读的文章或书籍。

## 实现思路

1. 动态卡片提供方应用通过[updateForm](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-app-form-formprovider#formproviderupdateform)接口向指定卡片同步数据，实现卡片主动刷新。

   ```
   // formId为请求更新的卡片标识
   formProvider.updateForm(formId, formMsg).then(() => {
     // ...
   }).catch((error: Error) => {
     // ...
   });
   ```

2. 卡片通过[router事件](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-ui-widget-event-router)传入对应参数，实现快速拉起动态卡片提供方应用的指定页面。

   ```
   postCardAction(this, {
     action: 'router',
     abilityName: 'EntryAbility',
     params: {
       formId: this.formId,
       targetChapter: this.bookChapter // 书籍章节
     }
   });
   ```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
