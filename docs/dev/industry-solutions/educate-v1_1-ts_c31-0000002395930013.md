## 问题现象

使用手机验证码登录账号后，应用闪退，在平板横屏时出现，竖屏正常。

## 背景知识

* [JS Crash（进程崩溃）检测](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/jscrash-guidelines)：Error是最基本的错误类型，其他的错误类型都继承自该类型。Error对象主要有两个重要属性message和name分别表示错误信息和错误名称。程序运行过程中抛出的异常一般都有具体的类型，Error类型一般都是开发人员自己抛出的异常。
* [150002 祖先节点无法获得焦点](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/errorcode-focus#section150002-祖先节点无法获得焦点)：当前节点对应的祖先节点中存在无法获焦节点。
* [onAreaChange](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-component-area-change-event#onareachange)：组件区域变化时触发该回调。仅会响应由布局变化所导致的组件大小、位置发生变化时的回调。

5. 对比竖屏横屏的日志可以发现，在横屏时会触发组件区域变化事件OccupiedAreaChange，随后应用会手动设置焦点。排查应用是否监听了组件区域变化事件，是否在[onAreaChange](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-component-area-change-event#onareachange)回调函数中手动设置了焦点。

## 分析结论

应用在手动设置焦点时，祖先节点不可获焦导致应用闪退。

## 修改建议

应用在首次组件区域变化时，手动设置焦点，之后当组件区域变化时，不再设置焦点。

```
@Entry
@Component
struct FocusPage {
  isFirst: boolean = true;

  build() {
    Column({ space: 15 }) {
      Text('控制仅在初次进入时获焦')
      TextInput()
        .id('testTextInput')
        .onAreaChange(() => {
          if (this.isFirst) {
            console.info('requestFocus on testTextInput');
            this.getUIContext().getFocusController().requestFocus('testTextInput');
            this.isFirst = false;
          }
        })
        .margin({
          right: 20,
          left: 20
        })
    }
    .width('100%')
    .height('100%')
    .justifyContent(FlexAlign.Center)
  }
}
```
