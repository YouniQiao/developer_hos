## 问题现象

应用内跳转页面，一直显示加载中，或长时间停留未跳转，点击或侧滑操作都无响应。

## 背景知识

* 用户在使用应用时，如果出现点击无反应或应用无响应等情况，并且持续时间超过一定限制，就会被定义为应用无响应，详情参考[AppFreeze（应用冻屏）检测](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/appfreeze-guidelines)。
* AppFreeze日志规格说明可以参考[日志规格](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/appfreeze-guidelines#日志规格)。
* 应用冷启动流程大致可分为以下部分：
  1. 应用进程创建、初始化：该阶段主要是系统完成应用进程的创建以及初始化的过程，解码启动页图标。
  2. 启动应用、Ability：该阶段主要是加载资源，创建和初始化Application和Ability对象，加载依赖模块。
  3. Ability和AbilityStage生命周期：该阶段主要执行Ability和AbilityStage启动生命周期的回调。
  4. 加载绘制首页：该阶段主要是加载首页内容、测量布局、刷新组件并绘制显示。
* [Network Kit（网络服务）](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/network-kit)主要提供的功能有http数据请求、WebSocket连接、Socket连接、网络连接管理和MDNS管理。日志http\_exec相关字段说明：
  + size：上传或者下载的数据大小（单位：byte）。
  + dns：域名解析耗时（单位：ms）。
  + connect：TCP握手耗时（单位：ms）。
  + tls：tls握手耗时（单位：ms）。
  + firstSend：开始发送数据时与发起请求的时间间隔（单位：ms）。
  + firstRecv：第一次收到服务器回复与发起请求的时间间隔（单位：ms）。
  + total：总耗时（单位：ms）。
  + redirect：表示重定向操作所花费的时间（单位：ms）。
  + errCode：表示错误代码，通常用于标识请求或响应中出现的问题。
  + RespCode：表示http响应的状态码。
  + httpVer：表示使用的http协议版本。
  + method：表示http请求的方法。
* [HiSmartPerf](https://developer.huawei.com/consumer/cn/doc/AppGallery-connect-Guides/smartperf-tool-0000001873208929)：无需ROOT设备，即可准确、高效地采集到应用运行时的CPU、GPU等性能数据，了解应用的性能状况。

## 场景一

### 问题定位

1. 当faultlogger目录下没有卡死日志生成时，可以结合trace进行分析。
2. 使用HiSmartPerf工具抓取该过程的trace信息，查看应用主线程，排查卡死阶段有没有耗时特别长的函数。

   ![](./img/2e3746cd.png "点击放大")
3. 从trace日志中看出，OnBackground函数执行时间超过了20s。

### 分析结论

函数执行时间过长导致应用卡死。

### 修改建议

排查函数执行耗时长的原因。

## 场景二

### 问题定位

1. 当faultlogger目录下没有卡死日志生成时，结合trace分析，也没有耗时特别长的函数，需要排查是否是页面跳转逻辑问题导致页面卡在启动页。
2. 结合代码分析页面跳转逻辑。

### 分析结论

存在页面跳转逻辑问题导致应用卡死在启动页。

### 修改建议

修复页面跳转逻辑。

## 场景三
