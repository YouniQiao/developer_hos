## 场景介绍

视频画面比例裁剪是拍摄美化类应用的高频使用场景之一，如视频编辑时可选择裁剪比例（9：16、16：9、1：1、4：3、3：4），并拖动边框调节视频显示区域，将视频播放画面裁剪为最佳比例。

本示例通过创建[AVPlayer](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/video-playback)类播放视频，使用[MP4Parser](https://ohpm.openharmony.cn/#/cn/detail/@ohos%2Fmp4parser)裁剪视频，最后将裁剪完的视频保存到相册。

## 实现思路

1. 创建[AVPlayer](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/video-playback)类，调用[media.AVPlayer.play()](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-media-avplayer#play9)进行播放。

   ```
   // 创建avPlayer实例对象
   let avPlayer: media.AVPlayer = await media.createAVPlayer();
   // 创建状态机变化回调函数
   this.setAVPlayerCallback(avPlayer);
   let fdPath = 'fd://';
   // 通过UIAbilityContext获取沙箱地址filesDir，以Stage模型为例
   // 打开相应的资源文件地址获取fd，并为url赋值触发initialized状态机上报
   try {
     let file = fileIo.openSync(sandUrl, fileIo.OpenMode.CREATE | fileIo.OpenMode.READ_WRITE);
     fdPath = fdPath + '' + file.fd;
     this.isSeek = true; // 支持seek操作
     avPlayer.url = fdPath;
   } catch (e) {
     hilog.error(DOMAIN, 'AVPlayerDemo', `${e}`);
   }
   ```
2. 使用[MP4Parser](https://ohpm.openharmony.cn/#/cn/detail/@ohos%2Fmp4parser)裁剪视频。

   ```
   MP4Parser.ffmpegCmd(`ffmpeg -i ${pathDir +'/flower.mp4'} -vf crop=${this.getUIContext().vp2px(this.dynamicWidth)}:${this.getUIContext().vp2px(this.dynamicHeight)}:${this.getUIContext().vp2px(this.left)}:${this.getUIContext().vp2px(this.top)} ${pathDir +'/result/flowercropped.mp4'}`, callBack);
   ```
3. 保存视频到相册。

   ```
   // 基于弹窗授权的方式获取媒体库的目标uri
   let desFileUris: Array<string> = await phAccessHelper.showAssetsCreationDialog(srcFileUris, photoCreationConfigs);
   // 将来源于应用沙箱的照片内容写入媒体库的目标uri
   let desFile: fileIo.File = await fileIo.open(desFileUris[0], fileIo.OpenMode.WRITE_ONLY);
   let srcFile: fileIo.File = await fileIo.open(srcFileUri, fileIo.OpenMode.READ_ONLY);
   await fileIo.copyFile(srcFile.fd, desFile.fd);
   fileIo.closeSync(srcFile);
   fileIo.closeSync(desFile);
   ```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
