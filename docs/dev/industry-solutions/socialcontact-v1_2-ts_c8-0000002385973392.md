## 问题现象

点击进入聊天界面进行信息回复，发送后返回消息列表，最后一条消息显示的还是对方的信息，刚刚回复的信息没有刷新显示。

## 背景知识

* [List组件](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-layout-development-create-list)常被用于进行列表类功能开发，如消息列表、商品列表等。
* [Emitter](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/itc-with-emitter)是一种作用在进程内的事件处理机制，为应用程序提供订阅事件、发布事件、取消事件订阅的能力。可以用于实现线程间通信功能。

3. 排查代码，查看发送按钮的[onClick事件](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-events-click)中是否有关通知刷新数据的操作。

## 分析结论

发送按钮的onClick事件回调内只进行了数据传输，没有请求数据刷新的操作，导致消息发送成功却未刷新消息列表显示的最后一条消息内容。

## 修改建议

在发送按钮的onClick事件回调中添加请求数据刷新的操作，使用Emitter通知消息列表刷新，确保每次发送都可以实时同步。

1. 导入线程通讯模块。

   ```
   import { emitter, Callback } from '@kit.BasicServicesKit';
   ```
2. 在消息接收页面实现数据的监听和展示。

   ```
   @Sendable
   class Sample {
     constructor(inputMessage: string) {
       this.count = inputMessage;
     }

     printCount(): string {
       return this.count;
     }

     count: string;
   }

   @Entry
   @Component
   struct ShowMessage {
     @Provide('pageInfos') pageInfos: NavPathStack = new NavPathStack();
     @State message: string | undefined = 'Hello!';
     event: emitter.InnerEvent = {
       // 定义一个eventId为1的事件。
       eventId: 1
     };
     // 定义一个事件的回调处理函数，当收到对应的事件后执行回调函数
     callback: Callback<emitter.GenericEventData<Sample>> = (eventData: emitter.GenericEventData<Sample>) => {
       this.message = eventData.data?.count;
     };

     @Builder
     pageMap(name: string) {
       if (name === "NavDestinationTitle1") {
         SendMessage();
       }
     }

     aboutToAppear(): void {
       // 收到eventId为1的事件后执行回调函数
       emitter.on(this.event, this.callback);
     }

     build() {
       Column() {
         Navigation(this.pageInfos) {
           Column({ space: 20 }) {
             Text(this.message) // 展示最后一条消息

             Button('跳转')
               .onClick(() => {
                 this.pageInfos.pushPath({ name: 'NavDestinationTitle1' }); // 点击跳转聊天界面
               })
           }
           .width('100%')
           .height('100%')
           .justifyContent(FlexAlign.Center)
         }
         .navDestination(this.pageMap)
       }
       .height('100%')
       .width('100%')
     }
   }
   ```
3. 在消息发送页面中实现数据的变更并通知消息接收页面更新展示。

   ```
   @Component
   export struct SendMessage {
     @Consume('pageInfos') pageInfos: NavPathStack;
     @State inputMessage: string = '';
     event: emitter.InnerEvent = {
       eventId: 1
     };

     build() {
       NavDestination() {
         Column({ space: 20 }) {
           TextInput()
             .width('90%')
             .onChange((value: string) => {
               this.inputMessage = value; // 记录输入框的信息
             })

           Button('发送')
             .onClick(() => {
               let eventData: emitter.GenericEventData<Sample> = {
                 data: new Sample(this.inputMessage)
               };
               emitter.emit(this.event, eventData); // 点击发送后刷新最后一条数据
             })
         }
         .width('100%')
         .height('100%')
         .justifyContent(FlexAlign.Center)
       }
       .hideBackButton(true)
       .onBackPressed(() => {
         this.pageInfos.pop(); // 弹出路由栈栈顶元素
         return true;
       })
     }
   }
   ```
