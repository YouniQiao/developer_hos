## 问题现象

* 华为账号登录报错，常见错误码是1001500001，主要报错信息如下：

  ```
  Failed to auth, errorCode=1001500001, errorMsg=Failed to check the fingerprint of the app bundle.Fingerprint verification error.
  ```
* 华为账号一键登录获取手机号码，常见错误码是1001500003，主要报错信息如下：

  ```
  Failed to login, errorCode=1001500003, errorMessage is The scopes or permissions are not supported.
  ```
* 错误码1001502014，主要报错信息如下：

  ```
   Failed to auth. Code: 1001502014, message: The app does not have the required scopes or permissions.no scope permission.
  ```
* 通过“/oauth2/v6/quickLogin/getPhoneNumber”获取手机号码报错，常见错误码是60180007,主要报错信息如下：

  ```
  scope not contain quickLoginMobilePhone.
  ```
* 通过“/oauth2/agent/v1/access-token”[获取用户级Token](https://developer.huawei.com/consumer/cn/doc/FASP-by-Template-develop-References/get-token-user-0000001622722589)报错：

  ```
  sub_error=20001, error_description=missing required parameter: client_id, error=1102.
  ```
* 通过“oauth2/v3/token”[获取用户级凭证](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-obtain-user-token)报错：

  ```
  sub_error=20154，error_description=client_id not match，error=1101.
  ```
* 使用华为账号服务报错，错误码1001502003：

  ```
  1001502003 Invalid input parameter value.clientId or appId is invalid.
  ```

  ```
  1001502003 Invalid input parameter value.
  ```

## 背景知识

华为账号服务提供了[华为账号一键登录](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-phone-unionid-login)、[华为账号登录](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-unionid-login)、[静默登录](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-silent-login)等多种登录方式，其中华为账号一键登录仅支持企业开发者使用，华为账号登录和静默登录既支持企业开发者也支持个人开发者使用。

使用[华为账号服务](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api)遇到其他错误码，可以查看[ArkTS错误码](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-error-code)。

## 解决方案

* **问题一**：错误码1001500001的含义为应用指纹证书验证失败。

  **可能原因**：应用的指纹证书未配置或配置错误（HarmonyOS 6.0后不再要求配置公钥指纹）。

  **检查步骤**：

  1. 登录[AppGallery Connect](https://developer.huawei.com/consumer/cn/service/josp/agc/index.html#/)查看[配置公钥指纹](https://developer.huawei.com/consumer/cn/doc/app/agc-help-cert-fingerprint-0000002278002933)：

     ![](./img/b5f8e887.png)
  2. 使用DevEco Studio打开项目工程，打开File->Project Structure->Project->Signing Configs界面，不能勾选“Automatically generate signature”进行自动签名，需要手动配置签名。可以按照开发准备里面的说明配置：[配置签名信息](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/application-dev-overview#配置签名信息)。

     ![](./img/cb536c35.png)
  3. 查看应用签名证书指纹，安装应用后执行：
     + Windows系统

       ```
       hdc shell bm dump -n 包名 | findstr appId
       ```
     + Mac系统

       ```
       hdc shell bm dump -n 包名 | grep appId
       ```
     + 将“appId”中包名下划线后的字符串拷贝到新文件中（文件名可自行定义，例如SHA.txt）再执行下面的命令行即可得到公钥指纹。检查应用签名证书指纹与AppGallery Connect是否一致。

       ```
       certutil -hashfile F:\\SHA.txt SHA256
       ```
  4. Client ID配置：

     检查module type为entry的模块下module.json5中的client\_id是否配置错误，请参考[配置Client ID](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-client-id)。

     注意：此处client\_id的获取路径为：AppGallery Connect中开发与服务->常规->应用->OAuth 2.0客户端ID（凭据->Client ID）。

     在工程中entry模块的module.json5文件中，新增metadata，配置name为client\_id，value为Client ID的值。

     + 错误示例1：client\_id在工程里面没有配置。
     + 错误示例2：client\_id的value写到string里面（比如：“value”: “$string:clientId”）也会导致授权报错。
     + 错误示例3：client\_id的value里面配置成APP ID，而不是Client ID的值。
     + 错误示例4：module.json5里面配置了多个client\_id。
* **问题二**：错误码1001500003的原因是当前登录华为账号或者设备不支持应用请求的数据。

  **可能原因**:
  + 在获取实名信息、实名信息校验、人脸核身等场景下，用户登录的账号为中国境内（香港特别行政区、澳门特别行政区、中国台湾除外）注册的华为账号，但使用了港澳台通行证进行实名。
  + 在实名信息校验、人脸核身等场景下，PC/2in1设备或前置摄像功能异常的设备不支持传入verifyRealName和verifyFace 2个scope进行授权。
  + 元服务开发过程中，调AuthorizationWithHuaweiIDRequest接口时传了profile这个scope，且supportAtomicService参数未设置为true。
  + 元服务开发过程中，调AuthorizationWithHuaweiIDRequest接口时传了phone这个scope。
  + 在实名信息校验、人脸核身场景，登录的华为账号是儿童账号。
  + Wearable、TV设备申请获取不支持的scope。

  **解决方案**：

  + 提示用户当前华为账号不支持，引导用户更换使用大陆身份证实名的华为账号。
  + 在实名信息校验、人脸核身等场景下，使用其他前置摄像功能正常的设备授权verifyRealName和verifyFace 2个scope。
  + 元服务开发过程中，调[AuthorizationWithHuaweiIDRequest](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-authentication#authorizationwithhuaweiidrequest)接口时传了profile这个scope，并将supportAtomicService参数设置为true。
  + 元服务不能直接调用该接口获取手机号，可参考场景化控件[快速验证手机号Button](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/scenario-fusion-button-getphonenumber)获取。
* **问题三**：错误码1001502014的含义为应用未申请scopes或permissions权限。

  可能原因和解决方法：[AuthorizationWithHuaweiIDRequest](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-authentication#authorizationwithhuaweiidrequest)中scopes配置与已申请的账号权限不匹配，具体参照[开发前提](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-get-phonenumber#开发前提)。详细排查步骤参考[1001502014 应用未申请scopes或permissions权限的可能原因和解决方法](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-faq-2)。
* **问题四**：错误码60180007的含义是通过Authorization Code无法获取到华为账号一键登录手机号。

  **可能原因**:

  + 应用未申请quickLoginMobilePhone权限。
  + 调用服务器接口/oauth2/v6/quickLogin/getPhoneNumber的入参code，不是通过调用华为账号的一键登录组件获取到的。

  **解决方案**：

  + [申请账号权限](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-config-permissions#section144591410155511)，确认应用满足华为账号一键权限申请条件并确认申请已通过。
  + 确认/oauth2/v6/quickLogin/getPhoneNumber需要的入参code是通过华为账号一键登录接口获取的。请参考[客户端开发](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-phone-unionid-login#客户端开发)。
* **问题五**：错误码1101、1102、1103的含义是使用oauth2/v3/token接口时入参错误。

  **可能原因**：

  + 调用服务器接口oauth2/v3/token的入参client\_id为空或不正确。
  + 调用服务器接口oauth2/v3/token的入参grant\_type不符合选值要求（“authorization\_code”、“refresh\_token”、“client\_credentials”）。
  + 其他参数无效或错误。

  **解决方案**：对照[开放接口调用凭证](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-obtain-token)排查对应接口的入参。
* **问题六**：错误码1001502003的含义为输入参数值无效。

  可能原因及解决方案：

  + client\_id未配置或配置错误，参考上文Client ID配置当中的排查方法。
  + client\_id配置value值错误，比如空格等异常字符。
  + 传入的匿名手机号不正确，或是未调用授权API（[AuthorizationWithHuaweiIDRequest](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-authentication#authorizationwithhuaweiidrequest)）获取匿名手机号。
  + 接口传参异常，如调用授权API（[AuthorizationWithHuaweiIDRequest](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/account-api-authentication#authorizationwithhuaweiidrequest)）时scope和permission都未传。
  + 检查应用的签名文件是否配置正确，且需要进行[手动签名](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-signing#section297715173233)，并使用调试证书而非发布证书。
  + 申请的Profile文件不包含APP ID，需要重新申请Profile文件。

## 常见FAQ

Q：AppGallery Connect和应用的证书指纹已经确认一致还是报错？

A：在AGC上面，添加公钥指纹（HarmonyOS API 9及以上）后，指纹有一定时间的缓存，不会立刻生效，建议配置完成后等半小时再尝试。检查手机是否有修改系统时间。

Q：点击《华为账号用户认证协议》无反应。

A：需排查[《华为账号用户认证协议》](https://privacy.consumer.huawei.com/legal/id/authentication-terms.htm?code=CN&language=zh-CN)的链接，目前链接已更改，需在之前的链接后增加&language=zh-CN。

Q：本地使用调试证书可以登录成功，但替换正式证书打包APP后登录失败。

A：发布证书签名的包是不能在本地进行调试安装。利用发布证书及对应的profile所打包的APP，只能用于上架发布，正式包可以通过[邀请测试](https://developer.huawei.com/consumer/cn/doc/app/agc-help-invite-test-0000002270829393)方式验证。

Q：新增HarmonyOS应用，client\_id和client\_secret和原来的不一样，传入的Authorization Code怎么区分是HarmonyOS还是其他平台？

A：服务端处理Authorization Code时，必须使用对应应用的Client ID/Secret组合换取Access Token。此时：

1. 若使用HarmonyOS应用的Client ID/Secret成功换取Token→判定为HarmonyOS来源。
2. 若使用其他平台应用的Client ID/Secret成功换取Token→判定为其他平台来源。

Q：华为账号一键登录获取公钥指纹报错：Failed to login with LoginWithHuaweiIDButton. errorCode=1001500001, errorMsg is Failed to check the fingerprint of the app bundle.Incomplete response information from gateway.

![](./img/393df17f.png "点击放大")

A：首先排查是否企业开发者，个人开发者暂不支持一键登录，再根据当前知识解决方案思路依次排查。

Q：TV设备在调用executeRequest时报错1001500003根据文档排查无法解决问题。

A：华为账号一键登录支持Phone、Tablet、PC/2in1设备。并且从5.1.1(19)版本开始，新增支持TV设备。需排查TV设备软件版本是否高于5.1.1(19)版本。

Q：为什么华为账号一键登录，获取匿名手机号，有些手机可以，有些手机报错误码:1001500001？

A：华为账号一键登录在API>=20以后，不需要配置公钥指纹，而低于API20版本的手机，依旧需要配置公钥指纹。参考[配置签名和指纹](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/account-sign-fingerprints)。

Q：利用手动签名和调试证书成功登录，如果接下来要发布软件如何发布？软件如何打包？其次发布的时候是不是要正式签名？或者说是正式证书？

A：正式上架需要使用发布证书构建出.app包，打包发布可以参照：[发布应用](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-publish-app)。

Q：为什么使用华为账号登录，使用调试证书或者发布证书，只有当前开发者可以正常登录，其他人报错误码:1001500001？

A：如您的应用当前集成的华为相关服务（如华为账号服务）依赖公钥指纹，您需要在应用调试或上架前将公钥指纹配置到AppGallery Connect。调试和发布使用的签名证书不同，因此调试和发布前请分别基于使用的证书配置相应的公钥指纹。发布应用前，需在AppGallery Connect中将调试应用的指纹更新为发布证书指纹。具体操作请参见[配置公钥指纹](https://developer.huawei.com/consumer/cn/doc/app/agc-help-cert-fingerprint-0000002278002933)。

Q：调试证书华为账号一键登录正常，提交上架的版本使用华为账号一键登录失败？

A：调试证书指纹已添加到应用下，发布证书未配置，请登录[AppGallery Connect](https://developer.huawei.com/consumer/cn/service/josp/agc/index.html#/)检查。
