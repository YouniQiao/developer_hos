## 问题现象

页面在向上滑动过程中，页面顶部区域透视出应用其他页面的内容，页面重叠显示。

## 背景知识

设置组件的透明度：[opacity](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-attributes-opacity)。

## 问题定位

1. 检查主页面在向上滑动过程中手机顶部状态栏以及自定义个人信息栏透明度变化情况。
2. 页面在向上缓慢滑动时，手机顶部状态栏以及自定义个人信息栏逐渐浮现出来，直至彻底显示在页面顶端。
3. 手机顶部状态栏以及自定义个人信息栏固定显示后，页面下滑，手机顶部状态栏以及自定义个人信息栏又会逐渐透明，直至彻底消失不见。
4. 根据上述UI状态变化，可以确定手机顶部状态栏以及自定义个人信息栏，透明度是随主页面向上滑动从0到1从而使手机顶部状态栏以及自定义个人信息栏组件逐渐显示出来。反之，透明度从1到0从而使手机顶部状态栏以及自定义个人信息栏组件逐渐透明消失。
5. 在出现或消失过程中，由于手机顶部状态栏以及自定义个人信息栏组件半透明，可以透视出主页面，出现页面重叠显示问题。

## 分析结论

由于手机顶部状态栏以及自定义个人信息栏透明度属性设置错误，导致主页面向上滑动过程中页面会重叠显示问题。

## 修改建议

主页面向上滑动一定距离后，手机顶部状态栏以及自定义个人信息栏透明度直接由0更新为1，而不是由0至1随主页面向上滑动距离渐变。

```
// 页面组件向上滑动距离，动态计算
let l : number = 0
@Component
struct demo {
  build() {
    Row()
      .opacity(l > 10 ? 1 : 0)
  }
}
```
