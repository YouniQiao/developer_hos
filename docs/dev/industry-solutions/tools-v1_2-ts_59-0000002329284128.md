## 场景介绍

图片解码为不同格式类型是实用工具类应用的高频使用场景之一。

本示例主要基于图像像素类[PixelMap](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-image-pixelmap)和[ImageSource](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-image-imagesource)类实现图片解码为不同格式类型。

## 实现思路

1. 使用[ImageReceiver](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-image-imagereceiver)接收预览帧图片并转换为指定格式类型的[PixelMap](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-image-pixelmap)。

   ```
   const dstBufferSize = width * height * 1.5;
   const dstArr = new Uint8Array(dstBufferSize);
   for (let j = 0; j < height * 1.5; j++) {
     const srcBuf = new Uint8Array(imgComponent.byteBuffer, j * stride, width);
     dstArr.set(srcBuf, j * width);
   }
   // 此处可指定将byteBuffer转换成pixelMap的像素格式pixelFormat，默认值为RGBA_8888
   this.pixmap = await image.createPixelMap(dstArr.buffer, {
     size: { height: height, width: width },
     srcPixelFormat: image.PixelMapFormat.NV21
   });
   ```
2. 设置图片解码选项，并将解码成固定类型的图片写入到文件中。

   ```
   // 创建PixelMap，开发者可指定具体的解码格式类型
   let opts: image.DecodingOptions = {
     editable: true, desiredPixelFormat: image.PixelMapFormat.NV21
   };
   imageSource.createPixelMap(opts).then((pixelMap: image.PixelMap) => {
     this.pixelMap = pixelMap;
     // 获取buffer数据
     const readBuffer: ArrayBuffer = new ArrayBuffer(this.pixelMap.getPixelBytesNumber());
     if (this.pixelMap !== undefined) {
       this.pixelMap.readPixelsToBuffer(readBuffer).then(() => {
         // YUV数据写入.yuv文件
         let path1: string = this.getUIContext().getHostContext()?.filesDir + '/image.yuv';
         let file1 = fileIo.openSync(path1, fileIo.OpenMode.READ_WRITE | fileIo.OpenMode.CREATE);
         let opt: WriteOptions = {
           length: readBuffer.byteLength
         };
         fileIo.writeSync(file1.fd, readBuffer, opt);
       });
     }
   });
   ```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
