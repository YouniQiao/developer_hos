## 问题现象

用户使用星闪扫描功能，实际预期的广播设备名应为：GAxxxxxR158xx11，但是系统获取的扫描结果中设备名称只有前9个字GAxxxxxR1，如何解决？

* 配置扫描过滤器的设备名GAxxxxxR158xx11：扫描不到。
* 配置GAxxxxxR1：可以扫到，但扫描结果显示设备名为GAxxxxxR1。
* 在设置界面打开星闪，显示的设备名称也是9个字节。

## 问题定位

以下为星闪广播字段：**01**0101**05**060d0609060c06**03**1d0906070304100006094741xxxxxxxxxx52313538xxxx31310000000000**0b**1f4741xxxxxxxxxx52313538xxxx313100000000000000000000000000000000ff02f0ff。

* 用户在0b中携带的设备名为GAxxxxxR158xx11。然而，除了0x0B设备完整本地名称中携带设备名称外，0x03标准服务数据信息也可以携带设备名称，并且从星闪代码逻辑看来，最终会以服务数据信息中的为准。
* 以下为截取出的广播数据中03的服务数据信息：

  031d09060703041000**0609**4741xxxxxxxxxx52313538xxxx31310000000000。

  ```
  // 通过对如下原始码流拆分：
  01010105060d0609060c06031d090607030410000609 4741xxxxxxxxxx52313538xxxx313100000000000 b1f4741xxxxxxxxxx52313538xxxx313100000000000000000000000000000000ff02f0ff
  广播数据类型： 0x03 标准服务数据信息；
  长度：0x0d  即29；
  服务uuid：0x0609 设备信息管理服务；
  类型：0x07 设备外观信息；
  长度：0x03；
  数据码流：041000；
  类型：0x06 设备本地名称；
  长度：0x09； // 由此看出：设备本地名称长度为9，导致解析后的设备名只有前9个字节。
  数据码流：4741xxxxxxxxxx5231；
  ```

## 分析结论

广播数据中字段填写不对导致。设备信息管理服务中的设备本地名称长度填写为0x09，所以设备名被解析为只有前9个字节。

## 修改建议

开发者检查广播发送方的广播内容，按照对应长度填写。

## 常见FAQ

Q：如何在代码中判断当前设备是否支持星闪？

A：从API19版本开始，星闪底层做了enable验证，如果设备不支持星闪功能，调用星闪API接口都会返回801错误码。

Q：SLE配对后，开关星闪，每次重新连接后显示的已配对设备名称会变化。

A：广播扫到的名称与DIS配置的名称配置一致，可使设备名称不再变化。
