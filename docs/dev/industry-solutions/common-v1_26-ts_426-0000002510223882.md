## 问题现象

应用调试阶段，AGC管理台开通APMS服务后，异常管理界面为什么没有崩溃数据？

## 背景知识

[性能监测](https://developer.huawei.com/consumer/cn/doc/app/agc-help-apms-0000002235870062)（Application Performance Management Service，简称APMS）服务是AppGallery Connect（简称AGC）向开发者提供的一个现网质量监测解决方案。

## 问题定位

1. 判断开通APMS服务功能的应用，与上报崩溃的应用是否是同一个应用，包名是否一致。
2. 应用是否在调试阶段，如果应用在调试阶段，检查应用是否使用[申请调试证书](https://developer.huawei.com/consumer/cn/doc/app/agc-help-debug-cert-0000002283256797)进行打包调试。
3. 检查手机系统版本是否是HarmonyOS 5.0.0 Release及以上版本。
4. 检查日志中是否存在崩溃，通过关键字“report success tags APMS”进行排查。
5. 应用创建完成时间是否未达到6小时。

## 分析结论

1. 需要保持开通APMS应用的包名和应用包名保持一致。应用包名的查询方法，请参考[查看应用信息](https://developer.huawei.com/consumer/cn/doc/app/agc-help-view-app-info-0000002282674569)。
2. 调试阶段，应用需要使用调试证书打包签名，才能正常上报异常。
3. 检查手机系统版本是否是HarmonyOS 5.0.0 Release及以上版本。
4. 通过日志排查是否上报，或手动上报，查看管理台数据。
5. 如果是刚创建的应用，后台需要6小时准备资源，请等待6小时后再前往AppGallery Connect查看应用崩溃情况。后续产生的崩溃数据将可实时在AppGallery Connect查看。

## 修改建议

1. 开通APMS服务的应用包名需要与应用app.json5中bundleName值保持一致。

   ![](./img/7e985e71.png "点击放大")
2. 应用打包签名，需要使用[申请调试证书](https://developer.huawei.com/consumer/cn/doc/app/agc-help-debug-cert-0000002283256797)进行打包签名。
3. 手机系统版本需要是HarmonyOS 5.0.0 Release及以上版本。
4. 在HiLog日志中，搜索关键字“report success tags APMS”，查看是否有上报异常数据。
5. 查看异常数据，请前往[AppGallery Connect](https://developer.huawei.com/consumer/cn/service/josp/agc/index.html)->APMS->异常管理进行查看。
6. 刚创建的应用，后台需要6小时准备资源，请等待6小时后再前往AppGallery Connect查看应用崩溃情况。

## 常见FAQ

Q：HiLog日志无内容的原因是什么？

A：可能原因如下：

1. 首次查看应用崩溃情况需等待6小时后再前往AGC控制台，详情可参考[测试崩溃实现](https://developer.huawei.com/consumer/cn/doc/app/agc-help-apms-crash-test-0000002240442986)。
2. PC端或测试设备未连接网络。

Q：在AGC上开启APMS服务之后，会在本地再跑一遍崩溃代码，跑完之后，搜索“report success tags APMS”，确认请求APMS是否成功。但是在有些时候，“report success tags APMS”搜索不到，是为什么呢？

A：只要有report success, reqId: \*\*\*\*\*, tags(APMS...等，表示成功的意思即可。

Q：已有report success, reqId: \*\*\*\*\*, tags(APMS...等，但异常管理界面为什么还是没有崩溃数据？

A：历史未开通APMS服务的应用需要通过更新应用信息来触发开通条件，需要按照[存量应用如何开通APMS服务](https://developer.huawei.com/consumer/cn/doc/app/agc-help-apms-faq-0000002271373129#section17901136608)操作。

Q：APMS异常管理可以上报自定义信息吗？

A：不支持的。

Q：什么条件的应用需要申请APMS服务？

A：2025年4月9日及之后新创建的HarmonyOS应用无需专门申请，默认开通，可直接使用，入口为：登录AGC平台--开发与服务--选择某一项目后左侧菜单栏即有质量模块下的APMS服务，点击菜单栏即可使用；

2025年4月9日前创建且从未使用过APMS服务的存量应用，可通过刷新【应用信息】的方式触发应用标识刷新，完成服务的开通。

Q：新上架的APP在AGC平台显示的新增用户数是否包含老用户？AGC平台统计的数据每个字段具体含义是什么？

A：新增用户数数据，包含老用户的访问，统计日当日，访问APP去重新增用户数。更多用户指标可参考[查看应用关键指标](https://developer.huawei.com/consumer/cn/doc/app/agc-help-anaiyze-app-keyindex-0000002236332564)。

Q：APMS异常管理下设备数是去重的吗？

A：设备数是去重后的数据。

Q：崩溃次数突然出现异常，是哪些原因导致？

![](./img/66c47a69.png "点击放大")

A：短时间内数据异常，检查数据是否来自同一设备或多个设备，如果大量异常集中在少数设备，当天的数据异常是由于压力测试导致。否则需要根据堆栈排查问题。

Q：为什么设备总数超过用户数？

A：多台设备（包括测试设备）登录同一账号时，APMS默认按设备而非账号统计，会导致设备总数超过用户数。
