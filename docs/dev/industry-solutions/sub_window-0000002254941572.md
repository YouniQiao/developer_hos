## 场景介绍

悬浮窗是影音娱乐类应用的典型场景之一，如用户在浏览应用内不同页面时，需通过悬浮窗查看歌曲信息，控制歌曲的播放、暂停等。

本示例基于子窗口实现应用内悬浮窗，通过[Window](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-window-window)实现悬浮窗的样式切换与拖动、贴边效果。

## 实现思路

1. 通过[WindowStage](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-window-windowstage)创建子窗口，加载自定义页面形成悬浮窗。
2. 使用[Window](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-window-window)定义悬浮窗的拖拽滑动、样式切换、贴边判断方法。
3. 定义switchPlayOrPause方法控制[AVPlayer](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-media-avplayer)的播放或暂停，并绑定悬浮窗点击事件。

```
// 点击歌曲列表创建悬浮窗
this.showSubWindow();

floatingWindow() {
  Column() {}
    .onClick(() => {
      // 悬浮窗展开时控制音频播放与暂停
      switchPlayOrPause();
      // 若悬浮窗已展开则转为折叠态
      switchFloatingWindowStyle();
      // 若悬浮窗为折叠态则定时展开
      this.debounce(() => {
        switchFloatingWindowStyleMove();
      })
    })
    .PanGesture()
      .onActionStart(() => {
        // 拖拽时若悬浮窗已展开则转为折叠态
        switchFloatingWindowStyle();
      })
      .onActionUpdate(() => {
        // 拖拽移动
        dragToMove();
      })
      .onActionEnd(() => {
        // 贴边判断
        edgeDetermination();
      })
}
```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
