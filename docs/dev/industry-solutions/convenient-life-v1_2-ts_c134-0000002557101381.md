## 问题现象

在ASCF元服务开发中，开发者常需要通过web-view组件内嵌外部H5页面以复用现有业务逻辑（如表单录入、问卷调查）。

在此场景下，普遍存在以下交互诉求：

1. **ASCF->H5**：ASCF元服务页面需要将动态数据（如业务类型、用户标识）传递给H5页面，以改变H5的渲染呈现。
2. **H5->ASCF**：H5页面完成业务交互后，需要将收集到的数据（如表单内容）回传给ASCF元服务，并在元服务页面进行展示或下一步业务流转。

## 背景知识

在ASCF框架下，web-view的通信机制与ArkTS环境有显著差异，需特别注意以下三点：

1. **引入SDK**：H5侧必须显式引入ASCF的JS SDK（@atomicservice/ascf-web-sdk），而非ArkTS元服务使用的aseweb-sdk。详见[开发web-view组件](https://developer.huawei.com/consumer/cn/doc/atomic-ascf/develop-web-view)。
2. **通信沙箱与队列机制**：H5页面没有直接调用系统级API（如路由跳转）的权限。H5调用[has.ascfweb.postMessage](https://developer.huawei.com/consumer/cn/doc/atomic-ascf/components-web-view#hasascfwebpostmessage)发送的消息会被暂存到底层的消息队列中。
3. **消息结算时机**：ASCF宿主页面的@message事件不会在H5调用时立刻触发，而是仅在特定生命周期（如H5页面主动调用[has.ascfweb.navigateBack](https://developer.huawei.com/consumer/cn/doc/atomic-ascf/components-web-view#hasascfwebnavigateback)时）将队列中的消息打包为数组，一次性派发给ASCF端。

## 解决方案

实现双向通信的核心思路分为两条单向链路：ASCF->H5依赖URL动态拼接，H5->ASCF依赖消息队列与缓存中转。

1. **ASCF向H5传值**：

   在ASCF页面（如首页）获取所需参数，通过路由将其传递给承载web-view的容器页。容器页接收参数并拼接到H5的URL中。

   1. **首页路由传参**：使用反引号和encodeURIComponent防止特殊字符导致截断。

      ```
      // 在 URL 后面拼接参数传给 webview 页面
      // 使用 encodeURIComponent 防止中文乱码或特殊字符导致解析错误
      const encodedType = encodeURIComponent(this.taskType || '默认');
      has.navigateTo({
        url: `/pages/index/webview?type=${encodedType}`,
        success: () => console.info('跳转 webview 容器页成功')
      });
      ```
   2. **容器页接收并绑定**：通过props接收路由参数，并在onShow生命周期中动态拼接。

      ```
      props: ['type'], // 显式声明以接收路由参数

      data() {
        return {
          // 基础 URL
          baseUrl: '此处输入html的地址',
          // 赋一个合法初始值防止 web-view 瞬间白屏
          dynamicUrl: '此处输入html的地址'
        }
      },

      onShow() {
        console.info('--- webview 容器 onShow 触发 ---');

        // 1. 获取参数
        let rawType = this.type || (this.$page && this.$page.query && this.$page.query.type) || '通用';

        // 2. 对拿到的参数进行解码
        // 因为首页用了 encodeURIComponent，这里先解码还原成中文
        try {
          rawType = decodeURIComponent(rawType);
        } catch (e) {
          console.error('解码失败:', e);
        }

        console.info('容器页成功提取参数: ', rawType);

        // 3. 拼接新的带参数的 URL (拼接到 URL 里时，为了安全再次编码)
        const targetUrl = `${this.baseUrl}?taskType=${encodeURIComponent(rawType)}`;

        // 4. 如果 URL 变了，才重新赋值，防止 web-view 陷入无限刷新死循环
        if (this.dynamicUrl !== targetUrl) {
          this.dynamicUrl = targetUrl;
          console.info('动态更新 H5 地址为: ', this.dynamicUrl);
        }
      },
      ```
   3. **H5侧解析URL**：使用URLSearchParams获取参数并更新DOM。

      ```
      // 页面加载时解析 URL 参数
      window.onload = function () {
          // 使用浏览器的 URLSearchParams API
          const urlParams = new URLSearchParams(window.location.search);
          const taskType = urlParams.get('taskType');

          if (taskType) {
              // 如果拿到了参数，动态修改 DOM
              // decodeURIComponent 会自动将 URL 编码转换回中文
              const decodedType = decodeURIComponent(taskType);
              document.getElementById('dynamicTitle').innerText = ` ${decodedType}信息录入`;
              console.log("成功接收ASCF端传值: " + decodedType);
          }
      };
      ```
2. **H5向ASCF传值**：

   H5侧需引入UMD格式的SDK，利用has.ascfweb.postMessage将数据推入队列，并立即调用has.ascfweb.navigateBack触发ASCF端结算。

   ```
   function submitAndBack() {
       if (typeof has === 'undefined' || !has.ascfweb) {
           return alert("环境未就绪，请在元服务中打开");
       }

       // 1. 获取输入框的值
       const nameValue = document.getElementById('userName').value;
       const dateValue = document.getElementById('bookDate').value;

       // 2. 构造数据 Payload
       const payload = {
           action: 'form_submit',
           data: {
               name: nameValue || '未填写姓名',
               date: dateValue || '未选择日期'
           }
       };

       // 3. 存入消息队列
       has.ascfweb.postMessage({ data: payload });

       // 4. 延迟触发返回（保证消息入队，并销毁当前容器触发宿主结算）
       setTimeout(() => {
           has.ascfweb.navigateBack();
       }, 300);
   }
   ```
3. **ASCF解析消息与缓存流转**：

   容器页在@message中拦截数据，借助[has.setStorageSync](https://developer.huawei.com/consumer/cn/doc/atomic-ascf/apis-data-storage#hassetstoragesync)写入本地缓存，最终在首页的onShow中读取。

   1. **容器页写入缓存**：

      ```
      handleH5Message(e) {
        const messageQueue = e.detail.data;

        if (messageQueue && messageQueue.length > 0) {
          const lastMessage = messageQueue[messageQueue.length - 1];

          if (lastMessage.action === 'form_submit') {
            console.info('接收到 H5 数据: ' + JSON.stringify(lastMessage.data));

            try {
              has.setStorageSync('h5_form_result', JSON.stringify(lastMessage.data));
              console.info('数据已成功写入本地缓存');
            } catch (err) {
              console.error('数据写入缓存失败: ', err);
            }
          }
        }
      }
      ```
   2. **首页读取并清理缓存**：使用防御性解析避免双重字符串化。

      ```
      const res = has.getStorageSync('h5_form_result');

      // 确保 res 存在且不为空字符串
      if (res && res !== '') {
        console.info('原始缓存读取内容:', res);

        // 防止底层双重 stringify
        let resultData = res;
        if (typeof resultData === 'string') {
          resultData = JSON.parse(resultData);
        }
        // 如果解析一次后还是字符串（说明被双重转义了），再解一次
        if (typeof resultData === 'string') {
          resultData = JSON.parse(resultData);
        }

        console.info('最终解析到的对象:', JSON.stringify(resultData));

        if (resultData && resultData.name) {
          // 赋值给 Vue 变量
          this.displayName = resultData.name;
          this.displayDate = resultData.date;

          console.info(`成功赋值 -> 姓名: ${this.displayName}, 日期: ${this.displayDate}`);

          // 防止 ASCF 在部分机型上丢失响应式
          if (typeof this.$forceUpdate === 'function') {
            this.$forceUpdate();
          }

          // 等页面彻底渲染稳定后，再清理缓存，避免 onShow 连击导致的错乱
          setTimeout(() => {
            has.removeStorageSync('h5_form_result');
            console.info('缓存已安全清理');
          }, 1000);
        }

      } else {
        console.debug('暂无回传数据 (缓存为空)');
      }
      ```

## 常见FAQ

Q：H5侧调用has.ascfweb.postMessage后，ASCF端的@message无反应？

A：ASCF采用消息暂存机制。单纯调用postMessage不会触发实时回调，必须在H5侧紧接着调用has.ascfweb.navigateBack()，或者由用户物理点击返回键销毁当前H5页面时，暂存的消息队列才会一并结算并触发宿主的@message。

Q：从容器页返回首页时，遇到输入法软键盘异常弹起怎么办？

A：若首页触发跳转的按钮使用的是<input type="button">标签，底层在页面重新聚焦时会尝试恢复输入法状态。应将按钮标签替换为<div>或<text>，并绑定@click事件。

## 总结

基于ASCF框架的web-view通信，必须严格遵循“宿主-客体”的沙箱隔离原则。ASCF->H5数据流依赖于宿主在初始化时的URL注入；H5->ASCF数据流则依赖于页面的生命周期。掌握“队列存入->触发返回->宿主拦截->缓存中转”这一标准范式，即可应对绝大多数ASCF元服务内嵌H5的复杂表单与交互场景。
