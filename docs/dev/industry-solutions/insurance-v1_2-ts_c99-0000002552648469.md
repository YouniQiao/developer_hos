## 问题现象

使用涉诈剧本检测功能时，当多次调用接口成功后，再次调用接口时会出现报错。

报错信息如下：

code:1012500001 message:Internal error.

## 背景知识

[涉诈剧本检测](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/devicesecurity-fraudriskdetection)：金融支付类应用在用户转账、支付前，通过调用Device Security kit的detectFraudRisk接口，检测用户是否受到欺诈威胁。该接口返回一个风险分，以及涉诈行为的线索，例如，接收到涉诈引导信息、设备有被操控风险等，应用可以根据风险分及线索，进行有效提示或拦截。

## 问题定位

1. 根据错误码[1012500001](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/devicesecurity-arktsapi-errcode-brid#section15009252203)信息，分析错误原因可能为接口执行流程中调用系统其它接口出现异常。
2. 检查nonce值是否符合参数要求：nonce值必须为24至80字节之间。
3. 更换设备重新执行检测，确认是否由于触发接口使用限制导致报错。
4. 检查接口的[约束与限制](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/devicesecurity-fraudriskdetection#section097084185914)，每个应用在每个设备上每天最多可以调用10次，超出限制则会报错。

## 分析结论

涉诈剧本检测接口存在使用约束与限制，每个应用在每个设备上每天最多可以调用10次。若调用接口达到限制次数后，再次调用接口会出现该报错。

## 修改建议

根据文档说明，每个应用在每个设备上每天最多可以调用10次此接口，目前最大调用次数无法自定义修改。

若应用处于调测阶段，可以通过以下方案来规避每日最大调用次数的限制：

* 通过以下hdc命令清理缓存重置接口调用次数：hdc shell bm clean -d -n com.huawei.hmos.brid。
* 在手机设置中修改手机系统的时间，规避接口每日使用次数限制。
