## 场景介绍

图片拼接是拍摄美化类应用的典型场景之一，如用户可以按无缝拼接、上下拼接和动态拼接等不同模板拼接多张图片。

本示例基于[Image Kit](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/image-overview)实现不同模板拼接图片的功能，通过[@ohos.file.photoAccessHelper](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-photoaccesshelper)保存图片到相册。

## 实现思路

1. 通过selectImages方法选择所需的图片并储存。
2. 根据所选图片的数量，弹出不同的拼接模板组件TwoImagePopBoxView或ThreeImagePopBoxView。
3. 根据模板方法threeImageStencilFirst、twoImageStencil、threeImageStencilSec进行拼接。
4. 通过savePixelMapToGalleryBySaveButton方法将拼接完成的图片保存。

```
// 首先选择所需要的图片并储存
export async function selectImages(maxSelectNumber: number) {
  // ...
}
// 根据所选图片的数量给出不同的拼接模板
if (this.joinImages.length === CONFIGURATION.IMAGE_STITCH_TWO) {
  TwoImagePopBoxView({
    // ...
  });
}else if (this.joinImages.length === CONFIGURATION.IMAGE_STITCH_THREE){
  ThreeImagePopBoxView({
    // ...
  });
}
// 然后根据模板方法进行拼接
export async function threeImageStencilFirst(images: Array<string> | Array<image.PixelMap>) {
  // ...
}
export async function twoImageStencil(images: Array<string> | Array<image.PixelMap>) {
  // ...
}
export async function threeImageStencilSec(images: Array<string> | Array<image.PixelMap>) {
  // ...
}
// 最后将拼接完成的图片保存
savePixelMapToGalleryBySaveButton(this, this.joinImagePixelMap);
```

## 约束与限制

* 本示例支持API Version 20 Release及以上版本。
* 本示例支持HarmonyOS 6.0.0 Release SDK及以上版本。
* 本示例需要使用DevEco Studio 6.0.0 Release及以上版本进行编译运行。
