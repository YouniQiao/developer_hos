# 接口说明

<strong>请求格式说明</strong> <strong>：</strong>

客户端：鲸鸿动能广告平台服务器

服务端：广告主服务器

请求协议：HTTPS

请求方式：POST

接口URL：https://www.huawei.com/rta (域名由广告主定义)

<strong>接口参数说明：</strong>

| --- | --- | --- | --- |
| <strong>字段</strong> | <strong>类型</strong> | <strong>说明</strong> | <strong>必填</strong> |
| requestId | string | 请求唯一id | 是 |

<strong>二次请求响应注意事项</strong>

请认真看否则可能会出现调价不预期.

CreativePriceInfo、DpaPriceInfo 字段中的 List&lt;OcpcPriceInfo&gt; ocpcPriceInfoList 子字段只能填一个单出价信息和双出价信息，不能有多个单出价和双出价信息。

否则华为DSP引擎会使用最后一个单、双出价信息 覆盖前面的出价信息。（因为华为DSP引擎 把convType 和 deepConvType 字段作为判断单、双出价的依据，并不会去使用这两个字段去给创意或者商品分类）

下面举例说明：

假设以下创意在华为平台的属性括号中所示：

creative1（非OCPC任务）

creative2（OCPC任务、单出价，convType = 4，）

creative3（OCPC任务、双出价，convType = 5，deepConvType = 15）

creative4（OCPC任务、双出价, convType = 5，deepConvType = 15）

creative5（OCPC任务、双出价, convType = 7，deepConvType = 25）

则响应体应该为：

```
{
    "retCode": 0,
    "errorMsg": null,
    … …
    "creativePriceInfos": [
    {   
        "ids": [creative1, creative2, creative3, creative4],
        "priceWeight": 0,
        "price": 15.89,
        "currency": "CNY",
        "ocpcPriceInfoList": [
            {
                "convType": 任意值,
                "currency": "CNY",
                "price": 25.7
            },
            {
                "convType": 任意值1,
                "deepConvType": 任意值2,
                "currency": "CNY",（国内这个字段不用填）
                "priceWeight": 1.87
             } 
        ]           
    },
    {
        "ids": [creative5],
        "priceWeight": 0,
        "price": 0,
        "currency": "CNY",
        "ocpcPriceInfoList": [
            {
                "convType": 任意值1,
                "deepConvType": 任意值2,
                "currency": "CNY",
                "priceWeight": 2.5
            }
        ]
    }
    ],
    "dpaPriceInfos": null
}
```

以上创意的调价结果为：

creative1 绝对值调价，调价为15.89元

creative2 绝对值调价，调价为25.7元

creative3、creative4 系数调价，调价为 原始出价\* 1.87。

creative5 系数调价，调价为 原始出价\* 2.58。

<strong>请求消息鉴权的要求如下：</strong>

- 参数生成方式：HmacSha256(requestId +requestTime+oaid, secretKey)，其中：

  签名密钥（secretKey）：由鲸鸿动能平台生成，线下传递密钥明文；

  若oaid不存在时，则生成方式为HmacSha256(requestId +requestTime, secretKey)
- <strong>举例如下：</strong>
