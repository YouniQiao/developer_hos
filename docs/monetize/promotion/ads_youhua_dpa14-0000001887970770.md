# RTA接口说明

RTA（Real Time API）广告形式，是指广告平台收到媒体请求后，除了引擎本身的各项过滤和排序逻辑，还会实时的请求、接收广告主对“是否目标用户”的反馈结果。如是广告主的目标用户，则继续走排序、竞价流程；如不是广告主的目标用户，则从广告队列内删除。在动态商品广告中，如您采用RTA可有效利用您的一方数据源进行商品精准推荐。

## 对接流程

您对接RTA的流程如下：

- 提供材料申请权限：请联系客户运营或服务商申请开通RTA权限。
- 对接测试：生成密钥明文并告知广告主。根据调研表信息，构造请求对接测试，详见接口文档[《鲸鸿动能RTA接口说明》](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260529160258.50715066752855837067358859045373:20260531101328:2800:96F84004314518DFD98F50E99A297E851463AAFB4DEB404CF88B490506DB713D.pdf?needInitFileName=true)。
- 详细对接流程指导请参考[《鲸鸿动能RTA接口说明》](https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20260529160258.77758867086898396563721436093149:20260531101328:2800:B13A7868CE813AD5678DB1AB983560FE671DD195541927FA9C49F2CE9DE333B1.pdf?needInitFileName=true)。

## RTA中DPA接口字段说明

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| <strong>字段</strong> | <strong>类型</strong> | <strong>含义</strong> | <strong>是否必须</strong> | <strong>生效优先级</strong> |
| rtaIdList | List&lt;String&gt; | RTA ID列表 | 否，rtaIdList和userIdList如果都没有，则platformId、productList、priceWeight也没有 | 1 |
| userIdList | List&lt;String&gt; | User ID列表 | 否，rtaIdList和userIdList如果都没有，则platformId、productList、priceWeight也没有 | 1 |
| platformId | String | 商品库ID | 否，如需返回商品，商品库id则必填 | - |
| recMode | Int | 商品的推荐模式 | 否，默认行为推荐 。1: LBS推荐 ，0: 行为推荐 | - |
| productList | List&lt;String&gt; | 多个商品ID | 否，如带商品库信息productList和productExtList至少携带一个 | - |
| productExtList | List&lt;Product&gt; | 带分值的商品列表 | 否，如带商品库信息productList和productExtList至少携带一个。productExtList优先级高于productList | 1 |
| priceWeight | Double | 出价比例 | 否，出价比例作用在投放端出价上，作为系数相乘，取值需大于等于0；oCPC出价需使用下述的ocpcPriceInfoList，不能使用此字段。 | - |

Product（扩展的商品信息，原来productList只能携带商品id）

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| <strong>字段</strong> | <strong>类型</strong> | <strong>含义</strong> | <strong>是否必须</strong> | <strong>生效优先级</strong> |
| productId | String | 商品ID | 是 | - |
| score | Int | 商品打分  0-4分，4分表示推荐可靠度高。 | 否 | - |

- DPA支持RTA优选推荐及华为商品推荐两种模式，通过RTA接口进行优选推荐时，最大支持返回20个商品id信息，系统将根据返回的创意id列表与该任务投放的类目范围进行匹配，如未返回商品列表或所有返回的商品ID不在任务投放类目内，则任务不参与竞价。
- 通过RTA接口反馈的优选商品，将按照productList中商品的排列顺序从前向后，或按照productExtList中商品加权分值匹配广告任务投放的类目，首先匹配的商品将直接应用于广告创意展示，其他商品将根据动态落地页中的配置获取商品信息进行展示。
- DPA模式下，RTA接口返回参竞但未返回商品id、商品库推荐模式、商品库id消息时，将使用华为商品推荐模式。
