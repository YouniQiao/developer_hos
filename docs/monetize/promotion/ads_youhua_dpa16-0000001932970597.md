# FAQ

<strong>Q1：</strong> <strong>什么是商品库、商品数据源？</strong>

A：商品库是商品的集合，可以包含多个商品数据源，由xml或xmlsitemap文件组成。商品数据源是广告主增加及更新商品的方式，一个商品数据源内包含若干个商品。

<strong>Q2：</strong> <strong>商品库中图片尺寸是否有要求、</strong> <strong>支持哪些创意形式和版位？</strong>

A：系统无法识别商品库中商品图尺寸，对接时要求商品库中各商品的图片尺寸唯一，如商品库中商品图尺寸不一致时，会导致创意拼接时出现商品图拉伸变形的情况，影响创意效果。

模板模式：图片样式支持1080\*1620、1080\*1920、1920\*1080、720\*1080、1080\*607、720\*1280、1080\*170、225\*150等尺寸。

直投模式：图片样式支持1080\*1620、1080\*1920、1920\*1080、720\*1080、1080\*607、1080\*170、720\*1280等尺寸。

<strong>Q3：</strong> <strong>动态创意模板支持哪些动态字段？</strong>

A：根据商品库的类型和广告主自定义的创意模板决定，不同商品库类型支持的动态字段不一样。除了最基础的商品图片、商品名称、现价、原价等信息，如需支持其他字段，请在提供创意模板时向运营申请。

<strong>Q4：</strong> <strong>商品库中的商品如何更新？</strong>

A：由广告主在自身服务器维护的商品数据源自主更新，广告平台通过定时拉取数据源的方式更新商品，同时可通过实时接口更新商品价格信息。

<strong>Q5：</strong> <strong>商品库支持对接多少个商品库，支持对接多少商品量级？</strong>

A：单广告主可建立多个商品库，商品量级无限制，超过200万商品库建议提前和产品沟通确认资源配置。

<strong>Q6：</strong> <strong>动态商品广告支持哪些行业？</strong>

A：动态商品广告目前支持电商、短视频、房产、本地生活服务、汽车出行行业，如您需投放动态商品广告，请联系客户运营。

<strong>Q7：</strong> <strong>RTA返回多个商品ID时，创意和落地页中如何展示</strong> <strong>？</strong>

A：按照RTA返回的商品序列展示，首个匹配的商品作为创意展示，且落地页如果采用动态组件，则首个商品位同样展示该商品，落地页中其他动态商品位置从优选列表中获取商品按商品序列展示。

<strong>Q8：</strong> <strong>有库存和可投放的枚举值是不一样的吗？</strong>

A：有库存和可投放的枚举值均为1，无库存和不可投放的枚举值均为0。

<strong>Q9：</strong> <strong>站内商品很多，如搜不到三级类目的情况应该怎么处理？</strong>

A：匹配不上类目的商品，建议放到“其他”类目下。

<strong>Q10：</strong> <strong>商品折扣信息中取值范围要求为：0.01-1.0；返回值应该是小数点后一位还是两位？</strong>

A：最多支持小数点后两位，依据实际情况可以填1位。

<strong>Q11：</strong> <strong>如果没有商品下线时间怎么办？</strong>

A：因下线时间字段为必填，建议统一填一个距离比较长的时间点。

<strong>Q12：</strong> <strong>url要不要加上cdata标签？</strong>

A：可以直接用cdata。

<strong>Q13：</strong> <strong>RTA接口的userinfolist里，priceweight不填会怎么样？</strong>

A：priceweight是用来修改创意竞价出价的，不填则无法修改出价。

<strong>Q14：</strong> <strong>DPA更新时间只能固定一个时间点吗？不能24小时随时更新吗？</strong>

A：整个商品库最短按天更新，您可通过Marketing API接口实现部分商品实时更新(如商品下架、商品价格）。

<strong>Q15：</strong> <strong>整个商品库的更新最短按天更新，这里的整个商品库的更新是指：xml链接变更？还是xml链接不变，但是里面全部商品的更新？</strong>

A：xml链接不变，更新xml链接里的全部商品。

<strong>Q16：</strong> <strong>打开创意中的动态应用直达的开关，就会自动调取xml里的deeplink吗？</strong>

A：打开创意中的动态应用直达开关后会对应调取创意展示商品的deeplink链接。

<strong>Q17：</strong> <strong>xml数据源里已经有deeplink了，创意设置处的应用直达链接还需要填么？</strong>

A：如果xml数据源里有填写deeplink链接，且想用户跳转至商品对应的直达链接，则需使用通配符“Android应用直达链接”。

<strong>Q18：</strong> <strong>RTA返回商品的时候, 是用哪个ID？products ID 还是HUAWEI侧生成的一个ID？</strong>

A：RTA的接口字段名称用products id，值就是xml数据源里面的products ID值（广告主自己的商品ID）。

<strong>Q19：</strong> <strong>如果广告主返回超时了，DPA任务是否会直接不参竞？</strong>

A：如果广告主返回超时了直接不参竞。DPA任务在RTA返回的信息不带商品ID、商品库ID、商品库、推荐模式字段的情况下，系统会使用华为商品推荐。

<strong>Q20：</strong> <strong>对接文档中h5落地页url是必填，拉活也是必填吗？</strong>

A：当前落地页url是必填字段，促活广告本身不需要url，当前因为字段必填所以可以放置一个统一的url链接，促活广告应用直达链接必填。

<strong>Q21：</strong> <strong>创意中动态文案=\{品牌名称\}，品牌名称=维纳斯。xml里每个商品对应一个品牌名称，实际是广告文案，请问用户实际看到的广告文案是xml填的，还是品牌名称“维纳斯”三个字？</strong>

A：创意中设置的品牌名称与文案是两个不同的展示位置，如投放信息流，品牌名称是显示在创意左下角的，实际看到的文案是动态文案=\{品牌名称\}中的内容。

<strong>Q22：</strong> <strong>动态落地页是否支持直接调用xml里的targetUrlMobile字段中的落地页链接展示给用户？</strong>

A：这种场景不支持,只支持用Venus动态落地页，实现创意商品和落地页内容的关联。

<strong>Q23：</strong> <strong>新建商品库后，为什么会没有立即生效？</strong>

A：商品库按照创建时设置的更新时间生效。

<strong>Q24：</strong> <strong>如果修改旧商品库的xml链接，修改完成后，商品库id是否会变化？</strong>

A：修改旧商品库的xml链接商品库ID不变。

<strong>Q25：</strong> <strong>如果修改旧商品库的xml链接，修改过程中和修改完成后，正在投放中的且使用该商品库的广告是否会受到影响？</strong>

A：更新数据源，商品文件正确的话，下次更新时会从新的数据源获取数据，正常入库没有影响，但需要保证RTA接口返回的商品id能在新的数据源中的找到。

<strong>Q26：</strong> <strong>DPA支持RTA直接出价吗？</strong>

A：与普通任务一致，支持RTA直接出价。
