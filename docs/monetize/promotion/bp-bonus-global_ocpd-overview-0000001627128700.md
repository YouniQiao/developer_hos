# 政策介绍

全域场景涵盖众多华为合作媒体，能有效扩大推广覆盖，是提升获量的必选手段。为降低开发者的试错成本，放心出价，全域推荐oCPD任务在推广初期提供成本保障能力，帮助任务快速起量。

## 激励范围

| 范围项 | 要求 | 说明 |
| --- | --- | --- |
| 任务类型 | 仅<strong>全域推荐</strong>的oCPD任务。 | 功能上线后，新建的任务可享受激励。  精选推荐和搜索等不参与。 |
| 转化目标 | oCPD支持的全部转化目标。 | 当前仅支持oCPD单目标场景，不支持oCPD双目标场景。 |
| 保障周期 | 首次投放起，及其之后的10个自然日。 | 首次投放即推广首次曝光时间。 |

## 激励条件

需同时满足以下条件，任意一条不满足，不享受激励。具体如下表所示。

| 条件项 | 要求 | 说明 |
| --- | --- | --- |
| 转化数要求 | 实际转化数&gt;=6 个。 | 回传的转化数为该任务的实际转化数，超过保障周期回传的转化数不算在内。如果转化目标为次留，需等到T+1日转化数据回传后再统计。 |
| 超成本比例 | 整个保障周期成本偏差&gt;= 20%。 | 保障周期结束后计算成本偏差，成本偏差=实际转化成本/目标出价-1。 |
| 修改次数要求 | 每日任务修改次数&lt;=2次。 | 修改任务的操作包含定向、关键词、创意、任务启停、出价（幅度&lt;= 20%）等调整行为。  <strong>注意：</strong>如果出现单次出价调整大于20%，则该任务不享受激励。 |
| 新任务首次曝光时间未超时 | 新任务首次曝光时间距离任务创建审核通过时间小于90天。 | - |

## 结算规则

| 规则项 | 要求 | 说明 |
| --- | --- | --- |
| 赔付金额 | 金额计算公式  单任务的赔付金额=此任务在保障周期内消耗金额-目标消耗\*120% | 目标消耗计算规则：  目标消耗=（*目标出价1*\*对应转化数）+（*目标出价2*\*对应转化数）+ （*目标出价3*\*对应转化数）+ ….+（*目标出价N*\*对应转化数）  <strong>说明：</strong>*目标出价1*对应的转化数，是在目标出价生效期间，所接收到的转化回传数量。最终需要统计目标转化出价期间，所获得的对应的实际转化数。  例如： 时间段00:00~20:00，出价为20，实际转化数为100；时间段20:00~24:00，出价为30，实际转化数为200，则当天的目标消耗=（20\*100）+（30\*200）=8000 |
| 结算返还方式 | 赔付金额计入赠送推广基金账户。 | 非现金，不可提现或退款，有效期为1年。 |
| 结算周期 | 每T+1自然月走一次结算流程，30个自然日内发放。 | 下月结算的任务需要满足保障周期和成本计算全部在本月内。如果出现跨月的场景，则下月无法进行结算，需要挪到下下月进行结算。  例如：   - 保障周期为5月1日至5月10日，5月11日完成成本计算（次留目标需5月12日完成成本计算），结算时间为6月。 - 保障周期为5月22日至5月31日，6月1日完成成本计算（次留目标需6月2日完成成本计算），结算时间为7月。 |
| 其他 | 异常不予结算的情况 | - 仅赔付金额&gt;0时，才触发结算，否则不予结算。 - 如果开发者数据回传出现异常，则不予结算。 |
