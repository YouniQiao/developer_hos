# 常见问题

## 测试失败问题

1. <strong>三方监测平台转化测试为什么失败 ，怎么解决？</strong>

   测试步骤的前提：您集成的SDK需要符合版本要求，且完整集成，应用能够获取OAID，三方监测SDK版本要求请参考[支持的三方监测工具](https://developer.huawei.com/consumer/cn/doc/promotion/tracking-overview-0000001170938773)。

   <strong>原因：</strong>

   1. 历史用户：测试设备曾经安装过这个应用，三方平台自动过滤历史用户。
   2. 未采集OAID：开发者使用OAID测试联调，但应用未完整集成三方监测平台SDK，获取不到OAID。
   3. 应用未激活：下载应用后，未在应用内完成对应转化操作。

   <strong>措施：</strong>

   1. 卸载应用后，重置设备OAID和GAID，再次测试，或者更换手机设备后再次测试。
   2. 检查SDK版本号是否符合要求，排查应用是否获取了OAID（可与您公司的开发人员确认）。
   3. 下载应用后，需要在应用内完成对应转化操作。
   4. 若以上措施无效，请联系您的华为运营或者[在线提单](https://developer.huawei.com/consumer/cn/support/feedback/#/)，提供如下信息，以便我们进行排查：

      |  |  |  |  |  |
      | --- | --- | --- | --- | --- |
      | 广告主账户ID | 每次测试设备的OAID | 数据回传秘钥 | 测试时间 | 点击监测地址 |
2. <strong>测试对手机有什么要求吗？</strong>

   全球手机都可以测试成功，手机网络没有限制，能访问目标网址并正常点击/下载即可。

## 曝光/点击问题

1. <strong>为什么在鲸鸿动能广告报表数据与第三方监测平台的数据，会出现曝光/点击数据差异？</strong>

   <strong>原因：</strong>

   曝光/点击数据，是由媒体SDK同时上报给鲸鸿动能广告平台和第三方监测平台的。由于网络丢包和平台之间反作弊规则的不同，可能导致两边存在数据差异，一般差异值在5%以内。

   <strong>措施：</strong>
   1. 若您在第三方监测平台看到的曝光/点击数与鲸鸿动能广告报表数据，差异超过5%，您可以向华为运营申请得到总曝光和总点击数据，进行核对。
   2. 若您在第三方监测平台看到的曝光/点击数与鲸鸿动能广告报表数据，差异超过10%，请联系您的华为运营或者[在线提单](https://developer.huawei.com/consumer/cn/support/feedback/#/)，提供如下信息，以便我们进行排查：

   |  |  |  |  |  |  |
   | --- | --- | --- | --- | --- | --- |
   | 您的广告账户ID | 任务ID | 投放时间 | 第三方监测平台曝光量 | 第三方监测平台点击量 | 数据截图 |
2. <strong>鲸鸿动能广告的点击数和三方平台的点击数，分别是什么含义？</strong>
   1. <strong>鲸鸿动能点击量：</strong>广告媒体上报经过有效性检测过滤后，得到的有效点击量（有效且计费的点击数量）。
   2. <strong>第三方监测平台点击量：</strong>广告媒体上报的总点击数，未进行有效性过滤，与鲸鸿动能广告的点击量有些许差异。

## 转化数据问题

1. <strong>如何查看三方监测的数据？</strong>
   1. 如果您设置了将数据回传到鲸鸿动能广告平台，且如果数据回传设置正确，第三方监测平台和鲸鸿动能广告平台都可以查看相关数据。如果您没有设置将数据回传到鲸鸿动能广告平台，您需要登录第三方监测平台查看。
   2. 部分三方监测平台，您可以将华为运营人员的账号加入项目成员中，加入后华为运营人员也可以查看广告的相关数据。
2. <strong>三方监测的数据与鲸鸿动能广告的数据不一致以哪个为准？</strong>
   1. 曝光、点击数据不一致：如果差额在10%以内，则以鲸鸿动能广告数据为准。如果偏差超过10%，则以经过排查后的鲸鸿动能广告数据为准。
   2. 后项转化数据不一致：以三方监测数据为准，例如：激活等。

      三方监测的转化数据回传鲸鸿动能广告后，会先经过鉴权、去重等过滤，再呈现报表。所以鲸鸿动能广告侧数据会比三方监测略少，正常相差在5%以内。
3. <strong>三方监测和</strong> <strong>鲸鸿动能广告</strong> <strong>的安装指标有何不同?</strong>
   1. 鲸鸿动能广告自己有安装指标，安装量指的是用户在点击广告之后，完成应用安装的次数。
   2. 安装在三方监测平台是install，代表的是激活，指的是用户在点击广告之后，下载并安装应用，首次打开应用的次数
4. <strong>鲸鸿动能广告</strong> <strong>报表的下载量和第三方监测平台的安装量，为什么数据差异较大？</strong>

   <strong>原因：</strong>
   1. <strong>统计口径不一致</strong>

      鲸鸿动能广告下载量：仅仅指用户完成下载应用动作的数量，不包含安装和应用打开。

      第三方监测平台安装量：用户完成下载/安装以及首次打开应用，整个过程的数量。此时第三方监测平台的安装量与鲸鸿动能广告平台的激活量概念一致。
   2. <strong>用户流失/误下载</strong>

      从点击广告到下载、安装、首次打开应用，整个过程中，存在用户流失/用户误下载/不打开应用等情况。下载量和安装量指标，数据差异在50%左右。

   <strong>措施：</strong>遇到转化数据差异问题，请先确保转化指标数据定义一致，再做比较。
5. <strong>三方监测后台安装量很少 ，甚至没有转化数据 ，为什么？</strong>

   <strong>原因：</strong>

   1. 您在集成三方监测平台SDK时，有遗漏，没有根据文档完整地集成，导致应用未采集到OAID。而OAID被用于归因，若应用未采集OAID，导致大部分鲸鸿动能广告的转化会被归到自然量或者其他渠道。

6. <strong>鲸鸿动能广告报表为什么没有付费金额？</strong>

   <strong>原因：</strong>

   1. 您的应用没有采集付费事件，或者没有在三方监测平台配置付费事件，没有配置付费金额回传。
   2. 若您是使用DTM工具回传，您没有在“自定义参数” 配置附加参数“revenue”和“currency”。
   3. 当三方监测平台有付费金额，鲸鸿动能广告报表没有，表示您可能没有将数据回传给鲸鸿动能广告平台。

   <strong>措施：</strong>

   1. 您的应用需要采集付费事件，或者在三方监测平台配置付费事件，并配置付费金额回传。
   2. 若您是使用DTM工具回传，请确保在“自定义参数” 配置模块，配置了附加参数“revenue”和“currency”，这样金额才会随转化事件发生上报。
   3. 当三方监测平台有付费金额，鲸鸿动能广告报表没有，请确保您已经将数据回传给鲸鸿动能广告平台。

## 其他问题

1. <strong>鲸鸿动能广告可以监控的转化指标有哪些？</strong>

   鲸鸿动能广告平台可直接监测曝光量、点击量。而广告后向转化的指标数据(激活量、付费量等等)，需要接入其他监测平台，并且您需要回传数据才可追踪。
2. <strong>归因模式有几种？</strong>

   三方监测进行转化效果分析，存在四种归因模式，常用的是设备ID归因。以下按照优先级从高到低进行排序：

   1. 渠道包归因（若没有渠道包归因,那么依次到以下归因）。
   2. install referrer 归因（调用AG接口读取referrer）。
   3. 设备ID归因（OAID、GAID）。
   4. 指纹归因（通过IP地址+user agent确认用户）。
3. <strong>鲸鸿动能广告平台的中国大陆区域账户和非中国大陆区域账户支持的第三方监测是否不同？</strong>

   不同，我们国内外对接的厂家不一样，对宏的要求也是不一样的。
4. <strong>鲸鸿动能广告平台何时发事件给三方监测平台？</strong>

   曝光：页面显示超过500毫秒，鲸鸿动能广告SDK自动发曝光事件给第三方监测平台。

   点击：用户点击广告 ，鲸鸿动能广告SDK自动发点击事件给第三方监测平台。
5. <strong>如果用户下载应用后，过了几个月再安装可以归因到该广告吗？</strong>

   点击转化归因窗口期最大30天，如果监测到的某种转化行为超过归因期限，该次转化不会被作为有效的转化。
6. <strong>附加参数相关问题（含转化金额、转化价值）</strong>。

   鲸鸿动能广告的转化追踪支持以下12个附加参数，对鲸鸿动能广告的所有转化事件适用，其中常用的三个附加参数为revenue、value、currency：

   | 附加参数枚举值 | 推荐值类型 | 说明 |
   | --- | --- | --- |
   | revenue | float | 收益金额。 |
   | value | float | 转化价值。 |
   | currency | string | 币种。 |
   | price | float | 单个商品价格，所有购买的总金额用revenue参数表示。 |
   | content\_type | string | 商品类型。 |
   | content\_list | array | 商品名称列表。 |
   | content\_id | string | 商品ID。 |
   | quantity | Int | 商品数量。 |
   | payment\_info\_available | boolean | 付款信息是否可用。 |
   | validated | string | 验证。 |
   | receipt\_id | string | 收据号。 |
   | order\_id | string | 订单号。 |

   1. revenue /value为数字类型，currency币种格式为：CNY/USD/EUR/JPY/GBP
   2. 目前只有转化金额/转化价值通过附加参数的形式，随着转化事件的发生而上报。举例：您想要监测付费事件的转化金额，那么可以在付费这个转化类型里配置“revenue”和“currency”，分别设定“10”和“CNY”，那么当付费事件被监测上报时，这两个参数的值也被一起上报，这样鲸鸿动能广告平台即可知道本次付费有10人民币的付费金额。
   3. 付费金额仅通过付费事件（paid）回传且仅支持在报表计划/任务/创意里累加，暂不支持查看每个用户付费的具体数据。
