# FAQ

## 所有下发的数据都是应用商店推广的用户数据么？

是的。在任务上填写推广监测链接后，下发的监测信息（展示、点击、下载、安装）都是推广任务的用户数据。

## 同一用户在不同任务多次下载，设备 ID 是否多次下发？下发的设备ID是 imei 还是 oaid 。

会多次下发。如多次曝光、多次点击、多次下发，每次行为都会转发。 设备 ID 只会下发oaid。

## 建议归因方案是first click还是last click？

建议last click方案。

## 没有接收到callBack参数，如何排查？

使用监测链接的开发者，监测任务上是否填写监测链接，以及是否添加宏\_\_CALLBACK\_\_\_。

## 建议归因时间窗设置多久？

归因时间窗，即开发者进行渠道归因时，会把转化设备和开发者在一定时间内通过监测链接接收的归因信息进行匹配，这个匹配的时间周期就是归因时间窗。

如您使用点击/下载/安装监测链接进行归因，建议的归因时间窗为30天。

如您使用曝光监测链接，由于数据量较大，建议归因时间窗为7天。

## oCPD支持哪些转化目标？

目前支持投放激活、注册、次日留存、授信、首次付费、每次付费、关键行为1、线索收集页面访问、老客激活、平台首日ROI、激活-首日ROI、激活-次日留存目标。其他转化目标暂不支持，目前还在规划中。

## oCPD任务的子任务的转化目标设置项没有选项，是什么原因？

如果直客账号切换后，创建了新数据源，新的数据源没有进行数据回传，此时投放任务中虽然有oCPD选项，但是转化目标设置项没有选项。当您往新数据源中回传数据后，转化目标设置项才会有选项。

## oCPD数据回传，显示归因不匹配，是什么原因？

可以在数据源概况中看到回传数据的情况，查看方法参见[查询数据源](/docs/monetize/promotion/bp-functions-ocpd-inquire-data-source-0000001282844469)。

各指标的含义如下：

| 字段 | 含义 |
| --- | --- |
| PV | 接口接收到的总请求量。 |
| 归因成功PV | 请求中callBack格式正确，且成功归因到任务上的请求量。 |
| 非归因PV | 请求中callBack为0的请求量。 |
| 归因不匹配PV | 一般是请求中callBack参数的值错误，请检查任务是否回传了callBack原值。 |
| 校验错误PV | 转化请求格式不对，缺少必填参数或者必填参数拼写错误的请求量。 |
| 归因重复PV | 多个请求oaid、callBack和actionTime完全一样，即同一用户多次转化，被判定重复。 |

## oCPD有哪些投放建议？

- 转化数据对接，回传口径务必和考核口径一致。如果不一致，会影响模型优化方向，可能导致优化效果不符合预期。
- 前期转化出价，建议参考CPD的转化成本。CPD转化成本在数据资产-详情-归因报表里可以查看。如获量较困难，可以稍微提升出价。
- 新建的oCPD任务，前期成本可能存在波动，建议设置预算在1万元左右，一般2-3天之后逐渐稳定，可以根据效果提升预算。
- 频繁调价会影响算法模型，稳定投放阶段不建议调整出价。如需修改出价建议一天不超过一次。
- 投放过程中，建议观察CPD和oCPD的激活率和激活成本情况，如果oCPD效果更优，建议逐渐提升oCPD任务预算，进而提升账户整体效果。

## 数据回传时接口响应数据源不存在，有哪些原因？

可能原因如下，请仔细检查，或者参考<strong>[对接华为商店数据回传接口报错：数据源不存在](https://developer.huawei.com/consumer/cn/forum/topic/0201638846933210033?fid=0101271690375130218)</strong>

- 没有创建数据源
- 不是直客账户创建的数据源和API
- 应用不在对应直客账户内
- 请求中appid不对（appid要求纯数字，也不能是快应用的appid）

## 是否需要实时回传用户转化数据？次日统一回传是否可以？

需要实时回传。次日才回传会导致算法模型无法实时调整，导致无法投放oCPD。

## 调用回传用户在推广应用内的行为数据接口时，oaid是否可以加密?

不支持加密，必须是原文。

## 调用回传用户在推广应用内的行为数据接口时，对于回传激活或者注册的用户，要求是通过智能分包转化来的吗？还是说我们回传从其他渠道转化来的也可以？

必须是通过华为应用市场应用推广系统提供的智能分包或点击链接方案归因来的用户。

## 在推广后台创建oCPD任务时，“转化目标”无法选择，为什么？

要首先调用一次回传用户行为数据接口才能看到转化目标，如果未完成数据对接，则无法选择到转化目标。

注：投放线索收集页面访问、平台首日ROI（限游戏品类使用）目标需同步开通白名单，请通过客服、邮箱developer@huawei.com或者在线工单系统与我们进行联系咨询，审核人员评估后会在1-2个工作日内进行答复。

## 回传的转化数据在哪里查看？“深度转化次数”的含义是什么？

回传的转化数据在投放平台 &gt; 整体报表中查看，报表包含激活量、注册量等深度转化数据，开发者回传的转化数据在对应的转化字段中可以查看。

当前深度转化次数是遗留字段。CPD任务的深度转化次数，仅展示2021.11.16之前的历史数据，最新的数据在对应的字段中查看。oCPD任务的深度转化次数，和oCPD任务所选目标相关，如果oCPD目标是激活，则深度转化次数等于激活量。

## 回传后接口报错\\{"code":136183818,"msg":"The xxx is mismatch."\\} 的原因是什么？

回传后接口报错不匹配的情况有3种，对应的原因和解决方案如下：

1. 接口报错\\{"code":136183818,"msg":"The callBack is mismatch."\\} ：
   1. 华为侧没有获取到用户设备的OAID，会导致无法匹配，一般您使用智能分包做归因时，会遇到这种情况。
   2. 如果您采用曝光归因（即把转化用户的OAID与曝光监测链接收到的OAID做匹配），可能出现这种情况。如您确定使用了曝光归因，可以联系对应的华为运营经理来解决。
   3. 暂不支持出端设备的归因（魅族，移动联通合约机除外）。
2. 接口报错\\{"code":136183818,"msg":"The deviceId is mismatch."\\} :

   回传的转化信息中，callBack可以匹配上，但是OAID和用户下载时的OAID无法匹配。原因可能是用户下载之后重置了OAID，这类报错属于正常情况。如果比例过大超过5%，请联系对应的华为运营经理协助排查。
3. 接口报错\\{"code":136183818,"msg":"The appId is mismatch."\\} ：

   您回传转化所使用的API客户端所属的账户，和你上传APP的账户不一致。请使用上传APP的账户创建API客户端后，用该API客户端进行数据回传。

## 次留数据回传的统计显示规则是怎样的？

当日激活用户在第二天再次打开，则计为一个次日留存行为。该指标会将收到的次日留存量回传数据，匹配到对应的激活时间上。例如：6月16日有10个激活，其中5个激活在6月17日再次打开，则6月16日的次留数为5，且在18号可以看到次留数据。

## oCPD任务投放精准度只和oCPD数据回传有关系吗？

不只是和oCPD数据回传有关联，和cpd任务的数据回传也有关联，请尽量保持CPD能持续回传，有更多转化数据模型会更准确。

## 如何申请数据资产回传功能？

该权限无需额外申请，开通归因后就默认开通数据回传接口。

## 同一天多次回传付费金额，推广平台上的付费金额如何显示

显示多次回传金额的总和。

## 投放激活-次日留存双目标时，“深度转化目标值”应该如何填写？

激活-次日留存双目标以您设置的激活出价和次日留存率为目标进行竞价和优化，您可通过“预期激活成本/预期次留成本”计算预期“次日留存率”进行填写。

注：部分oCPD子任务的“深度转化目标值类型”仍保留了“深度转化出价”选项，建议您尽快将子任务目标值类型修改为“次日留存率”，便于统一观测维度，跟踪投放数据。

## 创建oCPD子任务时最右侧的流量拓展开关是什么功能？

开启流量拓展开关可扩宽子任务的获量空间，在达成投放目标的基础上探索更多优质流量。

注：由于监测链接归因数据偏少，特别建议您开启该功能时同步使用“智能分包”的归因方式。
