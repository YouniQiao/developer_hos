# 创建任务

【简介】通过此接口可以创建任务

<strong>请求地址</strong>

https://ads.cloud.huawei.com/ads/v1/promotion/adgroup/create

<strong>请求方法</strong>

<strong>POST</strong>

<strong>请求参数</strong>

|  |  |  |  |
| --- | --- | --- | --- |
| <strong>参数名称</strong> | <strong>类型</strong> | <strong>是否必选</strong> | <strong>描述</strong> |
| advertiser\_id | long | 否 | 广告主ID，当登录授权的华为账号为如下场景时此字段必填：  1）授权账号关联的是经理账户；  2）授权账号关联的是服务商账户；  3）授权账号关联了多个子客账户。 |
| adgroup\_name | string | 是 | 任务名称，  最大长度不得超100；  不能使用“^”,“|”特殊字符  不能重名; |
| campaign\_id | string | 是 | 计划ID,  最大长度不得超50;  只能填写数字;  不能使用“^”和换行；  注：该任务所属计划和产品上的产品类型需一致  详见[【创建计划】](/docs/monetize/promotion/ads_new_api03-0000001456040061) |
| channel\_package\_id | string | 否 | 渠道包ID,  即查询渠道包接口中返回的channel\_package\_id；  注：推广产品类型product\_type为ANDROID\_APP时用到;  详见[【查询渠道包】](/docs/monetize/promotion/ads_new_api28-0000001405200532) |
| channel\_pkg\_number | string | 否 | 智能分包渠道号，  即查询智能分包接口中返回的channel\_pkg\_number；  注：推广产品类型product\_type为ANDROID\_APP时用到，只支持国内以及定向为未安装;  详见[【查询智能分包信息】](/docs/monetize/promotion/ads-cxzhinengfenbaoliebiao-0000001926859329) |
| targeting\_package\_id | long | 是 | 定向包ID，  即创建定向包接口中返回的targeting\_id；  注：非App的推广产品不能有App安装定向  商品广告任务不支持创建共有定向  详见[【创建定向包】](/docs/monetize/promotion/ads_new_api43-0000001456040081) |
| targeting\_package\_scope | string | 是 | 定向类型  详见[【定向类型】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#section7631729103714)  当campaign\_type=CAMPAIGN\_TYPE\_SHOPPING该字段仅支持PRIVATE |
| product\_id | long | 是 | 推广产品ID,  即创建推广产品中接口中返回的product\_id；  注：任务上非App的产品ID和任务ID为一对一对应关系。  计划和产品上的产品类型需一致  详见[【创建推广产品】](/docs/monetize/promotion/ads_new_api18-0000001405360436) |
| adgroup\_begin\_date | string | 是 | 投放开始日期，  格式：YYYY-MM-DD;  如：2020-01-01;  注：开始时间不能小于当前时间，开始时间不能大于结束时间;  开始时间和结束时间差值需要在查询版位获得的投放此版位的任务最短和最长投放时间范围之内 |
| adgroup\_end\_date | string | 否 | 投放结束日期，  格式：YYYY-MM-DD;  为空表示长期投放;  如：2020-01-02  注：若存在结束时间，那么结束时间不能小于开始时间; |
| time\_period\_type | string | 是 | 时间段类型，  可以从三种类型中根据自己需要选择一个。  版位查询到的is\_support\_time\_period要有权限  time\_period为空时，时间段类型不得为TIME\_PERIOD\_DAY\_SPECIFIC或TIME\_PERIOD\_HOUR\_SPECIFIC  详见[【时间段类型】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#section7362176515) |
| time\_period | string | 否 | time\_period\_type为TIME\_PERIOD\_DAY\_SPECIFIC、TIME\_PERIOD\_HOUR\_SPECIFIC时必填。  且满足正则^[01]\\{336\\}$;  按照特定时间段投放时，01010100010010……表示指定特殊时段，半小时为粒度，用0、1标识是否选中，1为选中，0为未选中。一天24小时，即每天24\*2=48位，共7天，即共48\*7=336位  如果time\_period\_type为TIME\_PERIOD\_ALL，则表示一周内每天的特定小时，需要每48个值都重复。  注：时间段要与任务的日期区间有交集。  例：如果任务开始时间是2020-06-22（周一），结束时间是2020-06-24（周三），且选择的类型是TIME\_PERIOD\_DAY\_SPECIFIC或TIME\_PERIOD\_HOUR\_SPECIFIC，那time\_period的入参需要与任务的时间段有交集；若用户想在22/23/24的每天早上6点到9点投放任务，则time\_period的参数可表示为00000000000011111100…0000…(共336位, 每48位代表一天24小时，重复7次) |
| price\_type | string | 是 | 出价方式，  详见[【付费方式】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#section1898795494716) |
| price | float | 是 | 出价，整数位不得超过9位，小数位不得超2位；  注：范围在计划日限额和对应出价方式查询版位底价之间  日限额和出价比值应在合理的范围内  出价方式price\_type为oCPC时，该字段非必填 |
| conversion\_cost | float | 否 | 期望转化成本，  price\_type为PRICING\_OCPC时专用，暂时只支持App和网页和快应用。  整数位不得超过9位，小数位不得超2位  取值区间在出价与计划日预算之间  日限额和期望转化成本比值应在合理的范围内 |
| roi\_ratio | float | 否 | ROI系数  转化目标为TRACKING\_IAA\_FIRST\_DAY\_ROI时填写  支持0~100之间，最多支持四位小数 |
| creative\_size\_id | long | 是 | 版位ID  详见[【查询版位】](/docs/monetize/promotion/ads_new_api24-0000001405520288) |
| dyn\_price\_flag | string | 否 | 智能提价标志，  仅当推广产品为ANDROID\_APP；  且定向模板中not\_installed\_apps\_struct值为true时有效  详见[【智能提价标志】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#section109373445518) |
| dyn\_price\_upper\_limit | float | 否 | dyn\_price\_flag为DYNAMIC\_PRICE\_ENABLE时必填  智能提价上限，范围在1到10之间，  整数位不得超过2位，小数位不得超1位 |
| tracking\_id | long | 否 | 转化跟踪指标ID，  price\_type为PRICING\_OCPC、PRICING\_CPA时必填  详见[【查询转化跟踪目标】](/docs/monetize/promotion/ads_new_api36-0000001405200536) |
| ocpc\_strategy | string | 否 | 投放策略  price\_type为PRICING\_OCPC时必填  详见[【投放策略】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#section940892575916) |
| deep\_optimize\_type | string | 否 | 深度优化类型  price\_type为PRICING\_OCPC时才可以填写  详见[【深度优化类型】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#section161959511115) |
| deep\_effect\_target | string | 否 | 深度转化目标  price\_type为PRICING\_OCPC时并且deep\_optimize\_type不为空是必填；  详见[【查询深度转化目标】](/docs/monetize/promotion/ads_new_api37-0000001455960425) |
| deep\_effect\_price | float | 否 | 深度转化价格  price\_type为PRICING\_OCPC时并且deep\_optimize\_type不为空是必填；  整数位不得超过9位，小数位不得超2位；  日限额和期望转化成本比值应在合理的范围内 |
| dpa\_recommand | string | 否 | 商品推荐方式  详见[【商品推荐方式】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#section5978193823810) |
| dpa\_filter | Struct1 | 否 | 商品任务过滤条件  campaign\_type=CAMPAIGN\_TYPE\_SHOPPING  时 此字段必填 |
| objective\_type | string | 否 | 推广目的  详见[【推广目的】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#section122581352419)  创建预约下载任务时，本字段必填。 |
| fc\_days | long | 否 | 多日频控N天M次中的N  取值1-5 |
| fc\_upper\_times | long | 否 | 多日频控N天M次中的M  取值1-10 |
| keywords | Struct3 | 否 | 关键词 |
| adgroup\_status | string | 否 | 任务需要设置的状态，  创建场景仅支持OPERATION\_ENABLE,OPERATION\_DISABLE  不填默认OPERATION\_ENABLE  详见[【操作类型】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#section349886194716) |
| impression\_monitor\_url | string | 否 | 曝光监测链接，最长2048  注：使用监测链接有如下要求：  1：计划类型是展示广告  2：推广产品是安卓应用  3：账户是国内账户  4：应用有资产，并且资产配置里设置了链接  5：资产下有已启用的事件 |
| click\_monitor\_url | string | 否 | 有效触点监测链接，最长2048  1：计划类型是展示广告  2：推广产品是安卓应用  3：账户是国内账户  4：应用有资产，并且资产配置里设置了链接  5：资产下有已启用的事件 |
| install\_monitor\_url | string | 否 | 安装监测链接，最长2048  1：计划类型是展示广告  2：推广产品是安卓应用  3：账户是国内账户  4：应用有资产，并且资产配置里设置了链接  5：用户需要有安装数据上报权限  6：资产下有已启用的事件 |
| nobid\_price\_cap | float | 否 | 转化单价上限，  price\_type为PRICING\_OCP，ocpc\_strategy为OCPC\_STRATEGY\_MAXIMUM\_CONVERSION时专用，暂时只支持APP和网页和快应用。  整数位不得超过9位，小数位不得超2位  取值区间在出价与计划日预算之间  日限额和期望转化成本比值应在合理的范围内 |
| reserved\_pkg\_name | string | 否 | 预约包名  当objective\_type为APP\_RESERVE\_DOWNLOAD可以填写。 |
