# 简介

## 概述

oCPC（Optimized Cost Per Click）是一种智能投放模式，广告主选择特定的转化目标，并提供愿意为该转化目标而支付的平均价格之后，系统将借助转化预估模型，实时预估每次广告请求的转化价值进行智能出价，为广告主获取更多优质流量、控制转化成本。

创建oCPC广告任务时，不做强校验事件联调状态，未启用的事件也可正常投放。

只要在事件管理平台创建了事件，无论指标是否激活启动，都可以进行选择投放，平台收到转化事件后将会自动激活事件，同时您仍可以前往事件资产管理进行手动联调。

技术跟踪方式详见[转化跟踪](/docs/monetize/promotion/ads_gongju08-0000001469108293)功能。

API接入：采用API方式将转化数据回传给广告平台, 需要集成开发；

SDK接入：通过归因SDK配置自动采集应用内转化数据, 需要集成开发；

HMS统计：采用鲸鸿动能AdsKit追踪，无需集成开发；

Venus统计：采用维纳斯落地页上各组件的转化事件数据, 无需集成开发。

## 工作原理

- oCPC投放分为两个阶段：

  第一阶段：**学习期**，该阶段系统进行数据积累，系统通过分析积累的转化数据，逐步建立智能投放模型。

  第二阶段：**稳定期**，该阶段系统会根据转化预估模型进行智能投放，提高转化效率并控制转化成本使其接近目标成本。
