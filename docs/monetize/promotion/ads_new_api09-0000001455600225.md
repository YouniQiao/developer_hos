# 编辑任务

【简介】通过此接口可以编辑任务

<strong>请求地址</strong>

https://ads.cloud.huawei.com/ads/v1/promotion/adgroup/update

<strong>请求方法</strong>

<strong>POST</strong>

<strong>请求参数</strong>

|  |  |  |  |
| --- | --- | --- | --- |
| <strong>参数名称</strong> | <strong>类型</strong> | <strong>是否必选</strong> | <strong>描述</strong> |
| advertiser\_id | long | 否 | 广告主ID，当登录授权的华为账号为如下场景时此字段必填：  1）授权账号关联的是经理账户；  2）授权账号关联的是服务商账户；  3）授权账号关联了多个子客账户。 |
| adgroup\_id | long | 是 | 任务ID |
| adgroup\_name | string | 否 | 任务名称必填，  最大长度不得超100，  不能使用“^”,“|”特殊字符，  不能重名 |
| channel\_package\_id | string | 否 | 渠道包ID |
| channel\_pkg\_number[h1] | string | 否 | 智能分包渠道号，  即查询智能分包接口中返回的channel\_pkg\_number；  注：推广产品类型product\_type为ANDROID\_APP时用到，只支持国内以及定向为未安装。  详见[【查询智能分包信息】](/docs/monetize/promotion/ads-cxzhinengfenbaoliebiao-0000001926859329) |
| targeting\_package\_id | long | 否 | 定向包ID  即创建定向包接口中返回的targeting\_id；  注：非App的推广产品不能有App安装定向  详见[【创建定向包】](/docs/monetize/promotion/ads_new_api43-0000001456040081) |
| targeting\_package\_scope | string | 否 | 定向类型  详见[【定向类型】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#定向类型)  当campaign\_type=CAMPAIGN\_TYPE\_SHOPPING该字段取值为PRIVATE |
| adgroup\_begin\_date | string | 否 | 投放开始日期，  格式：YYYY-MM-DD  如：2020-01-01  注：原始开始时间小于当前时间并且原始结束时间大于当前时间，说明任务已经开始，不能修改开始时间，只能修改结束时间；  修改的开始时间不能早于当天。 |
| adgroup\_end\_date | string | 否 | 投放结束日期，格式：YYYY-MM-DD，传"9999-01-01"表示长期投放  注：结束时间不能早于开始时间，也不能早于当前时间；原始结束时间小于当前时间说明任务已经结束，不能修改任何时间 |
| time\_period\_type | string | 否 | 时间段类型，可以从三种类型中根据自己需要选择一个。  time\_period为空时，时间段类型不得为TIME\_PERIOD\_DAY\_SPECIFIC或TIME\_PERIOD\_HOUR\_SPECIFIC  详见[【时间段类型】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#时间段类型) |
| time\_period | string | 否 | time\_period\_type为TIME\_PERIOD\_DAY\_SPECIFIC、TIME\_PERIOD\_HOUR\_SPECIFIC时必填。  按照特定时间段投放时，01010100010010……表示指定特殊时段，半小时为粒度，用0、1标识是否选中，1为选中，0为未选中。一天24小时，即每天24\*2=48位，共7天，即共48\*7=336位  如果time\_period\_type为TIME\_PERIOD\_ALL，则表示一周内每天的特定小时，需要每48个值都重复。  注：时间段要与任务的日期区间有交集。  例：如果任务开始时间是2020-06-22（周一），结束时间是2020-06-24（周三），且选择的类型是TIME\_PERIOD\_DAY\_SPECIFIC或TIME\_PERIOD\_HOUR\_SPECIFIC，那time\_period的入参需要与任务的时间段有交集；若用户想在22/23/24的每天早上6点到9点投放任务，则time\_period的参数可表示为00000000000011111100…0000…(共336位) |
| price | float | 否 | 出价 整数位不得超过9位，小数位不得超2位；  注：范围在此任务中对应的版位的CPM最低售价和对应的计划中的日限额之间。  日限额和出价比值应在合理的范围内 |
| conversion\_cost | float | 否 | 期望转化成本，price\_type为PRICING\_OCPC时专用  暂支持App和网页  整数位不得超过9位，小数位不得超2位，  出价方式由CPX切换成OCPC时必填  取值区间在出价与计划日预算之间  日限额和期望转化成本比值应在合理的范围内 |
| roi\_ratio | float | 否 | ROI系数  转化目标为TRACKING\_IAA\_FIRST\_DAY\_ROI时填写  支持0~100之间，最多支持四位小数 |
| dyn\_price\_flag | string | 否 | 智能提价标志  仅当推广产品为ANDROID\_APP且定向模板中installed\_apps\_struct值为true时有效  详见[【智能提价标志】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#智能提价标志) |
| dyn\_price\_upper\_limit | float | 否 | dyn\_price\_flag为DYNAMIC\_PRICE\_ENABLE时必填  智能提价上限，范围在1到10之间，  整数位不得超过2位，小数位不得超1位 |
| tracking\_id | long | 否 | 转化跟踪指标ID，  price\_type为PRICING\_OCPC、PRICING\_CPA时必填  详见[【转化跟踪状态】](/docs/monetize/promotion/ads_api73-0000001058884698#转化跟踪状态) |
| price\_type | string | 否 | 付费方式  允许切换成ocpc的任务才可以填写  详见[【付费方式】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#付费方式) |
| ocpc\_strategy | string | 否 | 投放策略 price\_type 为 PRICING\_OCPC时必填 详见[【投放策略】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#投放策略) |
| deep\_optimize\_type | string | 否 | 深度优化类型  price\_type为PRICING\_OCPC时才可以填写  详见[【深度转化目标映射表】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#深度转化目标映射表) |
| deep\_effect\_target | string | 否 | 深度转化目标 price\_type 为 PRICING\_OCPC时 并且 deep\_optimize\_type 不为空时必填 详见  [【深度转化目标】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#深度转化目标) |
| deep\_effect\_price | float | 否 | 深度转化价格 price\_type 为 PRICING\_OCPC时 并且 deep\_optimize\_type 不为空时必填，整数位不得超过9位数，小数位不得超过2位  日限额和深度转化价格比值应在合理的范围内 |
| dpa\_filter | Struct1 | 否 | 商品任务过滤条件  campaign\_type=CAMPAIGN\_TYPE\_SHOPPING  时，可以修改数据。 |
| fc\_days | long | 否 | 多日频控N天M次中的N  取值1-5 |
| fc\_upper\_times | long | 否 | 多日频控N天M次中的M  取值1-10 |
| keywords | Struct3 | 否 | 关键词 |
| adgroup\_status | string | 否 | 任务状态  详见[【操作类型】](/docs/monetize/promotion/ads_meijuzhi-0000001510863205#操作类型) |
| impression\_monitor\_url | string | 否 | 曝光监测链接，最长2048  注：使用监测链接有如下要求：  1：计划类型是展示广告  2：推广产品是安卓应用  3：账户是国内账户  4：应用有资产，并且资产配置里设置了链接 |
| click\_monitor\_url | string | 否 | 有效触点监测链接，最长2048  1：计划类型是展示广告  2：推广产品是安卓应用  3：账户是国内账户  4：应用有资产，并且资产配置里设置了链接 |
| install\_monitor\_url | string | 否 | 安装监测链接，最长2048  1：计划类型是展示广告  2：推广产品是安卓应用  3：账户是国内账户  4：应用有资产，并且资产配置里设置了链接  5：用户需要有安装数据上报权限 |
| nobid\_price\_cap | float | 否 | 转化单价上限，  price\_type为PRICING\_OCP，ocpc\_strategy为OCPC\_STRATEGY\_MAXIMUM\_CONVERSION时专用，暂时只支持APP和网页和快应用。  整数位不得超过9位，小数位不得超2位  取值区间在出价与计划日预算之间  日限额和期望转化成本比值应在合理的范围内 |
