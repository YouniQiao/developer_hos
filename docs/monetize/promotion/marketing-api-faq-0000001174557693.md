# 常见问题

1. <strong>是否可以实时查询数据报表？</strong>

   不可以。根据时间粒度，建议您每小时或者每天查询一次。
2. <strong>数据报表，小时粒度的24小时数据汇总是否等于天粒度的数据？</strong>

   小时粒度24小时的数据累加等于天粒度的数据，当前查询小时粒度的数据，只能查询一天。
3. <strong>数据报表，小时粒度、天粒度数据的更新时间？</strong>

   每5分钟更新一次，正常投放后15分钟后会产生数据。
4. <strong>数据报表，小时粒度、天粒度数据均有回溯？那我们是不是不能使用API抓取到的消耗数据、直接落入内部消耗表？还是必须查询财务报表的数据？</strong>

   每5分钟会基于请求时间更新小时粒度、天粒度的数据。此处查询的消耗数据不能用于财经结算。
5. <strong>如果我们每天都要例行使用某些接口，AT过期后需要重新“授权并登录”？</strong>

   不用，有refresh token可以刷新access token，详情参考：[刷新access\_token](https://developer.huawei.com/consumer/cn/doc/promotion/marketing-api-process-4-0000001128438066)。
6. <strong>我已经发邮件申请授权码模式，什么时候能帮我配置好这个模式呢？</strong>

   配置将在申请日第二个工作日下午5点完成，特殊情况会延迟。
7. <strong>每个账户都需要单独登录和授权吗？</strong>

   每个新的账户都需要的，防止越权。
8. <strong>给某个账户授权之后，客户端id下其他账户的token都会失效吗？</strong>

   不会失效，token可以理解和账户绑定的，每个token只会对应一个账户和一个客户端id，一个客户端id可以对应多个账户，每个账户获取的token对各自的账户生效，且账户之间不影响。
9. <strong>如果给第二个账户授权，第一个账户的token不会失效吗？</strong>

   不会，没有影响。
10. <strong>调用次数超过频次怎么办</strong> <strong>？</strong>

    [在线提单](https://developer.huawei.com/consumer/cn/support/feedback/#/)说明情况申请扩容。
11. <strong>非中国大陆地区鲸鸿动能广告Marketing API服务已配置成功，但回调链接有误，如何重新配置？</strong>

    回调地址可自行修改，请登录开发者联盟后台，在【HMS API服务】的【凭证】中选择归属项目下的对应客户端id数据点击编辑修改即可。
12. <strong>广告账户已申请开通非中国大陆地区鲸鸿动能广告Marketing API服务，但是想用别的广告账户进行投放，需要重新申请吗？</strong>

    不需要， 我们是按照client id配置的，只要client id不变就不用重新申请。
13. <strong>开发者联盟上的企业名称：存在多家企业名称的情况，是不是每家企业名称和华为账号，都需要进行申请?</strong>

    不用每家企业名称和华为账号都申请，只要客户端id有权限，广告主下所有的鲸鸿动能账号（包括子客直客经理账户）都可以接入Marketing API，具体详情请参考：&lt;https://developer.huawei.com/consumer/cn/doc/promotion/marketing-api-process-3-0000001128597856&gt;
14. <strong>访问数据的鲸鸿动能广告账户名称：一个企业下若存在多个广告账户，都要填写？</strong>

    &lt;https://developer.huawei.com/consumer/cn/doc/promotion/marketing-api-process-3-0000001128597856&gt;链接中描述的一个企业下若存在多个广告账户，可以不用都填写，按链接提示提交信息即可。
15. <strong>广告主可以通过API去调整：任务出价、任务开启/暂停、计划预算、新建计划/任务/创意吗？</strong>

    Marketing API是鲸鸿动能广告提供的对外开放接口，帮助您完成广告投放、报表分析、创意制作等操作，提升营销效率。具体详情请参考：&lt;https://developer.huawei.com/consumer/cn/doc/promotion/marketing-api-0000001174557681&gt;
16. <strong>可以通过Marketing API 接口获取合约广告的投放数据吗？</strong>

    目前不支持通过Marketing API接口获取合约广告的投放数据。
17. <strong>调用api时返回 "code": "102012","message": "Please sign the latest agreement.",怎么办？</strong>

    该信息提醒需要直接账密登录授权的主账户签署最新协议即可。
18. <strong>可以通过mapi拉取经理账号管理的多个账号的数据吗？</strong>

    可以，可以让经理账号代理mapi，查询的时候带下属的子客广告主id就可以查那个子客的信息，并且每次只能根据筛选条件查询一个子账号数据。
19. <strong>接入Marketing API调用查询计划接口携带campaign\_name参数但不支持模糊查询？</strong>

    是的，不支持模糊查询。
20. <strong>不同投放账号接入api可以共用一个clientid吗？</strong>

    可以使用同一个clientid，例如说有5个子客账号，5个直客账号，可以申请一个clientid，给这10个账号用。
21. <strong>任务数据报表中，字段start\_date 和end\_date，时间是只能查一周的数据吗？</strong>

    可以查最多一年的数据，并且起始时间跟结束时间不能超过7天，比如起始时间2024-5-21，那结束时间最多到2024-5-27。
22. <strong>在两个经理账户投放，现在的需求是通过请求Marketing API获取经理账户投放的的消耗数据，是否需要在应用后台需要两个client\_id对两个经理账户分别绑定呢？</strong>

    不需要，广告账户接入Marketing API，与客户端id申请的开发者账号主体没关系。
