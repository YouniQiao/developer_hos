# FAQ

<strong>Q1：客户端模式认证和授权码模式认证的区别？已按客户端模式认证接入的广告主如何兼容？</strong>

A：区别如下：

|  |  |  |  |
| --- | --- | --- | --- |
| <strong>模式</strong> | <strong>OAuth</strong> <strong>服务器是否让用户登录并授权</strong> | <strong>OAuth</strong> <strong>服务器是否验证client的账号密码</strong> | <strong>获取到的Token</strong> |
| 授权码模式 | ✔ | ✔ | 用户级Token |
| 客户端模式 | ✘ | ✔ | 应用级Token |

当前版本仅支持授权码模式接口。

<strong>Q2：当前版本，从开户到任务管理，哪些是在投放平台完成、哪些是可以通过接口完成?</strong>

A：见2.2章节的接口列表。

<strong>Q3：为什么任务上有“待上传创意”的状态？</strong>

A： API接口中，任务和创意可分别创建，任务创建完成后还未创建创意时，任务状态为“待上传创意”。在投放平台任务和创意是一起创建的，因此不会有这个状态。

<strong>Q4：数据报表，是否可以实时查询？</strong>

A: 各个维度报表都是离线报表，不支持实时查询，小时粒度建议每小时查询一次，天粒度建议每天查询一次。

<strong>Q5：数据报表，小时粒度的24小时数据汇总是否等于天粒度的数据？</strong>

A: 小时粒度24小时的数据累加等于天粒度的数据，当前查询小时粒度的数据，只能查询一天。

<strong>Q6：数据报表，小时粒度、天粒度数据的更新时间？</strong>

A: 每5分钟更新一次，正常投放后15分钟会有数据。

<strong>Q7：数据报表，小时粒度、天粒度数据均有回溯？那我们是不是不能使用API抓取到的消耗数据、直接落入内部消耗表？还是必须查询财务报表的数据？</strong>

A: 每5分钟会基于请求时间更新小时粒度、天粒度的数据。此处查询的消耗数据不能用于财经结算。

<strong>Q8：为什么开发者实名认证后看到的API凭证申请和我们文档的截图不一样？</strong>

A: 请先添加项目，再申请凭证就行。

<strong>Q9：</strong> <strong>广告主原来使用V1客户端模式，现在想用V2授权码模式，过程中遇到无接入权限，这样的情况需要重新申请授权码模式使用的Token吗？申请后会影响V1 的使用吗？</strong>

A:需要申请v2的token，不影响v1的使用。

<strong>Q10：如果我们每天都要例行使用某些接口，在AT过期后都需要手动“授权并登录”获取authorization\_code吗？</strong>

A：不用，有refresh token可以刷新access token，详情请见3.3章节。

<strong>Q11：授权码模式初次申请，申请邮件已发送，什么时候配置好？</strong>

A：自申请日第二个工作日下午5点，特殊情况会延迟。

<strong>Q12：每个账户都需要单独登录和授权吗？</strong>

A：每个新的账户都需要的，防止越权。

<strong>Q13：给账户授权之后，客户端id的token都会失效吗？</strong>

A：token可以理解和账户绑定的，每个token只会对应一个账户和一个客户端id，一个客户端id可以对应多个账户，每个账户获取的token对各自的账号生效，且账户之间不影响。

<strong>Q14：</strong> <strong>如果给第二个账户授权，第一个账户的token不会失效吗？</strong>

A：不会，没有影响。

<strong>Q15：调用次数超过频次怎么办？</strong>

A：可联系接口运营，发送邮件说明情况申请扩容。

<strong>Q16：经理账户如何调用marketing API接口？</strong>

A：可以通过参数advertiser\_id进行调用。详情请查询需调用接口入参advertiser\_id说明。

<strong>Q17：经理账户和广告账户调用方式有什么区别？</strong>

A：经理账户在调用时，需要传入advertiser\_id参数，而且是必填项，如果是广告账户则不需要传入advertiser\_id参数。

<strong>Q18：为什么通过API拉取的效果数据和投放端显示的不一致？</strong>

A：投放期间通过API拉取的数据时间和投放端更新的时间不同，可能导致差异。24小时后查询结果会完全一致。

<strong>Q21：经理账户模式接入marketing API介绍。</strong>

A：经理账户运用背景：经理账户是给广告主管理多个广告投放账户的管理账户，通过经理账户可以免登录切换进入已绑定的广告账户内，操作、查看广告账户，可以跨账户共享广告资产。当前只开放一个经理账户关联一个二代或者多个二代下的子客，不支持管理直客账户。

<strong>Q22：通过经理账户接入marketing API，实现下属企业免认证流程介绍。</strong>

A：1、如首次接入marketing API，需要在华为开发者联盟申请App ID，然后申请对应的marketing API接口权限，如已有账号接入过marketing API，不需要额外申请App ID。

2、 经理账户账号本身不具备marketing API接口接入权限，需要使用经理账户账号获取accessToken 接口接入时Authorization使用此token，接口参数中的advertiser\_id需要传入管理的下属企业ID，接口获取的数据为此advertiser\_id所有，切换advertiser\_id，可免重复获取token步骤。

3、 50005 User has no permissions 错误解释：

使用经理账户token，接口参数中不包含advertiser\_id，会出现此错误，经理账户本身不具备单独调用mapi权限，需要与下属企业的advertiser\_id配合使用。

50006 User id mismatch 错误解释：

1）使用经理账户获取token，advertiser\_id不为空，当advertiser\_id与经理账户无关联关系或者advertiser\_id对应的华为账号被停用时，会出现此错误。

2）非经理账户token，advertiser\_id不为空，token关联的华为账号ID与advertiser\_id不一致，会出现此错误。

4、当出现此错误时：

1）需要排查获取token的华为账号是否为经理账户账号，如果需要使用经理账户模式接入，则需要使用使用经理账户的华为账号获取token，接口参数中的advertiser\_id传入管理的下属企业advertiser\_id，配合使用。

2）如不使用经理账户模式接入，接口参数中可不传advertiser\_id，如果传入了advertiser\_id，需要保证advertiser\_id和token对应的华为账号ID保持一致。

<strong>Q23：之前已经发过邮件申请，现在新开了广告账户，是否需要发邮件申请marketing API权限？</strong>

A：之前已发过申请邮件，且权限已开通；客户端ID没有更新，仅回调地址更新；仅新开广告账户/经理账户，则不需要重新发邮件申请。
