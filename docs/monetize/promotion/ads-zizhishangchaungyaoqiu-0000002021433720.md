# 资质文件规范性要求

## 《营业执照》上传要求解读

1. <strong>市场主体变更登记涉及营业执照记载事项的，登记机关会为市场主体换发营业执照，此时旧执照与国家信用信息公示系统查询的信息不一致，证照无效，需上传最新的营业执照至广告平台。</strong>

   <strong>以营业执照上企业住所已变更为例</strong>：

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 国家企业信息公示系统上的企业住所为“XXX大街 99号A511B-Y214”，而营业执照上的住所为“XXX大街99号A725-Y214”，该企业住所已变更，需上传最新的营业执照，不予通过。 | 国家企业信息公示系统上的企业住所为“XXX大街99号A511B-Y214”与营业执照上的住所为“XXX大街99号A511B-Y214”一致，可通过。 |
2. <strong>市场主体被吊销营业执照、责令关闭、撤销，或者被列入严重违法失信企业名单，不支持开户。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 国家企业信息公示系统上查询该企业显示“已吊销 被列入经营异常名录”暂不支持开户，不予通过。 | 国家企业信息公示系统上查询该企业显示“存续（在营、开业、在册）”无经营异常，支持开户，可通过。 |
3. <strong>不支持使用《电子营业执照》截图进行开户申请，</strong> <strong>可上传</strong> <strong>《营业执照》原件、拍照件、扫描件或加盖公章的黑白复印件。</strong>

   <strong>注：电子营业执照的正确使用方式是，经营主体办事人员获得电子营业执照使用授权后，通过出示二维码的方式，需平台支持电子营业执照的扫码验证。而非直接通过保存图片、截图保存等方式提供电子营业执照的图片。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 暂不支持使用电子营业执照开户，不予通过。 | 支持使用营业执照原件（营业执照原件包含正本和副本）拍照件开户，可通过。 |
4. <strong>鲸鸿动能广告平台目前仅支持企业注册广告账户，暂不支持个体工商户进行广告投放。（政策如有变更将另行通知）</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 该市场主体所提供营业执照中类型字段为“个体工商户”，按照当前广告平台的开放政策，暂不支持，不予通过。 | 该市场主体所提供营业执照中类型字段为“有限责任公司（自然人投资或控股）”，支持该类型市场主体开户，可通过。 |
5. <strong>市场主体在鲸鸿动能进行相关活动（如推广产品/服务，或成为服务商代理推广产品/服务）时，营业执照需含有对应的经营范围。</strong>

   <strong>以注册服务商账户为例：</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 该市场主体注册服务商账户，代理广告主投放广告，所提供的营业执照中并未含有相关的经营范围，不予通过。 | 该服务商提供的营业执照中，经营范围含有“广告设计、代理”，可代理广告主投放广告，可通过 |

## 法定代表人身份证上传要求解读

1. <strong>身份证上的“姓名”必须与《营业执照》中的“法定代表人”一致。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 该企业的法定代表人是“张三”，提供的是“李四”的身份证，不予通过。 | 该企业的法定代表人是“张三”，提供的是“张三”的身份证，可通过。 |
2. <strong>如有换领、换发或者补领新证的，旧的身份证就不再具备法律效力，需上传最新领取的有效身份证。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 该法定代表人提交的身份证，是2017年在北京市办理的身份证，经审核查得，该公民在2025.02.13在长沙市重新办理的新的身份证，旧的身份证即使未超过有效期，但已不再具备法律效力，不予通过。 | 上传2025.02.13在长沙市重新办理的新的身份证，具备法律效力，可通过。 |
3. <strong>不支持使用《电子身份证》截图进行开户申请，可上传身份证原件、拍照件、扫描件或加盖公章的黑白复印件。</strong>

   <strong>电子身份证正确使用方式是，法定代表人领取电子身份证后，通过出示二维码进行扫描核验，需平台支持电子身份证的扫码验证。而非</strong> <strong>直接通过保存图片、截图保存等方式提供电子身份证的图片。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  | 汉文身份证（下图为样证，非真实居民的身份证）：    少数民族文字身份证（下图为样证，非真实居民的身份证）： |
   | 暂不支持使用电子身份证，包括但不限于以上两种电子身份凭证，均不予通过。 | 提交身份证原件，且清晰体现姓名、有效期、身份证号码等有效信息，可通过。 |
4. <strong>禁止使用未成年人的身份证</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 该法定代表人提交的身份证面相稚嫩，结合出生日期可判断为未成年，使用未成年身份证，均不予通过。*（上图为样证，非真实居民的身份证）* | 提交成年人身份证正反面，且符合其他资质上传规范，可通过。 |

## 资质文件上传要求解读

1. <strong>应按照系统提示的位置上传资质。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 在提示“请上传《计算机软件著作权登记证书》”的位置上传的是游戏版号，不予通过。 | 已按照系统提示的正确位置上传相对应的资质文件，可通过。 |
2. <strong>可上传资质原件、拍照件、扫描件，当所出示的资质文件是黑白复印件时，需加盖广告主鲜章。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 该资质为黑白复印件，未加盖广告主的鲜章，不符合规范要求，不予通过。 | 该资质为加盖了广告主鲜章的黑白复印件，符合规范要求，可通过。 |
3. <strong>加盖的企业公章需与证件中的主体保持一致。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 资质证件主体为A，加盖的企业公章是B，两者不一致，不予通过。 | 资质证件主体和公章主体均为A公司，符合规范要求，可通过。 |
4. <strong>不支持上传资质截图/摄屏等无效证件，可上传</strong> <strong>资质原件、拍照件、扫描件或加盖公章的黑白复印件。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 广告主上传的是截图/摄屏证件，不予通过。 | 广告主提供的是资质原件，符合规范要求，可通过。 |
5. <strong>所上传的资质文件需清晰可阅读，文字过于模糊无法阅读并且也无法成功扫描二维码辅助阅读的，无法进行审核。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 该广告主提供的资质文件过于模糊，无法阅读辨认，不予通过。 | 该资质图片完整清晰可辨识，符合资质上传规范，可通过。*（<strong>上图为参考图片，为避免出现企业的隐私信息，已做毛玻璃马赛克处理</strong>）* |
6. <strong>需上传完整的资质文件，不应有遮挡或者缺页。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 图中《食品经营许可证》不完整，遮挡了部分信息，不符合规范要求，不予通过。 | 完整上传《食品经营许可证》，未出现遮挡缺页等问题，符合资质上传规范，可通过。 |
7. <strong>请勿在资质文件中添加无效水印，包括但不限于：限制使用时间&lt;=提交当天、限制使用场景与推广无关、限制仅可使用的平台未包含鲸鸿动能等；</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 该肖像使用授权书中含有“仅供A广告平台投放使用”，不符合鲸鸿动能平台开户使用场景，不予通过。 | 该《计算机软件著作权登记证书》中含有“仅供广告投放使用”，并未限定某一广告平台，符合广告平台开户使用场景，可通过。 |
8. <strong>同一份文件中，不允许含有2个或以上限制了不同用途的水印。在已有指明用途的文件上再次添加其他用途的水印，已改变该文件原有的使用用途，不予支持。</strong>

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 该企业法定代表人身份证正反面存在两个内容不同的水印，需重新上传单水印合规文件，不予通过。 | 该企业法定代表人身份证正反面只有一个水印，符合资质文件规范，可通过。 |
9. <strong>当文件水印中明确了该文件的使用期限，则应提交该文件的日期也应当在有效期内。</strong>

   <strong>以提交时间在2024年11月21日为例</strong>：

   | 错误资质示例 | 正确资质示例 |
   | --- | --- |
   |  |  |
   | 资质水印有效期至2021年12月15日，提交时间为2024年11月21日，文件已过期，不予通过 | 资质水印有效期至2024年12月18日，提交时间为2024年11月21日，水印有效期内，且未限制在本平台无关的范围，符合资质文件规范，可通过。 |
10. <strong>当文件水印中明确“仅一次使用”，原则上该水印不应该超出1年。</strong>

    | 错误资质示例 | 正确资质示例 |
    | --- | --- |
    |  |  |
    | 该文件印章注明“本件仅限于申请开发者账号和投放账号<strong>一次使用</strong>”日期显示为2021年，申请开户为2024年，跨年使用该文件无效，不予通过。 | 该文件印章注明“本件仅限于办理本次，仅限于办理开通投放账户，他用无效一次使用（2024-04-15）”申请开户为2024年，在本年度使用于广告投放，且符合其他资质规范，可通过。 |
11. <strong>所提供的资质文件需在有效期内，有效期届满，需继续从事经营的，应当于相关法律规定的有效期届满提前日期申请续办。</strong>

    <strong>以《网络文化经营许可证》为例：《互联网文化管理暂行规定》第九条规定，《网络文化经营许可证》有效期为3年。有效期届满，需继续从事经营的，应当于有效期届满30日前申请续办。</strong>

    | 错误资质示例 | 正确资质示例 |
    | --- | --- |
    |  |  |
    | 该资质文件有效期至2021年9月25日止，申请开户为2024年，已过期。资质无效不予通过。 | 该资质文件有效期至2025年10月16日止，申请开户为2024年，在有效期内，且符合其他资质规范可通过。 |
12. <strong>当经营主体有变更的情况下，应该按照相关法律规定向监管机关换领新证。</strong>

    <strong>以《增值电信业务经营许可证》为例：《互联网信息服务管理办法》第十一条规定，互联网信息服务提供者变更服务项目、网站网址等事项的，应当提前30日向原审核、发证或者备案机关办理变更手续。</strong>

    | 错误资质示例 | 正确资质示例 |
    | --- | --- |
    |  |  |
    | 该资质文件审核时企业法定代表人已变更，应30日内向原发证登记机关办理变更手续，上传旧资质文件不予通过。 | 该资质文件审核时企业法定代表人已变更，上传变更后换发的新资质文件，可通过。 |
13. <strong>注册账户时，在“推广内容”中填写的产品，应提供该产品对应的资质。</strong>

    | 错误资质示例 | 正确资质示例 |
    | --- | --- |
    |  |  |
    | 该账户本次推广内容填写的是“良品严选”APP与上传的资质文件《计算机软著证书》对应软件名称为“电子商城应用服务系统”不匹配，不予通过。 | 该账户本次推广内容“枪战荣耀吃鸡”应用与《计算机软著证书》软件名称“枪战荣耀吃鸡软件”一致，可通过。 |
14. <strong>推广的产品/服务与广告主无关，若推广非开户企业所有的产品，须提供相关的合作协议、供货协议或授权书等。</strong>

    <strong>以应用授权代理推广为例</strong>：

    | 错误资质示例 | 正确资质示例 |
    | --- | --- |
    |  |  |
    | 该账户广告主为“南京xx科技有限公司”，推广内容为“奔跑吧”游戏软件，软件对应的《计算机软著证书》其著作权人显示为“上海xx科技有限公司”非本广告主，故推广产品与广告主无关，没有提供与本广告主的关联证明或产品合作协议等说明文件，不予通过。 | 该账户已补充对应的《推广授权书》，证明广告主为该应用软著著作权人授权代理推广的合法被授权人，可通过。 |
15. <strong>协议类文件需体现出首页合作双方、与本次推广有关的合作内容、内页合作期限、末页合作双方盖章。</strong>

    | 错误资质示例 | 正确资质示例 |
    | --- | --- |
    |  |  |
    | 该协议文件仅提供了末页甲乙双方盖章，未体现出合作内容及合作期限，不予通过。 | 该协议文件体现了首页合作双方、与本次推广有关的合作内容、内页合作期限、末页合作双方盖章，符合要求，可通过。 |
16. <strong>为了避免应用下载或促活失败影响广告投放效果，应推广Android 或鸿蒙应用，《计算机著作权登记证书》对应的版本需要与您申请推广的应用一致为Android 或鸿蒙版本</strong>。

    | 错误资质示例 | 正确资质示例 |
    | --- | --- |
    |  |  |
    | 上传的《计算机著作权登记证书》对应版本非Android 或鸿蒙应用，不予通过。 | 上传的《计算机著作权登记证书》对应版本为安卓版本/或不限制版本，可通过。 |
