# FAQ

## 为什么创建API客户端时报错403？

创建API客户端时项目应选择“N/A”，角色选择“管理员”，否则会鉴权失败导致报403错误。

## 调用查询统计报表接口或查询推广任务列表时，customerId必须携带吗？如何获取？

查询统计报表接口时，请求不需要携带customerId；查询推广任务列表时，直客投放模式不用携带，客投伙伴投放模式必须携带。

查询方式为：使用投放操作账户登陆[华为应用市场应用推广平台](https://developer.huawei.com/consumer/cn/service/apcs/app/home.html)，在“我的账号信息”中查看到的“华为账号”就是customerId。

## 查询统计报表接口返回\{"code":20770003,"msg":"too many effective tasks"\}的原因是什么？

原因为请求体中没有添加过滤条件，请添加过滤条件后查看返回是否正常。

## 调用查询统计报表接口获取的数据和投放平台报表数据是完全一致的吗？

是完全一致的，如不一致请详细描述并[提交工单](https://developer.huawei.com/consumer/cn/support/feedback/#/)反馈。

## 接口返回的记录条数是否有限制？

当前接口一次返回记录数不能超过1000个，超过的话要通过增加过滤条件减少返回记录数。

## 调用查询搜索类推广任务统计报表接口，返回\{"code": 20770003,"msg": "task list empty after filter"\}的原因是什么？

此返回报错是因为查询的推广任务为影子投放或OCPD任务，目前查询搜索类推广任务统计报表接口暂不支持影子投放任务和OCPD任务数据查询。

## 应用推广平台的报表中，整体报表数据，子任务报表数据，搜索类推广报表数据和Marketing API接口如何对应？

当应用推广平台查询报表时的过滤条件与接口Body中的传参一致时：

- 整体报表数据：与Marketing API的/ad接口返回数据对应。
- 子任务报表数据：与Marketing API的/ad/subtask接口返回数据对应。
- 搜索类推广任务报表：与Marketing API的/searchad接口返回数据对应。

## api接口的频次限制具体是怎么样的，为什么4次/秒就报频次限制？

api接口是做的用户级频次限制，同一个用户下调用的所有的接口加在一起不能超过10次/秒，搜索类报表是2-5次/20min。

## “查询账户余额”、“查询账户消耗明细”、“修改日预算和次日预算”和“查询日预算和次日预算”接口什么时候刷新数据？

其中“查询账户消耗明细”是T+1凌晨四点可以查询前一天的数据。其余三个都是实时查询数据。

## 为什么有的搜索任务归因信息中有子任务id，有的搜索任务归因信息中没有子任务id？

搜索词子任务和搜索ocpd通投子任务暂时不支持归因，其他类型任务支持子任务归因。

## 为什么查询推广子任务统计报表接口有时候返回子任务id，有时候不返回？

子任务报表接口，按照子任务分组，返回的任务id为0；按照任务id分组，就不返回子任务id。所以要看具体分组。
