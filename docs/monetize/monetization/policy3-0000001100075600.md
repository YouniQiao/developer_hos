#### 禁止无效的流量和行为

无效流量指通过作弊手段，导致广告客户费用或发布商收入虚增的点击或展示。而导致无效流量产生的活动称为无效活动。平台禁止任何不正当方式制造虚假或无效流量、点击量、下载量，或违规嵌入等行为，包括但不限于以下形式：

* **违规推广流量**
  + 禁止通过自行、强制、诱导、或程序访问、自动刷新等相关手段进行无效，或隐藏的推广行为，来赚取推广流量。
* **修改推广内容**
  + 将用户点击的推广页面重定向到别处，与最终用户直接访问推广页所看到的页面不同。
  + 对由推广内容任一部分、副本或衍生信息，采用“抓取”、“spider”、索引或任何其他方式加以存储或缓存。
  + 推广内容显示后，改变推广内容属性，比如：改变推广位置、推广内容大小等推广元素的各类属性。
* **诱导、强制用户点击**
  + 在非激励型广告合作形式中，对用户点击行为进行资源交换，奖励。
  + 强制用户点击推广提高用户使用应用的次数或者继续使用应用。
  + 遮挡、隐藏推广显示区域的任何部分，包括logo、链接、描述等，造成推广内容与页面内容发生重叠。
* **反编译SDK**
  + 禁止任何形式对鲸鸿动能广告SDK、华为移动基础服务SDK进行反编译，不得对SDK接口进行内容、功能、逻辑及界面的任何修改。

#### 展示位设置要求

**通用规则**

我们鼓励发布商测试不同的展示位置和广告形式，但必须遵守以下关于展示位设置的要求：

* **请求要求**
  + 禁止设置定时器循环请求广告。
  + 禁止失败后重复请求。
  + 禁止应用在后台运行时持续请求广告。
  + 不允许广告折叠展示（在一个物理广告位置上，叠加了多个广告展示位）。
  + 不允许存在请求成功不展示的情况。
  + 为发展健康生态，广告平台会定期清理低质量广告位。
* **展示要求**
  + 展示位置不应遮挡或盖住用户有可能看到的任何正常内容区域，不应放在任何用户容易误触的区域。
  + 广告不能放置在没有返回或关闭选项的页面，必须让用户不用点击广告也能退出屏幕，提供用户自主关闭广告的选项。
  + 不允许点击关闭按钮后，广告未关闭的现象。
  + 禁止点击除下载按钮外的区域，出现有下载的现象。
  + 没有任何内容或不以内容为主的屏幕上应避免展示广告。
  + 在设备后台或在应用环境之外运行的应用中不应请求广告。应让用户能清楚判断与广告关联的具体应用或投放广告的具体应用。
  + 广告素材全尺寸（画面不能隔断，完整展示广告素材）、等比例展示（不能将素材压缩）。
  + 禁止出现用户点击广告素材后，无法跳转落地页情况。
* **防止无效曝光**
  + 一次请求的广告不能重复展示，展示完后需重新实时获取。
* **防止误点击**
  + 除开屏外，其他展示位的可点区域仅限于广告图片、标题、按钮等素材。
  + 不应将广告背景设置为可点区域，只允许广告区域点击跳转落地页。

**开屏规则**

* 不允许把非开屏广告（例原生广告）部署在开屏的位置。

* 展示完毕后自动关闭并进入应用的主界面。
* 进入落地页后返回时，直接进入应用。
* 开屏仅允许打开app时展示。

**Banner规则**

* 建议调用轮播接口setBannerRefresh()，并设置轮播间隔为30s-120s。
* 在设备后台或在应用环境之外运行的应用中不应请求广告。
* 不允许将上一个展示广告进行缓存，用于下一次广告展示，保证广告实时请求实时展示，建议广告设置为60s自动刷新一次。
* 不应将横幅广告放置于文本、图片和应用的其他可点击部分，与其他内容周围应留出适合空间，避免误点。
* 禁止在onAdLoaded或者onAdFailedToLoad中控制Banner广告的显示。
* 避免与其他类型广告同时展示或重叠，如与插屏同时出现在一个页面，导致无效曝光。
* 适配机型：若竖屏展示，需考虑刘海屏和水滴屏是否遮挡广告展示。

**插屏规则**

* 插屏广告不应突然出现，不应干扰用户使用应用的正常流程，例如应用加载，退出应用、游戏开始时或者游戏过程中。
* 插屏广告场景应为应用功能自然体验中断，暂停。

**原生规则**

* 原生广告需在应用的内容流中与应用内容穿插展示，在游戏的暂停、结束或者等待过程中展示。
* 原生广告不应突然出现，不应放在任何用户容易误触的区域，需提供用户自主关闭广告的选项。
* 不允许原生广告全屏展示。
* 广告素材、广告标识、广告来源、广告标题（至少预留22个汉字字符位置，含标点符号）和关闭按钮、下载按钮必须清晰完整展示。
* 原生广告视图的一角需展示**“****为什么看到此广告****”**的图标**“i”**或**“****不再显示该广告****”**的图标**“x”。**

  ![](./img/policy3-0000001100075600_0.png)

  以下两种方案可二选一：

  方案一：点击图标“x”后，出现“隐藏广告”和“为什么看到此广告”的视图，就不用做“i”图标；

  方案二：仅做直接关闭广告的“x”图标，就保留一个“i”图标。
* 关闭按钮与安装按钮上下分离，关闭按钮与广告区域有适当距离。
* 关闭按钮与周边区域的颜色需区分，可设置白边承载按钮，使按钮明显可见。
* 将关闭按钮和周边区域设置一个可点击关闭的热区（32dp\*32dp大小的矩形区域）。
* 用户关闭落地页后或从落地页返回媒体界面，广告应不再继续展示。
* 禁止调用负反馈接口nativeView.onClose() 。

**激励视频规则**

* 获取广告接口与展示广告接口调用的场景化时间差建议不高于6s。
* 如请求无广告返回，则不向用户展示激励视频入口，不可出现有入口而无广告展示的情形。
* 用户观看完整的视频后，可获得应用内激励。
* 展示广告前向用户说明激励广告规则，明确告知用户观看完成视频后才能获得相应奖励。
* 不可通过提供金钱补偿或可转让给他人的激励形式诱导用户点击。

* 允许在前一个视频播放中请求下一个广告缓存，保证一次请求的广告仅展示一次。
* 需有机制保证用户短时间内重复点击广告入口不会触发多次展示。
* 禁止在用户没有点击激励视频广告按钮的条件下，自动展示激励视频广告。
