#### 请求与展示规范

* 禁止设置定时器循环请求广告。
* 禁止失败后频繁重复请求。
* 一次请求的广告不能重复展示，展示完后需重新实时获取。
* 预缓存的广告如激励视频，请注意缓存时间，超过一个小时需重新请求新的广告来展示，否则计为无效展示不计费结算。
* 广告位置不应遮挡或盖住用户有可能看到的任何正常内容区域，不应放在任何用户容易误触的区域。
* 广告不能放置在没有返回或关闭选项的页面，必须让用户不用点击广告也能退出屏幕，提供用户自主关闭广告的选项。
* 没有任何内容或不以内容为主的屏幕上应避免展示广告。
* 在设备后台或在应用环境之外运行的应用中不应投放广告，应让用户能清楚判断与广告关联的具体应用或投放广告的具体应用。
* 同屏内不可展示两个及以上广告。
* 广告素材全尺寸等比例展示。
* 除开屏外，其他广告位的可点区域仅限于广告图片、标题、按钮等素材。
* 不应将广告背景设置为可点区域，只允许广告区域点击跳转落地页。
* 禁止通过其他违规手段进行广告请求与展示。
* 为发展健康生态，广告平台会定期清理低质量广告位。

#### 广告样式规范

* 应用榜单/搜索（应用图标216\*216尺寸icon）
  + 广告区域与其他内容周围应留出适合空间，避免误点或重叠展示， 如与其他广告形式同时出现在一个页面，导致无效曝光。
  + 适配机型：若竖屏展示，需考虑刘海屏和水滴屏是否遮挡广告展示。
* 开屏广告
  + 展示完毕后自动关闭并进入应用的主界面。
  + 进入落地页后返回时，直接进入应用。
* 横幅（banner）广告
  + 建议设置广告自动刷新，并设置刷新时长为60s。
  + 不应将横幅（banner）广告放置于文本、图片和应用的其他可点击部分，与其他内容周围应留出适合空间，避免误点。
  + 避免与其他类型广告同时展示或重叠，如与插屏同时出现在一个页面，导致无效曝光。
  + 适配机型：若竖屏展示，需考虑刘海屏和水滴屏是否遮挡广告展示。
* 插屏广告
  + 插屏广告不应突然出现，不应干扰用户使用应用的正常流程，例如应用加载中。
  + 插屏广告场景应为应用功能自然体验中断，暂停，或者自然时间间隔之间。
* 原生图标/创意广告
  + 原生广告需在应用的内容流中与应用内容穿插展示。
  + 广告素材、广告标识、广告来源、广告标题（至少预留22个汉字字符位置，含标点符号）和关闭按钮、下载按钮必须清晰完整展示。
  + 关闭按钮与安装按钮上下分离，关闭按钮与广告区域有适当距离。
  + 关闭按钮与周边区域的颜色需区分，可设置白边承载按钮，使按钮明显可见。
  + 将关闭按钮和周边区域设置一个可点击关闭的热区（32dp\*32dp大小的矩形区域）。
  + 用户关闭落地页后，广告不再继续展示。
* 激励视频
  + 如请求无广告返回，则不向用户展示激励视频入口，不可出现有入口而无广告展示的情形。
  + 用户观看完整的视频后，方可获得应用内激励，但可随时退出观看视频。
  + 展示广告前向用户说明激励广告规则，明确告知用户观看完整的视频后才能获得相应奖励。
  + 不可通过提供金钱补偿或可转让给他人的激励形式诱导用户点击。
  + 允许在前一个视频播放中请求下一个广告缓存，保证一次请求的广告仅展示一次。
  + 需有机制保证用户短时间内重复点击广告入口不会触发多次展示。
* 互推盒子
  + 互推盒子不应突然出现，不应干扰用户使用应用的正常流程，例如应用加载过程中。
  + 快应用退出时允许出现一次互推盒子广告，关闭广告后需直接退出快应用，不应再弹出加桌提示。
